# LeetCode 刷题策略

这份策略根据你提供的图片整理，作为仓库内统一参考的刷题方法说明。

## 刷题节奏

- 不要只追求零散做题，要按题型系统化推进

## 要刷多少题

- 每个类型建议刷 `10~20` 题
- 动态规划建议多刷一些，尽量多覆盖
- 总量建议做到 `200~300` 题

## 如何刷同类题

- 同类型题目放在一起刷
- 可以按天安排专题，例如：
  - 周一：树 / 链表
  - 周二：搜索
  - 周三：动态规划

## 三遍法

### 第一遍

- 先自己想
- 想 `5` 分钟还没有思路，就直接看答案

### 第二遍

- 尝试不看答案，完整实现
- 一道题尽量不要超过 `60` 分钟

### 第三遍

- 尝试默写或快速重做
- 如果 `15~20` 分钟还实现不出来，就再次看答案

具体做法：

- 每道题至少看 `3~5` 种不同实现
- 分析别人的代码：
  - 优缺点是什么
  - 为什么有的实现更快
  - 为什么有的实现更慢
- 借这个过程学习：
  - 新语言
  - 算法写法
  - 数据结构
  - API 用法
  - 模板
  - 最佳实践

## 刷题过程中要培养的能力

### 1. 扩展能力

- 至少能看出一道题适合用什么算法

### 2. 复杂度意识

- 能根据数据规模推断时间复杂度

### 3. 代码表达能力

- 保持代码风格一致
- 注意命名、缩进、括号、换行
- 尽量使用有意义的变量名

## 在这个仓库里的落地方式

- `top100` 这条线按题单顺序逐题推进
- `java2python` 这条线按已有 Java 题解做迁移整理
- 每道题都尽量补齐题意、思路、代码、复杂度、易错点和 demo
- 不只做出答案，还要能讲清楚为什么这么做
