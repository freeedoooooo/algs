# 9063 单位收款方启用状态优化 需求文档

## 背景
当前单位收款方管理功能存在以下问题：
1. 新增或编辑收款方后，启用状态不会自动开启，需要手动操作
2. 已启用的收款方仍然显示删除按钮，可能导致误删除

## 目标用户
财务管理人员

## 功能描述
优化单位收款方管理的启用状态和删除逻辑：
1. 新增或编辑收款方保存后，自动将启用状态设置为"已启用"
2. 列表页中，已启用的收款方隐藏删除按钮

## 所属模块
- 前端：a-front
- 后端：fin-track-ms（端口 18615）

## 用户故事
- 作为财务管理人员，我希望新增或编辑收款方后自动启用，以便减少手动操作
- 作为财务管理人员，我希望已启用的收款方不显示删除按钮，以便避免误删除

## 验收标准
- [ ] 标准1：境内收款方新增后，启用状态自动为"已启用"
- [ ] 标准2：境内收款方编辑保存后，启用状态自动为"已启用"
- [ ] 标准3：境外收款方新增后，启用状态自动为"已启用"
- [ ] 标准4：境外收款方编辑保存后，启用状态自动为"已启用"
- [ ] 标准5：境内收款方列表，已启用的记录隐藏删除按钮
- [ ] 标准6：境外收款方列表，已启用的记录隐藏删除按钮
- [ ] 标准7：境内收款方列表，未启用的记录显示删除按钮
- [ ] 标准8：境外收款方列表，未启用的记录显示删除按钮

## 边界条件
- 当收款方被禁用后，删除按钮应重新显示
- 当通过开关切换为禁用状态后，删除按钮应显示

## 非功能需求
- 无特殊性能要求
- 无特殊安全要求

## 技术分析

### 涉及文件
**前端**：
- `ars-front-2/apps/a-front/src/views/fin-track/spendAllocation/payeeManagement/index.vue` - 列表页

**后端**：
- `ars-fin-track/fin-track-ms/src/main/java/com/dibcn/ars/finTrack/service/spendAllocation/impl/PayeeManagementServiceImpl.java` - Service 实现

### 数据结构
- 表名：`t_payee_management`
- 启用字段：`c_is_enabled`（"0" = 未启用，"1" = 已启用）
- 类型字段：`c_type`（"0" = 境内，"1" = 境外）

### API 接口
- 保存接口：`POST /api/v1/payee-management/save`
