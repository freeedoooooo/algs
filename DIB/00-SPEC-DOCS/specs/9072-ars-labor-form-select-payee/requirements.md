# 9072 劳务费信息新增"选择收款方"按钮 需求文档

## 背景
当前劳务费报销表单中的"劳务费信息"表格，点击"新增"按钮会弹出收款方选择弹窗。用户希望将"选择收款方"功能独立为一个按钮，同时保留"新增"按钮用于直接添加空行。

## 目标用户
财务人员、报销申请人

## 功能描述
1. 在"劳务费信息"表格右上角新增一个"选择收款方"按钮
2. 原有的"新增"按钮保留，功能改为直接添加空行

## 所属模块
- 前端：a-front
- 涉及页面：费用报销 - 劳务费报销表单

## 用户故事
- 作为报销申请人，我希望能通过"选择收款方"按钮从收款方库中选择已有收款方，以便快速填写劳务费信息
- 作为报销申请人，我希望能通过"新增"按钮直接添加空行手动填写，以便录入新的收款方信息

## 验收标准
- [x] 标准1：劳务费信息表格右上角显示"选择收款方"按钮，位于"导入"按钮左侧
- [x] 标准2：点击"选择收款方"按钮弹出收款方选择弹窗，选择后添加到表格
- [x] 标准3：原有"新增"按钮保留在原位置（表格底部）
- [x] 标准4：点击"新增"按钮直接在表格中添加一个空行，不弹出弹窗
- [x] 标准5：禁用状态下（查看模式）"选择收款方"按钮不显示
- [x] 标准6：选择收款方后不清空手动新增的数据行

## 边界条件
- 当表单处于禁用/查看状态时，"选择收款方"按钮应隐藏
- 选择收款方后，已选择的收款方应正确回显到表格中
- 选择收款方后，不清空手动新增的数据行（payeeId 为空的行）

## 非功能需求
- 按钮样式与结算信息的"选择收款方"按钮保持一致（type="primary"）

## 待确认问题
- [x] 按钮位置：选择收款方在导入按钮左侧 ✓
- [x] 新增按钮位置：保持原位置（表格底部） ✓
