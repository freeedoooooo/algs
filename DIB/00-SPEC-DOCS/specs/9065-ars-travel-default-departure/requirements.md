# 9065 事前申请-出差默认出发地点 需求文档

## 背景

在事前申请-差旅费申请中，用户每次填写行程信息时都需要手动选择出发地点。对于大多数用户来说，出发地点通常是固定的（如公司所在地）。为了提升用户体验，减少重复操作，需要支持配置一个默认的出发地点。

## 目标用户

- 需要填写差旅费申请的普通员工
- 系统管理员（配置默认地址参数）

## 功能描述

在系统参数中配置一个默认出发地点的地区代码，当用户新建差旅费申请的行程时，出发地点自动填充为配置的默认值。如果未配置该参数，则出发地点为空，由用户自行选择。

## 所属模块

- 前端：a-front
- 后端：common-ms（参数服务，已有，无需修改）

## 用户故事

- 作为普通员工，我希望新建差旅行程时出发地点能自动填充默认值，以便减少重复选择操作
- 作为系统管理员，我希望能配置默认出发地点，以便统一管理公司员工的出差起点

## 验收标准

- [ ] 标准1：系统参数中新增 `ft_travel_default_departure` 参数，值为完整的地区层级代码（如 `110000,110100,110101`）
- [ ] 标准2：新建差旅行程时，如果配置了默认出发地点参数，出发地点自动填充为配置值
- [ ] 标准3：如果未配置默认出发地点参数或参数值为空，出发地点保持为空，用户自行选择
- [ ] 标准4：自动填充的出发地点用户可以手动修改
- [ ] 标准5：编辑已有行程时，不覆盖已保存的出发地点值

## 边界条件

- 当配置的地区代码无效或不存在时，系统应忽略该配置，出发地点保持为空
- 当配置的地区代码格式不正确时，系统应忽略该配置，出发地点保持为空
- 当用户点击"新增"按钮添加新行程时，应自动填充默认出发地点

## 非功能需求

- 性能：获取默认参数应在页面加载时异步完成，不影响页面渲染速度
- 安全：无特殊安全要求

## 待确认问题

- [x] 参数配置方式：使用后台 common-ms 的 base-data 参数服务
- [x] 地区代码格式：完整的层级代码，用逗号分隔（如 `110000,110100,110101`）
- [x] 用户是否可修改：是，用户可以修改自动填充的值
- [x] 是否需要后端支持：否，纯前端实现，参数服务已有
