## 注意事项

### 1. 正则表达式说明
- **修改前**：`/^[0-9]\d{9,29}$/` - 10-30 位数字
- **修改后**：`/^[0-9]\d{8,29}$/` - 9-30 位数字
- **解释**：
  - `^` - 字符串开始
  - `[0-9]` - 首位可以是 0-9（**包括 0**）
  - `\d{8,29}` - 后续 8-29 位数字（总共 9-30 位）
  - `$` - 字符串结束

### 2. 首位可以为 0
与之前的理解不同，银行账号的首位**可以为 0**。这是符合实际业务需求的，因为某些银行的账号确实以 0 开头。

### 3. 验证范围
- **最小长度**：9 位（如 `012345678`）
- **最大长度**：30 位
- **允许字符**：仅数字 0-9
- **首位限制**：无限制，可以为 0

### 4. 影响模块
- 合同管理：合同起草、相对方准入、供应商准入
- 采购管理：询价、订单、入库、验收
- 财务管理：账户管理
