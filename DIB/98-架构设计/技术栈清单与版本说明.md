# 技术栈清单与版本说明

## 1. 汇报范围

本文面向技术管理汇报，说明当前平台底座所使用的主要技术栈、版本级别和选型定位。

说明原则：

- 以前端统一技术口径为主，不展开老旧前端历史栈
- 版本优先采用“主版本级别 + 当前代码基线”的表达方式
- 对已确认存在的国产化兼容能力单独列出

## 3. 前端技术栈

### 3.1 建设定位

当前前端采用微前端化组织方式，以 `app-nav` 作为前端平台/壳层，叠加多个业务应用和共享包形成统一前端底座。

主要应用：

- `app-nav`：前端平台、导航与壳层入口
- `app-system`：系统管理应用
- `app-report`：报告业务应用
- `app-data`：数据业务应用

主要共享包：

- `shared-utils`
- `shared-components`
- `shared-composables`
- `shared-config`

### 3.3 工程特征

- 前端通过 `Nginx` 承载各模块静态资源
- 通过统一域名入口和反向代理对接后端 API
- `app-nav` 承担导航、壳层与统一入口能力，`app-data`、`app-report`、`app-system` 在其之上承载业务能力
- 共享包承担跨应用通信、状态共享、公共组件、公共构建配置等能力
- 微前端开发阶段支持统一开发网关与独立应用并行调试

## 6. 数据库与国产化兼容

### 6.1 当前口径

平台当前以关系型数据库为核心数据底座，主流部署场景以 `MySQL 8.x` 为主，同时已具备国产数据库兼容能力。

|---------------------|---------------|-----------------------------------------------------------|

### 6.3 汇报建议

- 平台当前以 `MySQL` 为主流数据库底座
- 在信创项目中，已具备 `DM8 / GBase` 的技术适配能力
- 部分项目已经完成国产数据库接入，说明平台具备面向不同客户环境的落地能力

## 8. 选型判断与关注点

### 8.1 主要特点

- 前后端分层明确，平台底座与业务服务边界清晰
- 前端多应用与共享包结合，便于按业务域持续扩展
- 后端基于公共父工程与 starter 体系，复用能力较强
- 已形成从接入、安全、主数据、业务能力到监控的一套完整基础底座
- 已具备国产化数据库兼容基础，满足信创延展需求

### 8.2 当前关注点

- Spring Boot 基线还存在版本口径并存，需要后续统一
- 部分前端和历史工程仍带有演进痕迹，长期需要继续收敛
- 国产数据库兼容已经具备基础，但仍建议继续形成标准化验证清单

## 9. 演进建议

- 统一前端技术基线与工程规范，减少历史栈差异
- 统一后端父依赖与框架版本，降低长期维护成本
- 继续沉淀信创数据库适配模板，形成项目级交付标准
- 进一步平台化 Nacos、Redis、MinIO、监控等基础设施接入规范
