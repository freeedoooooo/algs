## 一句话结论
这题要设计一个 `RandomPool` 结构，同时支持 `insert`、`delete`、`getRandom` 三个操作，并且都要做到 `O(1)`。
核心做法是“数组 + 两张哈希表”或“动态数组 + 值到下标映射”：随机取数靠数组，删除时用最后一个元素补洞，哈希表负责在 `O(1)` 时间维护位置关系。

## 核心知识点
- `getRandom()` 想做到等概率且 `O(1)`，底层必须能按下标随机访问。
- `insert/delete` 想做到 `O(1)`，必须能在常数时间定位元素所在位置。
- 所以常见组合是：数组负责随机访问，哈希表负责值与下标的双向映射。
- 删除时不能真的把数组中间元素整体左移，否则会退化成 `O(N)`。
- 正确删除方式是“末尾覆盖待删位置 + 更新映射 + 删除末尾”。

### 题目二
设计 `RandomPool` 结构

【题目】

设计一种结构，在该结构中有如下三个功能：

- `insert(key)`：将某个 `key` 加入到该结构，做到不重复加入
- `delete(key)`：将原本在结构中的某个 `key` 移除
- `getRandom()`：等概率随机返回结构中的任何一个 `key`

【要求】

- `insert`、`delete` 和 `getRandom` 方法的时间复杂度都要是 `O(1)`

把它补成完整说明，就是：

- 结构中维护一批不重复元素
- 支持插入、删除、随机返回任意元素
- 随机返回时必须保证当前所有元素被选中的概率相同
- 三个操作都不能退化成线性时间

## 图解
可以把这个结构理解成两部分：

### 1. 一个连续数组

```text
index: 0   1   2   3
value: a   b   c   d
```

这样 `getRandom()` 时，只要随机生成：

```text
0 ~ size-1
```

中的一个下标，就能等概率取到任意元素。

### 2. 一张值到下标的表

```text
a -> 0
b -> 1
c -> 2
d -> 3
```

这样删除某个值时，就能立刻知道它在数组里的位置。

### 3. 删除为什么要“末尾覆盖”
假设现在要删掉 `b`：

```text
index: 0   1   2   3
value: a   b   c   d
```

如果直接把 `b` 后面的元素都左移：

```text
a c d
```

虽然能删掉，但左移是 `O(N)`。

正确做法是：

1. 取出末尾元素 `d`
2. 用 `d` 覆盖 `b` 原来的位置
3. 删除末尾
4. 更新映射

过程变成：

```text
原数组: [a, b, c, d]
删除 b 后:
[a, d, c]
```

映射更新为：

```text
a -> 0
d -> 1
c -> 2
```

这样就把删除控制在 `O(1)`。

## 解题思路
### 1. 为什么单独用哈希表不够
如果只用哈希表，`insert` 和 `delete` 当然可以很快。

但 `getRandom()` 想要：

- 等概率
- `O(1)`

就麻烦了，因为普通哈希表不支持“按第 k 个元素直接访问”。

所以必须补一个能通过下标随机访问的结构，也就是数组或动态数组。

### 2. 需要维护哪些信息
一种经典设计是维护三份信息：

- `indexKeyMap`：下标 -> 元素
- `keyIndexMap`：元素 -> 下标
- `size`：当前元素个数

等价地，也可以理解成：

- 一个数组
- 一张值到数组下标的哈希表

### 3. `insert(key)` 怎么做
流程：

1. 如果 `key` 已存在，直接忽略
2. 否则把它放到数组末尾
3. 在两张映射里建立双向关系
4. `size++`

因为是尾插，所以时间是 `O(1)`。

### 4. `delete(key)` 怎么做
删除是整题最关键的地方。

流程：

1. 先通过 `keyIndexMap` 找到 `key` 所在下标 `deleteIndex`
2. 取出最后一个元素 `lastKey`
3. 用 `lastKey` 覆盖 `deleteIndex`
4. 更新 `lastKey` 的下标映射
5. 删除最后一个位置
6. 删除 `key` 在映射中的记录
7. `size--`

这样避免了数组中间元素整体搬家。

### 5. `getRandom()` 怎么做
只要当前结构里有 `size` 个元素，就生成一个：

```text
[0, size-1]
```

之间的随机整数，然后返回该下标对应的元素即可。

因为每个下标被选中的概率相同，所以每个元素被返回的概率也相同。

## 复杂度
- `insert`：`O(1)`
- `delete`：`O(1)`
- `getRandom`：`O(1)`
- 额外空间复杂度：`O(N)`

这里的 `O(1)` 指平均 / 期望意义下的常数时间，建立在哈希表查找近似常数的前提上。

## 典型例子
初始空结构：

```text
[]
```

执行：

```text
insert(10)
insert(20)
insert(30)
```

此时：

```text
数组: [10, 20, 30]
映射: 10->0, 20->1, 30->2
```

执行：

```text
delete(20)
```

删除时用末尾 `30` 覆盖：

```text
数组: [10, 30]
映射: 10->0, 30->1
```

此后 `getRandom()` 只会在 `10` 和 `30` 中等概率返回。

## 易错点
- 删除时不要做“数组中间元素整体左移”，那会变成 `O(N)`。
- `getRandom()` 的等概率前提是数组中元素必须连续存放在 `0...size-1`。
- 用最后一个元素补洞后，别忘了更新它在哈希表中的下标。
- 删除最后一个元素本身时，也要正确处理覆盖逻辑，虽然覆盖对象就是自己。
- `insert` 题目要求“不重复加入”，所以要先判断是否已存在。

## 代码 / 伪代码
经典实现框架如下：

```java
class RandomPool<K> {
    HashMap<K, Integer> keyIndexMap;
    HashMap<Integer, K> indexKeyMap;
    int size;

    void insert(K key) {
        if (!keyIndexMap.containsKey(key)) {
            keyIndexMap.put(key, size);
            indexKeyMap.put(size, key);
            size++;
        }
    }

    void delete(K key) {
        if (!keyIndexMap.containsKey(key)) {
            return;
        }
        int deleteIndex = keyIndexMap.get(key);
        int lastIndex = size - 1;
        K lastKey = indexKeyMap.get(lastIndex);

        keyIndexMap.put(lastKey, deleteIndex);
        indexKeyMap.put(deleteIndex, lastKey);

        keyIndexMap.remove(key);
        indexKeyMap.remove(lastIndex);
        size--;
    }

    K getRandom() {
        if (size == 0) {
            return null;
        }
        int randomIndex = (int) (Math.random() * size);
        return indexKeyMap.get(randomIndex);
    }
}
```

## 记忆点
- 想 `O(1)` 随机取元素，必须有连续下标结构。
- 想 `O(1)` 删除指定元素，必须有值到下标的映射。
- 删除核心：最后一个元素补洞。
- 本题本质是“数组 + 哈希表”的组合设计题。
