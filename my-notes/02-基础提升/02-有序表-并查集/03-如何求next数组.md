## 一句话结论
`next` 数组是 KMP 提前为模式串做好的“回退表”。  
它能在线性时间里求出，本质是把前一个位置已经知道的最长相等前后缀结果复用到下一个位置，而不是每次从头枚举。

## 核心知识点
- `next[i]` 表示 `match[0..i-1]` 这段前缀的最长相等前后缀长度。
- 计算 `next[i]` 时，比较的是 `match[i - 1]` 和 `match[cn]`。
- `cn` 表示当前候选前后缀长度，也等价于当前拿来比较的前缀位置。
- 如果 `match[i - 1] == match[cn]`，就能把当前候选长度再扩一位。
- 如果失配，就把 `cn` 跳到 `next[cn]`，继续尝试更短候选。
- 因为 `i` 只会右移，`cn` 只会增长或回退，所以总复杂度是 `O(M)`。

给定模式串 `match`，请构造它的 `next` 数组。  
其中 `next[i]` 表示在子串 `match[0..i-1]` 中，最长的“前缀 = 后缀”的长度，但前缀和后缀都不能取整段本身。  
要求你说明：

- `next` 数组该如何从左到右递推
- 为什么失配时可以跳到 `next[cn]`
- 为什么构造过程是线性时间

```text

```

这里正在求 `next[N]`。  
原始笔记说“`cn` 开始等于 8”，意思是：当前先尝试长度为 `8` 的候选前后缀能不能继续扩一位。

```text

        ^           ^
     先比较这两个字符
```

- 如果 `e == ?`，说明长度 `8` 的候选可以扩成长度 `9`，于是 `next[N] = 9`
- 如果不相等，就不能直接重来，而是要退到更短候选继续试

```text
失败后跳到长度 4
再失败后跳到长度 1
还失败就 next[N] = 0
```

这正是 `cn = next[cn]` 的过程。

## 图解
### 1. `next[i]` 到底看的是哪一段
当我们要求 `next[i]` 时，研究的是 `i` 前面的整段，不包含 `i` 位置本身。

```text
模式串: a b b s t a b b ?
下标  : 0 1 2 3 4 5 6 7 8
                      ^
                      i
```

此时要求的是：

```text
next[8]
```

真正要分析的串是：

```text
a b b s t a b b
```

问题等价于：

```text
这段串里，最长的“前缀 = 后缀”长度是多少？
```

比如：

```text
前缀候选: a, ab, abb, abbs, ...
后缀候选: b, bb, abb, tabb, ...
```

最长相等前后缀是：

```text
abb
```

所以：

```text
next[8] = 3
```

### 2. `cn` 失配时为什么能直接跳
可以把 `cn` 理解成“当前最有希望继续成功的候选长度”。

```text
当前已知：
match[0..cn-1] == match[i-cn .. i-1]
```

如果下一位扩展失败，说明长度 `cn` 不行了。  
但能继续尝试的更短长度，不需要从 `cn-1、cn-2、...` 逐个枚举，因为：

- 新候选必须仍然是“这段前缀自己的前后缀”
- “这段前缀自己的最长相等前后缀”早就记录在 `next[cn]` 里

所以失配时可以直接：

```text
cn = next[cn]
```

图上看就是：

```text
当前候选长度 cn
      ↓ 失配
更短候选长度 next[cn]
      ↓ 再失配
更更短候选长度 next[next[cn]]
```

## 解题思路
### 1. 先定好基础值
经典课程版 KMP 一般约定：

```text
next[0] = -1
next[1] = 0
```

这两个值的作用是：

- `next[0] = -1` 表示第一个字符前面没有可回退的位置
- `next[1] = 0` 表示长度为 `1` 的前缀不可能有非空相等前后缀

### 2. 用 `i` 和 `cn` 从左到右递推
设：

- `i` 是当前要填写的下标
- `cn` 是当前候选前后缀长度

初始：

```text
i = 2
cn = 0
```

然后循环分三种情况：

1. 如果 `match[i - 1] == match[cn]`

说明当前候选可以再扩一位，于是：

```text
next[i] = cn + 1
i++
cn++
```

2. 如果 `match[i - 1] != match[cn]` 且 `cn > 0`

说明当前候选长度不行，但还可以尝试更短候选：

```text
cn = next[cn]
```

3. 如果 `match[i - 1] != match[cn]` 且 `cn == 0`

说明已经没有更短的非空候选了：

```text
next[i] = 0
i++
```

### 3. 为什么是线性时间
虽然里面有“失配回退”的动作，但不是暴力重算：

- `i` 从左到右只走一遍
- `cn` 每次回退都会严格变小
- `cn` 的总增长次数和总回退次数都不会超过串长的常数倍

所以构造 `next` 的总时间复杂度是：

```text
O(M)
```

## 复杂度
- 时间复杂度：`O(M)`
- 空间复杂度：`O(M)`

其中 `M` 是模式串长度，额外空间就是 `next` 数组本身。

## 典型例子
假设模式串是：

```text
abbstabb?
```

现在要算最后这个 `?` 对应位置的 `next`。

如果前面已经知道当前候选长度是 `3`，对应公共前后缀：

```text
abb
```

那么下一步不是重头检查，而是直接比较：

```text
match[i - 1] 和 match[3]
```

- 相等：`next[i] = 4`
- 不等：令 `cn = next[3]`
- 再比较更短候选
- 一直退到 `0` 还不行，就填 `0`

这和原始笔记里记的“失败后往更短前后缀跳”是同一个过程。

## 易错点
- `next[i]` 研究的是 `i` 前面的串，不包含 `i` 本身。
- 失配时跳的是 `next[cn]`，不是 `next[i - 1]`。
- `cn` 既代表长度，也代表下一次要比较的前缀下标。
- “最长相等前后缀”里不能把整段字符串本身当成前缀和后缀。
- 这题是“如何构造 `next`”，不是在讲主串和模式串的正式匹配流程。

## 代码 / 伪代码
```java
int[] getNextArray(char[] match) {
    if (match.length == 1) {
        return new int[] { -1 };
    }
    int[] next = new int[match.length];
    next[0] = -1;
    next[1] = 0;
    int i = 2;
    int cn = 0;
    while (i < next.length) {
        if (match[i - 1] == match[cn]) {
            next[i++] = ++cn;
        } else if (cn > 0) {
            cn = next[cn];
        } else {
            next[i++] = 0;
        }
    }
    return next;
}
```

可以把整段逻辑记成一句话：

```text
能扩就扩，不能扩就按 next 回退，退无可退就填 0
```

## 记忆点
- `next[i]` 看的是 `i` 前面的串。
- `cn` 是当前候选前后缀长度。
- 失配时不用重来，直接跳 `next[cn]`。
- `next` 构造和 KMP 匹配一样，关键都是“复用已经算出的结构信息”。
