## 一句话结论
KMP 解决的是字符串匹配问题：在主串里高效寻找模式串第一次出现的位置。
它比暴力匹配快的核心，不是“比较更少字符”这么简单，而是利用 `next` 数组把模式串自身的重复结构提前算好，从而在失配时一次性跳过大量不可能成功的起点。

## 核心知识点
- KMP 的目标是在 `str` 中查找 `match` 是否出现，以及出现位置。
- `next` 数组描述的是模式串每个位置之前那一段前缀的“最长相等前后缀长度”。
- 失配时，主串指针不用回退，模式串指针按照 `next` 跳转。
- KMP 的本质是“拿模式串自己的结构信息，加速匹配过程”。
- 整体复杂度是 `O(N + M)`，其中 `N` 是主串长度，`M` 是模式串长度。

## 图片转写 / 题意还原
原始笔记里已经给出了几条非常关键的提示：

- `next` 数组
- 某个字符 `A` 前面的字符串 `S`，最长的前缀与后缀匹配长度
- 这个前后缀不包含字符 `A` 本身
- 而且不能取到整体
- KMP 加速的实质是：
  - 模式串可以直接右移到某个更合理的位置继续匹配
  - 主串中一大段起点其实已经被证明不可能成功

把题目补成完整说明，就是：

- 给定两个字符串：
  - 主串 `str`
  - 模式串 `match`
- 需要判断 `match` 是否是 `str` 的子串
- 如果是，返回第一次出现的位置
- 如果不是，返回 `-1`
- 要求匹配过程优于朴素暴力匹配的 `O(N * M)`

这篇笔记真正要回答的是：

```text
KMP 为什么能在线性时间里做字符串匹配
以及 next 数组到底记录了什么
```

## 图解
### 1. `next` 数组到底在看谁
假设模式串是：

```text
a b b s a b b w
0 1 2 3 4 5 6 7
```

当我们讨论位置 `7` 的 `next[7]` 时，关注的不是位置 `7` 这个字符本身，而是它前面的那一段：

```text
a b b s a b b
```

我们要问：

```text
这段字符串里
最长的“前缀 = 后缀”的长度是多少
```

例如前缀：

```text
a
ab
abb
abbs
...
```

后缀：

```text
b
bb
abb
sabb
...
```

其中最长相等的是：

```text
abb
```

所以长度是 `3`，这就是某些位置 `next` 值的来源。

### 2. 为什么失配时可以跳
假设主串和模式串已经匹配到一半：

```text
主串:   a b b s a b b t ...
模式串: a b b s a b b w
```

在 `t` 和 `w` 这里失配。

暴力算法会让模式串整体右移一位，重新从头比。

但 KMP 会发现：

- 前面已经匹配好的部分里，存在一段最长相等前后缀
- 这意味着模式串没必要退回到最开头
- 可以直接跳到那个前后缀对齐的位置继续比

于是：

```text
主串指针不回退
模式串指针跳到 next 指向的位置
```

这就是 KMP 的加速来源。

## 解题思路
### 1. 暴力匹配为什么慢
朴素做法是：

- 主串从每个位置都尝试作为起点
- 模式串从头开始逐字符匹配
- 一旦失配，主串起点后移 1 位，模式串重置到开头

问题在于：

```text
很多比较其实是重复的
```

因为模式串前面那一段已经暴露出自己的结构信息，但暴力法完全没利用。

### 2. `next` 数组到底是什么
对于模式串位置 `i`，`next[i]` 表示：

```text
在 i 之前那一段字符串中
最长相等前缀和后缀的长度
```

并且这个前后缀：

- 不包含位置 `i` 本身
- 不能取整个前缀串本身

这也就是原始笔记里说的：

```text
与字符 A 无关
而且不能取到整体
```

### 3. 为什么默认前两个值是 `-1`、`0`
课程里常见写法中：

- `next[0] = -1`
- `next[1] = 0`

含义是：

- 第一个字符前面没有东西，所以记为 `-1`
- 第二个字符前面只有一个字符，最长相等前后缀长度只能是 `0`

这两个初值让匹配过程写起来最顺。

### 4. KMP 为什么能跳过一大段起点
这是原始笔记里最重要的两句。

失配时，KMP 利用了一个事实：

- 模式串当前已经匹配成功的部分中，有一段最长相等前后缀
- 那么模式串整体右移后，可以直接让这两段对齐

同时还能推出：

```text
主串中间被跳过的那些位置
不可能成为有效起点
```

否则就会和 `next` 数组的定义矛盾。

所以 KMP 跳过的不是“可能成功的位置”，而是“已经证明不可能成功的位置”。

### 5. 匹配流程怎么写
设：

- `x` 指向主串
- `y` 指向模式串

循环过程中：

1. 如果 `str[x] == match[y]`，二者同时右移
2. 如果失配且 `next[y] == -1`，说明模式串已经没法再退，主串右移
3. 否则让 `y = next[y]`

这样：

- 主串指针不会反复回退
- 模式串也不会傻傻回到 0

## 复杂度
- 构建 `next` 数组：`O(M)`
- 匹配过程：`O(N)`
- 总时间复杂度：`O(N + M)`
- 空间复杂度：`O(M)`

其中：

- `N` 是主串长度
- `M` 是模式串长度

## 典型例子
主串：

```text
abbsabbtcabbsabbe
```

模式串：

```text
abbsabbw
```

前面已经匹配到：

```text
abbsabb
```

此时主串下一个字符是：

```text
t
```

模式串下一个字符是：

```text
w
```

失配。

如果是暴力法，会让模式串回到开头。
如果是 KMP，会根据 `next` 直接跳到“最长前后缀对齐”的位置继续比较。

这就是为什么原始笔记会说：

- 模式串直接从某个更后的位置开始继续匹配
- 主串一大段位置都被证明不可能成功

## 易错点
- `next[i]` 看的是 `i` 前面的那段，不包含 `i` 本身。
- “最长相等前后缀”不能取整个字符串本身。
- KMP 不是主串和模式串都跳，真正不回退的是主串指针。
- `next[0] = -1, next[1] = 0` 是常见实现约定，不同教材可能写法略有差异，但本质相同。
- 理解 KMP 最重要的不是背代码，而是理解“为什么这些位置可以跳过”。

## 代码 / 伪代码
匹配核心框架：

```java
int getIndexOf(String s, String m) {
    if (s == null || m == null || m.length() < 1 || s.length() < m.length()) {
        return -1;
    }
    char[] str = s.toCharArray();
    char[] match = m.toCharArray();
    int x = 0;
    int y = 0;
    int[] next = getNextArray(match);
    while (x < str.length && y < match.length) {
        if (str[x] == match[y]) {
            x++;
            y++;
        } else if (next[y] == -1) {
            x++;
        } else {
            y = next[y];
        }
    }
    return y == match.length ? x - y : -1;
}
```

`next` 数组的具体构造会在下一题单独展开。

## 记忆点
- KMP 的关键不是“少比较”，而是“利用模式串自己的结构信息跳过无效起点”。
- `next[i]` = `i` 前面那段字符串的最长相等前后缀长度。
- 失配时主串不回退，模式串按 `next` 跳。
- 复杂度从暴力的 `O(N * M)` 降到 `O(N + M)`。
