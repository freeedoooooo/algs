本章围绕哈希函数的核心性质展开，从基础原理出发，逐步延伸到哈希表实现、大数据分流、布隆过滤器、一致性哈希等工程场景，最后以岛问题收尾，引出并行化思路。

核心主线是：

```text
哈希函数的均匀性、离散性
-> 如何用它分桶、分流
-> 如何在此基础上设计高效数据结构
-> 如何在分布式场景下扩展应用
```

## 本章结论
- 哈希函数最底层的要求，是输出尽量均匀、输入变化后结果能被充分打散。
- 哈希表、RandomPool、布隆过滤器、一致性哈希，本质上都是“如何使用哈希结果”这一层的不同工程落地。
- 大数据题里，哈希函数最重要的价值之一不是查表，而是分流：相同 key 稳定进入同一桶。
- 分布式场景下，一致性哈希解决的是“节点变化时尽量少迁移”，虚拟节点解决的是“尽量负载均衡”。
- 岛问题虽然表面是矩阵题，但进阶版把这一章和并查集、并行计算串了起来。

## 图片转写 / 章节笔记还原
原始章节页虽然没有单独配图，但从条目排列顺序里能还原出一条很清晰的学习路线：

### 1. 先讲“哈希函数本身要满足什么性质”
第一题先把最底层的两个关键词讲清楚：

- 均匀性
- 离散性

这一步的作用是打地基。因为如果不理解“为什么哈希结果必须尽量均匀地分散”，后面哈希表、布隆过滤器和一致性哈希都会只剩结论，难以真正理解设计原因。

### 2. 再讲“哈希函数怎么用来切大问题”
第二题开始进入大数据场景：

- 数据量太大，直接统计装不下
- 于是先按哈希值分桶
- 再逐桶做局部统计

这一步把哈希函数从“算个下标”提升成了“做稳定分流”的工具。

### 3. 接着落到数据结构实现
第三题和第四题其实是一组：

- 题目1：哈希表本身怎么实现
- 题目2：如何借助哈希表设计更复杂的数据结构

这里重点是体会：

```text
哈希表不是终点
而是很多 O(1) 结构设计的基础零件
```

### 4. 再进入更强的工程结构
第五题和第六题对应两个很经典的系统设计结构：

- 布隆过滤器：极致省空间的存在性判断
- 一致性哈希：分布式系统里的分片与扩缩容

这说明本章后半段已经不再只是“算法题”，而是在往工程系统设计过渡。

### 5. 最后用岛问题把哈希、并查集、并行计算连起来
第七题表面是矩阵连通块计数，但它的进阶版提醒的是：

- 当数据规模大到单机不够时
- 可以分块并行
- 最后再通过边界归并得到全局答案

这让本章和下一章“并查集”的联系自然接上了。

## 记忆点
- 哈希函数本身讲的是“怎么打散”，后续结构讲的是“怎么使用这种打散结果”。
- 哈希表解决快速定位，布隆过滤器解决省空间判存在，一致性哈希解决分布式迁移成本。
- 大数据题里优先想：能不能借助哈希函数把数据稳定切开。
- 这一章的后半段已经明显从算法实现过渡到了工程系统设计。
