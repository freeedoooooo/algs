## 一句话结论
打表找规律，不是“靠猜答案”，而是先用暴力方法把小规模输入全部跑出来，再从结果表里观察周期、奇偶性、取模规律或临界点。
这类题通常参数类型简单、实际参数很少、返回值也简单，所以非常适合先穷举小样本，再把暴力结论优化成公式或 `O(1)` 判定。

## 核心知识点
- 适合打表的题，往往只有 1 个实际参数，返回值也只有 1 个。
- 第一步不是直接找公式，而是先写出正确但可能很慢的暴力解。
- 打表重点看：奇偶性、模运算、周期性、从某个临界值开始的稳定规律。
- 一旦规律足够稳定，就把暴力求值改写成常数时间结论。
- 打表法本质上是“小范围枚举 + 数学归纳 + 规律压缩”。

1. 某个面试题，输入参数类型简单，并且只有一个实际参数
2. 要求的返回值类型也简单，并且只有一个

### 图 2：题目一，买苹果最少袋数
商店只有两种袋子：

- 每袋装 `6` 个苹果
- 每袋装 `8` 个苹果

每种袋子数量无限，小虎要求：

- 使用的袋子数尽量少
- 每个袋子必须装满

给定正整数 `N`，返回至少需要多少袋；如果无法恰好装完，返回 `-1`。

### 图 3：题目二，牛羊吃草博弈
给定正整数 `N`，表示仓库里有 `N` 份草。

- 牛先吃，羊后吃
- 双方轮流吃
- 每一轮只能吃 `1, 4, 16, 64...`，也就是 `4` 的某次方
- 谁先把草吃完谁获胜
- 双方都绝顶聪明，都会做出最优决策

根据唯一参数 `N`，返回最后谁会赢。

### 图 4：题目三，是否能表示成连续正数和
定义一种数：

- 可以表示成若干个数量大于 `1` 的连续正整数之和

例如：

- `5 = 2 + 3`，所以 `5` 是这种数
- `12 = 3 + 4 + 5`，所以 `12` 是这种数
- `1` 不是，因为项数必须大于 `1`
- `2 = 1 + 1` 也不行，因为右边不是连续正整数

给定参数 `N`，判断它是否可以表示成若干连续正整数的和。

## 图解
这类题的通用路线是：

```text
先写暴力
-> 打印小样本答案
-> 观察表格中的稳定模式
-> 总结成公式 / 取模规律 / 位运算规律
```

### 例 1：苹果装袋

```text
N   结果
1   -1
6    1
8    1
10  -1
12   2
14   2
16   2
18   3
20   3
22   3
24   3
26   4
28   4
30   4
```

从表里很容易看出：

- 奇数一定不行
- 小于 `18` 的偶数里，只有 `6、8、12、14、16` 可以
- 从 `18` 开始，答案出现稳定规律：每增加 `8` 个苹果，最少袋数加 `1`

### 例 2：牛羊吃草

```text
N   赢家
0   羊
1   牛
2   羊
3   牛
4   牛
5   羊
6   牛
7   羊
8   牛
9   牛
10  羊
```

规律会稳定成：

```text
N % 5 == 0 或 2 -> 后手赢 -> 羊赢
其他情况        -> 先手赢 -> 牛赢
```

### 例 3：连续正数和

```text
N   是否可以表示
1   否
2   否
3   是   1+2
4   否
5   是   2+3
6   是   1+2+3
7   是   3+4
8   否
9   是   4+5
10  是   1+2+3+4
```

继续打表后会发现：

```text
不能表示的数：1, 2, 4, 8, 16...
也就是 2 的幂
```

所以结论是：

```text
N 只要不是 2 的幂，就一定能表示成若干连续正整数和
```

## 解题思路
### 1. 什么时候该想到打表
如果题目满足下面特征，就要优先怀疑它能不能“先暴力、再归纳”：

- 参数很少
- 参数规模可以从小到大枚举
- 返回值不是很复杂
- 暴力版虽然慢，但写得出来且保证正确

### 2. 先写一个绝对正确的暴力版本
打表最怕的不是“没发现规律”，而是暴力本身写错了。

所以第一步一定是：

- 不考虑复杂度
- 只保证小范围输入下答案绝对正确
- 把 `0 ~ 50`、`0 ~ 100` 或者更大的小样本全打出来

### 3. 看表时重点看什么
通常优先观察：

- 奇偶性
- 对某个数取模后的分类
- 从某个临界点开始是否出现固定递推
- 是否出现指数级特殊点，比如 `1, 2, 4, 8, 16`

这三道例题分别对应：

- 苹果装袋：奇偶 + 临界点 + 线性递增
- 牛羊吃草：小表推出模 `5` 周期
- 连续正数和：特殊点是 `2` 的幂

### 4. 再把暴力规律压成快速解
最终目标不是保留表，而是把规律写成更直接的判断。

常见压缩形式有：

- `if-else` 小范围分类
- `N % k`
- 位运算
- 从某个阈值开始的通项公式

## 复杂度
- 暴力打表阶段：依赖暴力解本身，通常会比最终解更慢
- 规律版求解：常见是 `O(1)`
- 额外空间复杂度：常见是 `O(1)`

## 典型例子
### 例 1：苹果装袋
当 `N = 26` 时：

- 试 `8` 袋优先，剩余 `18`
- `18 = 6 + 6 + 6`
- 所以最少是 `8 + 6 + 6 + 6`，共 `4` 袋

而不是：

- `6 + 6 + 6 + 6 + 2`，不合法

### 例 2：牛羊吃草
当 `N = 7` 时，牛可以吃：

- 吃 `1`，剩 `6`
- 吃 `4`，剩 `3`

无论哪种，都会把局面交给对手的必胜态，因此 `7` 是先手必败态，最后羊赢。

### 例 3：连续正数和
当 `N = 8` 时：

- `8 = 8`
- `8 = 3 + 5` 不是连续
- `8 = 1 + 2 + 5` 也不是连续

最终发现它不能写成项数大于 `1` 的连续正数和；而 `8` 正好是 `2` 的幂。

## 易错点
- 打表不是“拍脑袋猜规律”，必须先有正确暴力解。
- 小样本表要打得足够长，否则很容易把假规律当真规律。
- 观察到规律后，最好再用更多样本回验一次。
- 苹果装袋题里，奇数直接不可能，但偶数也不一定都可以。
- 连续正数和这题，判断标准不是“是不是奇数”，而是“是不是 2 的幂”。 

## 代码 / 伪代码
仓库里目前没有这一题对应的现成实现文件，这里保留课程标准伪代码。

### 1. 苹果装袋

```java
int minBags(int n) {
    if ((n & 1) != 0) {
        return -1;
    }
    if (n < 18) {
        if (n == 0) return 0;
        if (n == 6 || n == 8) return 1;
        if (n == 12 || n == 14 || n == 16) return 2;
        return -1;
    }
    return (n - 18) / 8 + 3;
}
```

### 2. 牛羊吃草

```java
String winner(int n) {
    return (n % 5 == 0 || n % 5 == 2) ? "羊" : "牛";
}
```

### 3. 连续正数和

```java
boolean canSplit(int n) {
    return (n & (n - 1)) != 0;
}
```

其中：

```text
n 是 2 的幂 <=> (n & (n - 1)) == 0
```

## 记忆点
- 打表的核心是“先暴力打印，再观察规律”。
- 规律最常落在：奇偶、取模、临界点、幂次点。
- 苹果装袋题的结论是“奇数不行，18 之后按 8 递推”。
- 牛羊吃草题的结论是“看 `N % 5`”。
- 连续正数和题的结论是“不是 2 的幂就行”。
