## 章节导读
这一章是一条很标准的图论入门主线，学习顺序非常自然：

```text
先会表示图
-> 再会遍历图
-> 再做图上的约束排序、最优连通和最短路径
```

这是一整条图论入门主线。

## 本章结论
- 图题第一步不是直接写算法，而是先把题目输入转换成你熟悉的图结构。
- 图通常由点集和边集构成；有向图和无向图都可以转成统一结构表达。
- BFS 用队列，适合逐层扩展；DFS 用栈或递归，适合沿路走到底再回退。
- 拓扑排序只适用于有向无环图。
- Kruskal 用并查集，Prim 用“已解锁节点 + 最小边”思想。

## 图片转写 / 章节笔记还原
原始笔记里这一章主要包含下面几层意思：

### 1. 图怎么表示
- 图由点的集合和边的集合构成
- 实际上编码时常统一用有向图表达
- 边可能带权值
- 先把题目里的图转成自己熟悉的 `Node / Edge / Graph` 结构，再套模板

### 2. 图算法怎么准备
1. 先用自己最熟悉的方式实现图结构
2. 在这套结构上实现 BFS、DFS、拓扑、最小生成树、最短路等模板
3. 面试里把题目输入转成这套结构，再调用或改写模板
