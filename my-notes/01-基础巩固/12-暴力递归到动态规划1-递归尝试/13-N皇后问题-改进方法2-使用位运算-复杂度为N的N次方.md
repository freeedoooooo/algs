## 一句话结论
`N` 皇后问题是典型的搜索与剪枝题。普通做法按行尝试列位置，位运算优化则把列限制、左斜线限制、右斜线限制压成三个整数位掩码。

## 核心知识点
- 每行必须放一个皇后。
- 决策顺序通常按“第几行”展开。
- 不能放在已有列、左斜线、右斜线冲突位置。
- 位运算能快速算出当前可放位置集合。

## 图片转写 / 题意还原
原图内容为：

- 在 `N * N` 棋盘上摆 `N` 个皇后
- 任意两个皇后不能同行、同列、也不能在同一条斜线上
- 给定整数 `n`，返回摆法数量
- `n=1` 返回 `1`
- `n=2` 或 `3` 返回 `0`
- `n=8` 返回 `92`

## 解题思路
### 1. 普通递归
按行放皇后：

- 第 `row` 行尝试每一列
- 检查和前面行是否冲突
- 合法则递归下一行

### 2. 位运算优化
维护三个限制状态：

- `colLim`：哪些列已经占了
- `leftDiaLim`：左斜线限制
- `rightDiaLim`：右斜线限制

当前能放的位置：

```text
pos = limit & (~(colLim | leftDiaLim | rightDiaLim))
```

每次取出最右侧 `1` 作为一个尝试分支。

## 复杂度
- 普通递归较慢
- 位运算优化在工程上会快很多，常作为 `N <= 32` 的高效写法

## 代码 / 伪代码
```java
int process(int limit, int colLim, int leftDiaLim, int rightDiaLim) {
    if (colLim == limit) return 1;
    int pos = limit & (~(colLim | leftDiaLim | rightDiaLim));
    int ans = 0;
    while (pos != 0) {
        int mostRightOne = pos & (-pos);
        pos -= mostRightOne;
        ans += process(limit,
                       colLim | mostRightOne,
                       (leftDiaLim | mostRightOne) << 1,
                       (rightDiaLim | mostRightOne) >> 1);
    }
    return ans;
}
```

## 记忆点
- `N` 皇后是按行搜索。
- 位运算版用三个位掩码表示限制。
- 每次取最右侧 `1` 做一个分支。
