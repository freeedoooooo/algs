## 一句话结论
当字符串包含**重复字符**时，直接枚举会产生重复子序列。本题在上一题基础上增加**去重要求**，最简单的方法是用 `HashSet` 收集结果，利用集合的天然去重特性。

## 理论 / 应用价值

### 在知识体系中的位置

```
基础子序列（无重复）
  ↓ 理解二叉决策树
去重子序列（有重复字符）
  ↓ 学习结果去重技巧
高级去重（剪枝优化）
  ↓ 在搜索过程中去重
回溯算法中的去重策略
```

### 为什么值得学

1. **理解"结果去重" vs "过程去重"**
   - 本题采用结果去重（简单但效率低）
   - 后续可学习过程去重（剪枝，效率高）
   - 这是两种不同的去重策略

2. **掌握 Set 的应用**
   - `HashSet` 天然去重
   - 理解何时用 List、何时用 Set

3. **为回溯去重打基础**
   - 全排列去重、组合去重都用类似思路
   - 本题是最简单的去重案例

### 解决的痛点

- **实际场景常见**：真实数据往往有重复
- **面试考察点**：检验是否考虑边界情况
- **性能优化意识**：从暴力去重到剪枝去重的思维跃迁

### 实际应用场景

- 数据库查询结果去重
- 文本处理中的唯一子串提取
- 基因序列分析中的独特模式识别

## 核心知识点
- **递归结构不变**：与无重复版完全相同
- **去重策略**：用 `HashSet` 替代 `ArrayList`
- **时间复杂度不变**：仍然遍历所有 2^N 条路径
- **空间复杂度增加**：需要存储所有唯一子序列

## 题意还原

**要求**：
- **输入**：一个可能包含重复字符的字符串，如 `"acc"`
- **输出**：打印它的全部**不重复**子序列
- **规则**：
  - 子序列保持字符的相对顺序
  - 空串 `""` 也算一个合法子序列
  - **关键**：输出结果中不能有重复的子序列

**示例**：
```
输入: "acc"

暴力展开会有 2^3 = 8 条路径:
  "", "c", "c", "cc", "a", "ac", "ac", "acc"
  
去重后得到 6 个唯一子序列:
  "", "a", "c", "ac", "cc", "acc"
```

**与上一题的区别**：
- 上一题：假设字符串无重复字符，直接收集即可
- 本题：字符串可能有重复字符，需要去重

## 解题思路
- 仍然做“要 / 不要”
- 到叶子节点时，把答案放进 `Set`
- 最后输出 `Set` 中内容

## 复杂度
- 时间复杂度：`O(2^N)`
- 空间复杂度：`O(2^N)`

## 代码 / 伪代码
```java
void process(char[] str, int index, String path, Set<String> ans) {
    if (index == str.length) {
        ans.add(path);
        return;
    }
    process(str, index + 1, path, ans);
    process(str, index + 1, path + str[index], ans);
}
```

## 记忆点
- 递归树不变。
- 重复发生在结果层。
- 最直接就是 `HashSet` 去重。
