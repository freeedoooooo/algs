## 一句话结论
当字符串包含**重复字符**时，直接枚举会产生重复子序列。本题在上一题基础上增加**去重要求**，最简单的方法是用 `HashSet` 收集结果，利用集合的天然去重特性。

## 核心知识点
- **递归结构不变**：与无重复版完全相同
- **去重策略**：用 `HashSet` 替代 `ArrayList`
- **时间复杂度不变**：仍然遍历所有 2^N 条路径
- **空间复杂度增加**：需要存储所有唯一子序列

## 题意还原

**要求**：
- **输入**：一个可能包含重复字符的字符串，如 `"acc"`
- **输出**：打印它的全部**不重复**子序列
- **规则**：
  - 子序列保持字符的相对顺序
  - 空串 `""` 也算一个合法子序列
  - **关键**：输出结果中不能有重复的子序列

**示例**：
```
输入: "acc"

暴力展开会有 2^3 = 8 条路径:
  "", "c", "c", "cc", "a", "ac", "ac", "acc"
  
去重后得到 6 个唯一子序列:
  "", "a", "c", "ac", "cc", "acc"
```

**与上一题的区别**：
- 上一题：假设字符串无重复字符，直接收集即可
- 本题：字符串可能有重复字符，需要去重

## 图解

### 递归树与去重过程（以 `"acc"` 为例）

```
process("acc", 0, "")
├─ 不要a: process("acc", 1, "")
│   ├─ 不要c1: process("acc", 2, "")
│   │   ├─ 不要c2: → ✅ ""       (加入Set)
│   │   └─ 要c2: → ✅ "c"        (加入Set)
│   └─ 要c1: process("acc", 2, "c")
│       ├─ 不要c2: → ✅ "c"      (重复！Set自动忽略)
│       └─ 要c2: → ✅ "cc"       (加入Set)
└─ 要a: process("acc", 1, "a")
    ├─ 不要c1: process("acc", 2, "a")
    │   ├─ 不要c2: → ✅ "a"      (加入Set)
    │   └─ 要c2: → ✅ "ac"       (加入Set)
    └─ 要c1: process("acc", 2, "ac")
        ├─ 不要c2: → ✅ "ac"     (重复！Set自动忽略)
        └─ 要c2: → ✅ "acc"      (加入Set)

最终 Set 中的结果:
["", "c", "cc", "a", "ac", "acc"]  共6个
```

**关键观察**：
- 🔴 **红色标记**：重复的子序列（被 Set 自动过滤）
- ✅ **绿色标记**：唯一的子序列（被保留）
- 📊 **路径数**：仍然有 8 条路径
- 📊 **结果数**：去重后只有 6 个唯一值

## 解题思路

### 核心思想：结果去重

递归过程与无重复版**完全相同**，唯一区别是收集结果的数据结构：
- 无重复版：用 `List<String>` 收集
- 去重版：用 `Set<String>` 收集

`HashSet` 的 `add()` 方法会自动判断是否已存在，如果已存在则忽略。

### 代码实现

```java
void process(char[] str, int index, String path, Set<String> ans) {
    // Base Case: 走到末尾，加入Set（自动去重）
    if (index == str.length) {
        ans.add(path);  // Set自动处理重复
        return;
    }
    
    // 选择1: 不要当前字符
    process(str, index + 1, path, ans);
    
    // 选择2: 要当前字符
    process(str, index + 1, path + str[index], ans);
}

// 调用方式
Set<String> ans = new HashSet<>();
process(str.toCharArray(), 0, "", ans);

// 输出结果
for (String s : ans) {
    System.out.println(s);
}
```

### 两种去重策略对比

| 策略 | 实现方式 | 优点 | 缺点 |
|------|---------|------|------|
| **结果去重**（本题） | 用 Set 收集 | 简单直观，代码改动小 | 仍然遍历所有路径，效率低 |
| **过程去重**（进阶） | 递归时剪枝 | 减少无效搜索，效率高 | 实现复杂，需排序+判断 |

**本题采用结果去重**，因为：
1. 代码最简单，只需改一个数据类型
2. 易于理解，适合入门
3. 为后续学习过程去重做铺垫

## 复杂度

- **时间复杂度**：`O(N * 2^N)`
  - 仍然遍历所有 2^N 条路径
  - 每个叶子节点插入 Set 需要 O(N)（字符串哈希计算）
  - 总计：O(N * 2^N)
  
- **空间复杂度**：`O(2^N)`
  - 最坏情况下（无重复字符），Set 存储 2^N 个子序列
  - 每个子序列平均长度 O(N)
  - 递归栈深度 O(N)

## 典型例子

以 `"aab"` 为例：

```
暴力展开 2^3 = 8 条路径:
  "", "b", "a", "ab", "a", "ab", "aa", "aab"
  
去重后得到 6 个唯一子序列:
  "", "a", "b", "aa", "ab", "aab"
```

**注意**：`"a"` 和 `"ab"` 各出现两次，被 Set 自动去重。

## 易错点

- ❌ **忘记去重**：直接用 List 收集，导致结果有重复
- ❌ **误以为递归会减少**：即使去重，仍然要遍历所有 2^N 条路径
- ❌ **混淆两种去重策略**：本题是结果去重，不是过程去重
- ❌ **Set 的迭代顺序不确定**：HashSet 不保证输出顺序

## 扩展思考

### 如何优化？（过程去重）

如果想提高效率，可以在递归过程中剪枝：

```java
// 先排序，让相同字符相邻
Arrays.sort(str);

void process(char[] str, int index, String path, Set<String> ans) {
    if (index == str.length) {
        ans.add(path);
        return;
    }
    
    // 不要当前字符
    process(str, index + 1, path, ans);
    
    // 要当前字符：如果与前一个相同且前一个没选，则跳过
    if (index == 0 || str[index] != str[index-1]) {
        process(str, index + 1, path + str[index], ans);
    }
}
```

这样可以**减少无效搜索**，但实现更复杂。
