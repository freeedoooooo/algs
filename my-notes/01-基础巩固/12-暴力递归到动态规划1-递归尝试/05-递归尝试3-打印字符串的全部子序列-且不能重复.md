## 一句话结论
如果原字符串里有重复字符，直接打印全部子序列会产生重复答案。这题的核心不是改变递归树，而是在收集答案时做去重。

## 核心知识点
- 递归过程和“全部子序列”一模一样。
- 去重最直接的方法是 `HashSet<String>`。
- 也可以理解为：路径树不去重，结果集合去重。

## 图片转写 / 题意还原
原始笔记只保留了题目名称，本节按课程标准内容补全。

完整题意是：

- 给定一个可能包含重复字符的字符串 `str`
- 仍然要求打印它的全部子序列
- 但输出结果中不能出现重复答案

例如 `str = "acc"` 时：

- 按“要 / 不要”暴力展开，路径数仍然有 `2^3 = 8` 条
- 但其中 `"c"` 会出现两次，`"ac"` 也会出现两次
- 最终输出时，这些重复子序列只能保留一份

所以这题和上一题的区别不在“怎么搜”，而在“结果如何去重”。

## 解题思路
- 仍然做“要 / 不要”
- 到叶子节点时，把答案放进 `Set`
- 最后输出 `Set` 中内容

## 复杂度
- 时间复杂度：`O(2^N)`
- 空间复杂度：`O(2^N)`

## 代码 / 伪代码
```java
void process(char[] str, int index, String path, Set<String> ans) {
    if (index == str.length) {
        ans.add(path);
        return;
    }
    process(str, index + 1, path, ans);
    process(str, index + 1, path + str[index], ans);
}
```

## 记忆点
- 递归树不变。
- 重复发生在结果层。
- 最直接就是 `HashSet` 去重。
