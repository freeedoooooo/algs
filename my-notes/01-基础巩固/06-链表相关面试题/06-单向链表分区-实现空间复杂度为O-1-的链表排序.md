## 一句话结论
这题本质上是把数组里的 `partition` 思想搬到链表上：按照某个基准值，把链表重新分成“小于区、等于区、大于区”或“小于区、大于等于区”。
笔试时可以直接借助数组做分区，面试更推荐手动拆成多段链表再拼接，这样能把额外空间压到 `O(1)`。

## 核心知识点
- 链表没有随机访问能力，所以不能像数组那样直接交换任意下标。
- 如果想保留稳定性，最自然的做法是分别维护几个子链表的头尾指针。
- 这题和快排里的荷兰国旗问题很像，只不过“交换元素”变成了“摘节点、接链表”。
- 面试里通常更看重稳定、低空间、边界清晰的链表拼接写法。

## 图片转写 / 题意还原
原图内容可以整理为：

- 常见面试题
- 将单向链表按某个值划分成左边小、中间相等、右边大的形式
- 做法 1：把链表放入数组里，在数组上做 `partition`，适合笔试
- 做法 2：分成小、中、大三部分，再把各个部分串起来，适合面试

也就是说，这题想讲的不是单一代码，而是两种解法思路：

- 数组版：实现快，空间大
- 链表版：更贴合链表结构，空间更优

## 解题思路
### 1. 笔试版：借助数组
做法是：

1. 先把链表节点全部收集进数组
2. 对数组做一次类似荷兰国旗问题的 `partition`
3. 再按数组顺序把节点重新串回链表

优点：

- 写起来直观
- 很容易复用数组 `partition` 模板

缺点：

- 需要 `O(N)` 额外空间
- 不够“链表味”

### 2. 面试版：拆成多个子链表
更推荐的写法是边遍历边分类：

- 小于基准值的节点接到 `small` 链表
- 等于基准值的节点接到 `equal` 链表
- 大于基准值的节点接到 `big` 链表

或者像仓库这份 LeetCode 实现那样，题目只要求：

- 小于 `x`
- 大于等于 `x`

那就只需要两个链表：

- `left`
- `right`

最后再把这些子链表按顺序拼起来。

### 3. 为什么这版空间是 O(1)
因为整个过程中没有开新数组，也没有新建节点。

我们只是：

- 记录几个头指针、尾指针
- 把原有节点从旧链表摘下来
- 接到新的分区尾部

所以额外空间就是常数级。

### 4. 为什么要断开原链表连接
每处理一个节点后，最好立刻：

```java
head.next = null;
```

否则节点还挂着原链表后继，后面再拼接时很容易形成脏链或环。

## 典型例子
链表：

```text
7 -> 9 -> 1 -> 8 -> 5 -> 2 -> 5
```

假设基准值是：

```text
5
```

如果按“小于 / 等于 / 大于”三段来分，可以得到：

- 小于区：`1 -> 2`
- 等于区：`5 -> 5`
- 大于区：`7 -> 9 -> 8`

最后拼起来：

```text
1 -> 2 -> 5 -> 5 -> 7 -> 9 -> 8
```

如果题目像 LeetCode 86 那样只要求“小于 x 的在前，大于等于 x 的在后”，那就可以整理成：

- 左区：`1 -> 2`
- 右区：`7 -> 9 -> 8 -> 5 -> 5`

并且仍然要保留各自分区中的原始相对次序。

## 复杂度
- 时间复杂度：`O(N)`
- 空间复杂度：
  - 数组版：`O(N)`
  - 多链表拼接版：`O(1)`

## 易错点
- 如果题目要求“保留原始相对顺序”，就必须用稳定写法，不能随便换节点顺序。
- 每处理完一个节点后不及时断开 `next`，非常容易形成环。
- 拼接多个子链表时，要考虑某一段可能为空。
- 这题叫“链表分区”，本质不是排序到完全有序，而是按规则重组。

## 代码 / 伪代码
仓库中的对应实现：

- [PartitionList.java](../../../solutions-java/leetcode/p086_partition_list/PartitionList.java)

这份实现对应的是“两段版”：

- 小于 `x`
- 大于等于 `x`

核心写法是维护两个链表的头尾：

```java
ListNode leftHead = null, leftTail = null;
ListNode rightHead = null, rightTail = null;
```

遍历原链表时：

```java
if (head.val < x) {
    if (leftHead == null) {
        leftHead = head;
    } else {
        leftTail.next = head;
    }
    leftTail = head;
} else {
    if (rightHead == null) {
        rightHead = head;
    } else {
        rightTail.next = head;
    }
    rightTail = head;
}
ListNode next = head.next;
head.next = null;
head = next;
```

最后再拼接：

```java
if (leftHead == null) {
    return rightHead;
}
leftTail.next = rightHead;
return leftHead;
```

如果是课程原题中的三段版，只要把 `left/right` 扩展成：

- `small`
- `equal`
- `big`

再按顺序连接即可。

## 记忆点
- 数组版适合笔试，链表拼接版适合面试。
- 这题是“链表版 partition”，不是完整排序。
- 想保稳定性，就维护多个子链表尾指针。
- `head.next = null` 是防止脏链和成环的关键细节。
