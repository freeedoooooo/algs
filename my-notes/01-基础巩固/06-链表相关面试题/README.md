## 章节导读
这一章不是在讲“链表的基础定义”，而是在讲链表面试里最常出现的几类高频套路：

- 快慢指针
- 栈与链表反转
- 分段重连
- 环检测与相交分类讨论
- 特殊指针结构的复制

如果把 10 道题按知识主线归类，可以这样看：

```text
链表高频套路
├─ 快慢指针
│  ├─ 找中点
│  ├─ 回文判断
│  └─ 判断有环 / 找入环点
├─ 栈与反转
│  ├─ 整链表入栈
│  ├─ 半链表入栈
│  └─ 右半部分反转
├─ 链表重组
│  ├─ 按值分区
│  └─ 删除指定节点
└─ 结构关系题
   ├─ Random 链表复制
   └─ 两链表相交
```

## 本章结论
- 链表题的难点通常不在“循环怎么写”，而在“指针关系什么时候断、什么时候接、有没有恢复”。
- 快慢指针是本章的总枢纽技巧，后面的回文链表、找环入口、相交链表都会反复复用。
- 链表不能随机访问，所以很多数组上的思路都要改写成“走一遍 + 存现场 + 重新接链”。
- 面试里除了做对，还要主动说清楚空间复杂度、是否会破坏原链表、边界节点怎么处理。

### 1. 快慢指针是链表题第一套路
- 链表没有下标，不能像数组那样直接取中点。
- 所以一旦题目出现“中点、上中点、下中点、倒数、环、相交前分段”等关键词，就应该先想到快慢指针。
- 快慢指针的本质，是通过不同速度制造相对位移，从而在一次遍历中拿到结构信息。

这一组题对应：

- [快慢指针](01-链表-介绍快慢指针.md)

- [两个单向链表第一个相交节点](09-两个单向链表第一个相交节点.md)

### 2. 回文链表是一组“空间优化题”
这一组题其实是同一个题目的三种版本：

- 版本 1：整条链表入栈，最好写，空间 `O(N)`
- 版本 2：半条链表入栈，利用快慢指针降到 `O(N/2)`
- 版本 3：右半部分原地反转，空间压到 `O(1)`

这组题最值得记住的，不只是答案本身，而是优化路径：

```text
直接做
-> 用结构换空间
-> 再用原地操作继续压空间
```

对应条目：

- [版本1【笔试】，利用栈，额外空间N](03-版本1-笔试--利用栈-额外空间O(N).md)
- [版本2【面试】，利用栈，额外空间N/2](04-版本2-面试--利用栈-额外空间O(N减半).md)
- [版本3【面试】，链表右半部分反转](05-版本3-面试--额外空间O(1).md)

### 3. 链表题经常考“拆开再拼回去”
很多链表题本质并不是算答案，而是把节点重新组织：

- 按某个值分区，拆成多段再拼接
- 复制 Random 链表，构造新旧节点映射
- 做完判断后恢复原链表

这些题里最容易出错的地方通常是：

- 忘记先断开旧的 `next`
- 拼接时尾指针处理不干净
- 恢复链表时漏节点或形成环

对应条目：

- [单向链表分区，实现空间复杂度为O(1)的链表排序](06-单向链表分区-空间复杂度O(1).md)
- [克隆Random链表](07-克隆Random链表.md)

### 4. 环和相交题要先做结构分类
很多人会直接上手写相交链表，但完整题目其实要先分类：

1. 两条都无环
2. 两条都有环
3. 一条有环、一条无环

先分类，再决定做法，才不会把题做残。

这一章里很重要的一点是：

- “是否有环”本身就是前置判断
- “第一个相交节点”不是只有无环版本
- 不同结构的链表，处理逻辑完全不同

对应条目：

- [单向链表有环](08-单向链表有环.md)
- [两个单向链表第一个相交节点](09-两个单向链表第一个相交节点.md)

### 5. 有些题是在考你是否真正理解链表模型
比如这题：

- [不使用头节点，删除链表任意一个节点](10-不使用头节点-删除链表任意一个节点.md)

它不是推荐写法题，而是在考：

- 单链表删除节点为什么通常需要前驱
- 只给当前节点时，为什么只能“伪删除”
- 这种写法为什么删不了尾节点

这类题更接近“数据结构本质理解题”。

## 记忆点
- 链表题最常见三件套：快慢指针、反转、分段重连。
- 一看到“中点 / 环 / 回文 / 相交”，优先想快慢指针。
- 一看到“按条件重排节点”，优先想拆成几段链表再拼接。
- 一旦改动了原链表，记得想清楚是否需要恢复。
