## 一句话结论
版本 1 是判断回文链表最直接、最好写的做法：把整条链表全部压入栈，再从头节点重新遍历并依次弹栈比较。
它的优点是思路最简单、最适合笔试快速拿分，缺点是额外空间要用到 `O(N)`。

## 核心知识点
- 栈天然支持“从后往前”访问，正好补足链表不能逆向遍历的问题。
- 整条链表入栈后，栈顶顺序就等价于链表从尾到头的顺序。
- 再从头节点正向遍历一遍，就能完成首尾对照比较。
- 这是三种做法里实现最短、最不容易写错的一种。

## 图片转写 / 题意还原
这一题原始笔记没有配图，核心意思就是：

- 给定一个单链表
- 判断它是不是回文结构
- 版本 1 允许使用栈
- 额外空间不做严格限制，直接用 `O(N)` 换实现简单

这也是为什么标题里强调“笔试”。

## 图解
以：

```text
1 -> 2 -> 3 -> 2 -> 1
```

为例，整链表入栈后，栈顶到栈底相当于：

```text
1, 2, 3, 2, 1
```

然后从头开始比较：

```text
链表头 1  vs 栈顶 1
链表头 2  vs 栈顶 2
链表头 3  vs 栈顶 3
...
```

本质上就是：

```text
链表正向遍历
对比栈提供的反向遍历结果
```

## 解题思路
### 1. 第一遍遍历：整条链表入栈
从头到尾扫描链表，把每个节点或每个节点值都压进栈里。

扫描结束后，栈里从栈顶到栈底的顺序，就是链表从尾到头的顺序。

### 2. 第二遍遍历：从头与栈顶比较
再从头节点开始走：

- 当前节点代表“从左往右”
- 当前栈顶代表“从右往左”

如果某一对值不同，就不是回文。

### 3. 为什么这样一定对
因为回文的定义就是：

```text
第 1 个 = 倒数第 1 个
第 2 个 = 倒数第 2 个
...
```

而整链表入栈，正好把“倒序访问”这个能力造出来了。

## 典型例子
链表：

```text
1 -> 2 -> 3 -> 2 -> 1
```

全部入栈后，栈顶到栈底是：

```text
1, 2, 3, 2, 1
```

然后从头开始比较：

- 头节点 `1` 和栈顶 `1`
- 下一个 `2` 和栈顶 `2`
- 中间 `3` 和栈顶 `3`

全部相等，所以是回文。

如果链表是：

```text
1 -> 2 -> 3
```

那么第一轮比较就是：

- `1` 和 `3`

立刻失败。

## 复杂度
- 时间复杂度：`O(N)`
- 空间复杂度：`O(N)`

时间上是两次线性遍历，空间上是整条链表全部入栈。

## 易错点
- 栈里可以存节点，也可以只存节点值；如果只为判断回文，存值就够了。
- 不要把“比较节点引用是否相同”和“比较节点值是否相同”搞混。
- 空链表和单节点链表都应返回 `true`。
- 这是最简单的方法，但面试官通常会追问能不能把空间继续优化。

## 代码 / 伪代码
仓库里当前保留的是版本 3 的最终优化实现：

- [PalindromeLinkedList.java](../../../solutions-java/leetcode/p234_palindrome_linked_list/PalindromeLinkedList.java)

版本 1 的伪代码可以写成：

```java
Stack<ListNode> stack = new Stack<>();
ListNode cur = head;
while (cur != null) {
    stack.push(cur);
    cur = cur.next;
}
cur = head;
while (cur != null) {
    if (cur.val != stack.pop().val) {
        return false;
    }
    cur = cur.next;
}
return true;
```

## 记忆点
- 版本 1 最适合笔试：短、稳、快。
- 整链表入栈 = 人工获得倒序访问能力。
- 代价是空间 `O(N)`。
- 如果面试官继续追问，就自然过渡到版本 2 和版本 3。
