## 一句话结论
快慢指针本质上是在同一条链表上用不同速度制造“相对位移”，从而在一次遍历里拿到中点、上中点、下中点，或者它们前一个节点。
它是链表题里最常见的基础技巧之一，后面的回文链表、找环入口、链表归并切分都会反复用到。

## 核心知识点
- `slow` 一次走一步，`fast` 一次走两步，是最常见配置。
- 通过不同的初始位置和循环条件，可以控制最后停在“上中”“下中”“上中前一个”“下中前一个”。
- 快慢指针适合链表，因为链表没有随机访问能力，不能像数组那样直接取中间。
- 这类题最核心的不是记代码，而是先定义清楚“偶数长度时到底要哪个中点”。

## 图片转写 / 题意还原
原图列出了快慢指针的 4 个经典场景：

1. 输入链表头节点，奇数长度返回中点，偶数长度返回上中点
2. 输入链表头节点，奇数长度返回中点，偶数长度返回下中点
3. 输入链表头节点，奇数长度返回中点前一个，偶数长度返回上中点前一个
4. 输入链表头节点，奇数长度返回中点前一个，偶数长度返回下中点前一个

也就是说，这题不是单纯讲“找中点”，而是讲：

- 同样是中间位置
- 对于奇偶长度不同的链表
- 你到底要返回哪一个节点

## 图解
用偶数长度链表来感受最清楚：

```text
1 -> 2 -> 3 -> 4 -> 5 -> 6
```

如果我们的目标不同，那么“中间位置”的定义也不同：

```text
上中点       = 3
下中点       = 4
上中点前一个 = 2
下中点前一个 = 3
```

快慢指针的本质，就是通过不同起点制造这种落点差异：

```text
slow 一次 1 步
fast 一次 2 步
```

可以粗略记成：

```text
slow 起点越靠后，最后停得越靠后
fast 边界卡得越早，slow 停得越靠前
```

## 解题思路
### 1. 先区分 4 类目标
以链表长度为例：

- 长度 `5`：中点是第 `3` 个节点
- 长度 `6`：上中点是第 `3` 个节点，下中点是第 `4` 个节点

所以真正要做的是精确定义落点。

### 2. 为什么快慢指针能做到
如果 `fast` 每次走两步，`slow` 每次走一步，那么：

- 当 `fast` 到终点附近时
- `slow` 恰好只走了一半左右

因此只要控制好：

- 初始时 `slow`、`fast` 从哪里出发
- 循环在什么条件下停止

就能让 `slow` 稳定落到我们想要的位置。

### 3. 四种常见写法
#### 情况 1：奇数返回中点，偶数返回上中点
常见写法：

```java
slow = head.next;
fast = head.next.next;
while (fast.next != null && fast.next.next != null) {
    slow = slow.next;
    fast = fast.next.next;
}
```

这样：

- 奇数长度时，`slow` 最后在中点
- 偶数长度时，`slow` 最后在上中点

#### 情况 2：奇数返回中点，偶数返回下中点
让 `slow`、`fast` 都从 `head.next` 一带更靠后的位置开始，或者直接让 `slow = head.next, fast = head.next`。

本质就是让偶数长度时 `slow` 多走半拍。

#### 情况 3：奇数返回中点前一个，偶数返回上中点前一个
让 `slow` 比情况 1 再早一格开始，通常从 `head` 开始。

#### 情况 4：奇数返回中点前一个，偶数返回下中点前一个
通常让 `slow = head, fast = head.next`，这样偶数长度时会落在更靠前的位置。

### 4. 最稳妥的学习方式
不要死背代码，建议每种情况都拿下面两组长度手推一遍：

- 奇数：`1 -> 2 -> 3 -> 4 -> 5`
- 偶数：`1 -> 2 -> 3 -> 4 -> 5 -> 6`

只要能手推清楚，代码就不容易写错。

## 典型例子
链表：

```text
1 -> 2 -> 3 -> 4 -> 5 -> 6
```

### 如果要求“偶数返回上中点”
答案应为：

```text
3
```

### 如果要求“偶数返回下中点”
答案应为：

```text
4
```

### 如果要求“偶数返回上中点前一个”
答案应为：

```text
2
```

### 如果要求“偶数返回下中点前一个”
答案应为：

```text
3
```

这 4 种落点，正是这题要刻意区分的地方。

## 复杂度
- 时间复杂度：`O(N)`
- 空间复杂度：`O(1)`

整个过程只扫描链表一遍，也不需要额外容器。

## 易错点
- 没先说清楚“偶数长度到底取上中还是下中”，代码就很容易错。
- 边界长度 `0`、`1`、`2`、`3` 的处理最容易写崩。
- 循环条件里经常要写 `fast.next != null && fast.next.next != null`，顺序不能乱。
- 有些题要返回“中点前一个”，这时要特别注意起始位置是否要前移。

这题可以先记一个最常用的模板：奇数返回中点，偶数返回上中点。

```java
if (head == null || head.next == null || head.next.next == null) {
    return head;
}
ListNode slow = head.next;
ListNode fast = head.next.next;
while (fast.next != null && fast.next.next != null) {
    slow = slow.next;
    fast = fast.next.next;
}
return slow;
```

## 记忆点
- 快慢指针不是只有一种写法，关键在“你想停在哪”。
- 先定奇数时返回谁，再定偶数时返回谁。
- 起始位置决定偏移，循环条件决定停止时机。
- 后面的回文链表、找环入口、链表归并切分，都会复用这个套路。
