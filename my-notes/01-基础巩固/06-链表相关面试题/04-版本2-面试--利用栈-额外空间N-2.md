## 一句话结论
版本 2 仍然借助栈判断回文，但不再把整条链表都压栈，而是只把后半段压栈，因此额外空间可以从 `O(N)` 降到 `O(N/2)`。
它比版本 1 更像面试里的“进一步优化”，代码依然不算难，但已经需要先用快慢指针找到中点。

## 核心知识点
- 如果只需要比较左右两半，其实没必要把整条链表都入栈。
- 快慢指针可以帮助我们快速定位后半段起点。
- 只把后半段入栈后，再从头节点与栈顶逐个比较即可。
- 它是版本 1 和版本 3 之间的过渡方案。

这题的重点是：

- 不改变链表结构
- 仍然借助栈

因此自然想到：

- 先找到中点
- 只对后半部分做栈处理

## 解题思路
### 1. 用快慢指针找到中点
和上一题一样：

- `slow` 一次一步
- `fast` 一次两步

循环结束后：

- 奇数长度时，`slow` 落在中点
- 偶数长度时，`slow` 落在上中点

### 2. 只把右半部分压栈
接下来从 `slow.next` 开始遍历，把右半段节点压栈。

这样栈顶顺序就是右半段从尾到头的顺序。

### 3. 从头节点开始比对
再让一个指针从 `head` 出发，每弹出一个栈顶，就和当前左侧节点比较。

因为栈里只有一半元素，所以只需要比较后半段长度那么多次。

### 4. 为什么空间能降一半
因为回文判断只关心对称位置：

- 左半和右半一一对应
- 中点本身不用重复存

所以我们不需要整条链表都入栈，只需要把“右侧答案”准备好。

## 典型例子
链表：

```text
1 -> 2 -> 3 -> 2 -> 1
```

快慢指针找到中点 `3`。

把右半部分：

```text
2 -> 1
```

压栈后，栈顶依次是：

```text
1, 2
```

然后从头开始比较：

- 左侧 `1` 对栈顶 `1`
- 左侧 `2` 对栈顶 `2`

栈空后说明后半段全部匹配成功，结果为回文。

## 复杂度
- 时间复杂度：`O(N)`
- 空间复杂度：`O(N/2)`，通常也可以写成 `O(N)`

虽然渐进复杂度写法上仍然属于 `O(N)`，但相比版本 1，常数空间已经更优。

## 易错点
- 奇数长度时，中点不用压栈，应该从 `slow.next` 开始。
- 偶数长度和奇数长度的中点定义要先统一，否则后半段起点容易错。
- 这一版虽然省了一半空间，但仍然不是最优解。
- 口头说 `O(N/2)` 是为了体现优化幅度，严格大 O 记号通常仍写 `O(N)`。

## 代码 / 伪代码
这一版可以写成：

```java
if (head == null || head.next == null) {
    return true;
}
ListNode slow = head;
ListNode fast = head;
while (fast.next != null && fast.next.next != null) {
    slow = slow.next;
    fast = fast.next.next;
}
Stack<ListNode> stack = new Stack<>();
ListNode cur = slow.next;
while (cur != null) {
    stack.push(cur);
    cur = cur.next;
}
cur = head;
while (!stack.isEmpty()) {
    if (cur.val != stack.pop().val) {
        return false;
    }
    cur = cur.next;
}
return true;
```

可以把它看成：

- 版本 1：整条链表入栈
- 版本 2：后半条链表入栈
- 版本 3：后半条链表不入栈，直接逆序

## 记忆点
- 版本 2 的关键，不是“还是用栈”，而是“只用半个栈”。
- 快慢指针先切半，再用栈保存右半边。
- 中点本身通常不参与压栈。
- 它是从笔试做法过渡到最优做法的中间方案。
