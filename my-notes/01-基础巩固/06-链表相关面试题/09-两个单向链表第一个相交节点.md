## 一句话结论
两个单链表找第一个相交节点，不能只考虑“都无环”的简单情况，完整题目要同时讨论无环和有环两种结构。
核心结论只有一个：先分别判断两条链表是否有环，再按“都无环 / 都有环 / 一有环一无环”分类处理。

## 核心知识点
- 两条无环链表如果相交，从相交点开始后面的所有节点都会重合。
- 一条有环、一条无环，不可能相交。
- 两条有环链表要继续分成“同一个入环点”和“不同入环点”两种情况。
- 这题本质是“判环 + 分类讨论 + 长度对齐”。

## 图片转写 / 题意还原
原图内容可以整理为：

- 常见面试题
- 给定两个可能有环也可能无环的单链表，头节点分别是 `head1` 和 `head2`
- 实现一个函数：
  - 如果两个链表相交，返回相交的第一个节点
  - 如果不相交，返回 `null`
- 要求：
  - 如果两个链表长度之和为 `N`
  - 时间复杂度达到 `O(N)`
  - 额外空间复杂度达到 `O(1)`

原始笔记还强调了一个很关键的判断：

- 不存在“一条有环、一条无环却相交”的情况

## 图解
### 情况 1：两个无环链表相交

```text
head1: 1 -> 2 -> 3 -> 6 -> 7
                    ^
head2:      4 -> 5 -|
```

从 `6` 开始，两条链表后续完全重合，所以第一个相交点就是 `6`。

### 情况 2：两个有环链表，共同入环点相同

```text
head1: 1 -> 2 -> 3 -> 4 -> 5
                 ^         |
                 |_________|

head2:    9 -> 8 -|
```

两条链表都从 `3` 进入同一个环，因此答案可能在入环前，也可能就是 `3`。

### 情况 3：两个有环链表，入环点不同但在同一个环里

```text
head1: 1 -> 2 -> 3 -> 4 -> 5 -> 6
                    ^         |
                    |_________|

head2: 9 -> 8 -> 5 -> 6 -> 3 -> 4 -> ...
```

这时：

```text
loop1 != loop2
但 loop2 在 loop1 所在的环上
```

所以它们仍然相交。

## 解题思路
### 1. 第一步：先判断每条链表是否有环
先分别对 `head1`、`head2` 调用“找入环点”函数。

会有 3 种总情况：

1. 两条都无环
2. 两条都有环
3. 一条有环，一条无环

### 2. 情况一：两条都无环
这是最经典的版本。

做法：

1. 分别走到两条链表尾部，顺便统计长度差
2. 如果尾节点不是同一个节点，说明不相交
3. 如果尾节点相同，说明一定相交
4. 让较长链表先走长度差步
5. 两个指针再一起往前走，第一次相遇的位置就是第一个相交节点

这是仓库里 `p160` 那份代码的做法。

### 3. 情况二：一条有环，一条无环
这种情况直接返回 `null`。

原因是：

- 如果无环链表和有环链表在某处相交
- 那么从相交点之后，无环链表也会顺着公共路径进入那个环

它就不再是无环链表了，和前提矛盾。

所以这种情况不可能发生。

### 4. 情况三：两条都有环
这时再分两种子情况。

#### 子情况 A：入环点相同
假设两条链表入环点都是 `loop`。

那说明它们在入环之前可能就已经相交，也可能恰好在入环点才第一次相交。

这时只需要把“入环点当成终点”，在 `[head, loop)` 这段无环区域里，重复无环链表的处理逻辑即可。

如果在入环前没有提前相交，那么答案就是这个共同的入环点。

#### 子情况 B：入环点不同
假设入环点分别是 `loop1` 和 `loop2`，并且 `loop1 != loop2`。

这时只需要从 `loop1` 出发，沿着环走一圈：

- 如果过程中能遇到 `loop2`
  - 说明两个链表进入的是同一个环
  - 它们一定相交
  - 返回 `loop1` 或 `loop2` 都可以，因为两者都是公共环上的节点
- 如果走一圈都遇不到 `loop2`
  - 说明两个环完全独立
  - 两条链表不相交

## 典型例子
### 例 1：无环相交

```text
1 -> 2 -> 3 -> 6 -> 7
         /
4 -> 5 -
```

两条链表从 `6` 开始重合，所以第一个相交节点是：

```text
6
```

### 例 2：有环且同一个入环点

```text
1 -> 2 -> 3 -> 4 -> 5
              ^     |
              |_____|

9 -> 8 -------|
```

这时两条链表第一个相交节点可能在入环前，也可能就是共同入环点 `3`。

### 例 3：有环但不同入环点、共享同一个环

```text
head1: 1 -> 2 -> 3 -> 4 -> 5 -> 6 -> 3 ...
head2: 9 -> 8 -> 5 -> 6 -> 3 -> 4 -> 5 ...
```

两条链表入环位置不同，但进入了同一个环，所以它们仍然相交。

## 复杂度
- 时间复杂度：`O(N)`
- 空间复杂度：`O(1)`

其中 `N` 可以理解为两条链表整体访问到的节点总规模。

## 易错点
- 不要只会做“两个无环链表相交”这一半，完整题目要把有环情况补齐。
- “一条有环、一条无环”是直接不可能，不要再浪费时间细分。
- 两条有环链表入环点不同，不代表一定不相交，可能是在同一个环上不同位置入环。
- 无环相交的判断标准是“尾节点是否同一个对象”，不是尾节点值是否相同。

## 代码 / 伪代码
仓库中已有无环版本的实现：

- [IntersectionOfTwoLinkedLists.java](../../../solutions-java/leetcode/p160_intersection_of_two_linked_lists/IntersectionOfTwoLinkedLists.java)

这份代码的核心逻辑是：

```java
1. 分别走到两条链表尾部
2. 统计长度差 diff
3. 如果尾节点不同，返回 null
4. 长链表先走 diff 步
5. 再一起走到第一次相遇
```

关键代码片段：

```java
while (point1.next != null) {
    point1 = point1.next;
    diff++;
}
while (point2.next != null) {
    point2 = point2.next;
    diff--;
}
if (point1 != point2) {
    return null;
}
```

完整题的总框架可以写成：

```java
loop1 = getLoopNode(head1);
loop2 = getLoopNode(head2);

if (loop1 == null && loop2 == null) {
    return noLoop(head1, head2);
}
if (loop1 != null && loop2 != null) {
    return bothLoop(head1, loop1, head2, loop2);
}
return null;
```

## 记忆点
- 先判环，再分类，是这题唯一正确的总路线。
- 无环相交看尾节点；有环相交看入环点关系。
- 一有环一无环，不可能相交。
- 两个不同入环点，也可能在同一个环里相交。
