## 一句话结论
判断链表是否为回文，最直接的方法是借助栈；但如果想把额外空间压到 `O(1)`，就要用快慢指针先找到中点，再把右半部分逆序后做双侧比对。
这题的重点不只是“判断回文”，而是理解快慢指针如何把链表拆成左右两半，以及为什么比较完还要把链表恢复原状。

## 核心知识点
- 回文结构的本质是“从左往右”和“从右往左”读起来一样。
- 链表不能像数组那样从尾部直接访问，所以常见做法是借助栈或逆序链表。
- 快慢指针负责找中点，链表反转负责制造“从右往左”的比较路径。
- 工程上如果题目没有明确允许修改链表，做完判断后最好把链表恢复。

## 图片转写 / 题意还原
原始笔记虽然只有题目名，但题目可以还原成：

- 给定一个单链表头节点
- 判断这个链表是不是回文结构
- 进一步要求使用快慢指针进行优化

也就是说，这题在为后面 3 个版本做铺垫：

- 版本 1：整条链表入栈，空间 `O(N)`
- 版本 2：半条链表入栈，空间 `O(N/2)`
- 版本 3：右半部分原地逆序，空间 `O(1)`

这一题主要是在讲版本 3 背后的总体思路。

## 图解
以回文链表：

```text
1 -> 2 -> 3 -> 2 -> 1
```

为例，整体流程可以看成：

### 第 1 步：快慢指针找到中点

```text
1 -> 2 -> 3 -> 2 -> 1
          ^
         slow
```

### 第 2 步：逆序右半部分

```text
左边：1 -> 2 -> 3
右边：2 -> 1

逆序后右边变成：
1 -> 2
```

### 第 3 步：左右同时比较

```text
left : 1 -> 2 -> 3
right: 1 -> 2
```

### 第 4 步：恢复链表
把右半部分再反转回去，重新接回中点后面。

## 解题思路
### 1. 先用快慢指针找中点
让：

- `slow` 一次走一步
- `fast` 一次走两步

当循环结束时：

- 奇数长度时，`slow` 落在中点
- 偶数长度时，`slow` 落在上中点

这样就能把链表分成左右两段。

### 2. 逆序右半部分
从 `slow.next` 开始，把右半部分逆序。

逆序之后，链表会变成一种“左右两端都能往中间走”的结构：

```text
head -> ... -> slow <- ... <- tail
```

这样就能用两个指针：

- `left` 从头往右
- `right` 从尾往左

逐个比较值是否相等。

### 3. 为什么要先断开
在逆序前，代码里有一步：

```java
slow.next = null;
```

这一步非常关键。

因为如果不断开，后面在反转指针方向时，可能形成环，链表结构会被搞乱。

### 4. 双指针比较
逆序完成后：

- `left = head`
- `right = 逆序后右半部分的头`

只要有一对值不相等，就不是回文。

### 5. 比较完再恢复链表
这一步不是所有题解都做，但仓库里的实现做了，而且这是好习惯。

原因是：

- 判断回文只是查询行为
- 不应该把原链表永久改坏

所以比较完成后，再把右半部分逆回去，恢复原结构。

## 典型例子
### 例 1：奇数长度回文

```text
1 -> 2 -> 3 -> 2 -> 1
```

快慢指针找到中点 `3`。

把右半部分：

```text
2 -> 1
```

逆序后变成：

```text
1 -> 2
```

再与左侧：

```text
1 -> 2 -> 3
```

从两边往中间比较，就会全部相等。

### 例 2：偶数长度非回文

```text
1 -> 2 -> 3 -> 4
```

上中点是 `2`，右半部分逆序后从 `4` 开始比较：

- `1` 和 `4` 不等

立刻就能返回 `false`。

## 复杂度
- 时间复杂度：`O(N)`
- 空间复杂度：`O(1)`

其中时间主要花在：

- 找中点一次
- 逆序一次
- 比较一次
- 恢复一次

总量仍然是线性级别。

## 易错点
- 偶数长度时，`slow` 最终落在上中点，不要把右半部分起点搞错。
- 逆序前如果不先断开 `slow.next`，容易形成环。
- 比较时不一定要两边同时走到 `null`，只要右侧那段走完即可。
- 很多人会漏掉“恢复链表”这一步，面试里最好主动提一下。

## 代码 / 伪代码
仓库中的对应实现：

- [PalindromeLinkedList.java](../../../solutions-java/leetcode/p234_palindrome_linked_list/PalindromeLinkedList.java)

核心流程可以概括成：

```java
1. 用快慢指针找到中点 slow
2. 从 slow.next 开始逆序右半部分
3. left 从 head 出发，right 从右半部分头出发
4. 逐个比较
5. 把右半部分再逆回去
```

仓库代码里最值得注意的几行是：

```java
while (fast.next != null && fast.next.next != null) {
    slow = slow.next;
    fast = fast.next.next;
}
```

以及：

```java
slow.next = null;
```

还有最后的恢复过程：

```java
cur = pre.next;
pre.next = null;
while (cur != null) {
    next = cur.next;
    cur.next = pre;
    pre = cur;
    cur = next;
}
```

## 记忆点
- 回文链表 = 左右对称，本质上要拿到“反向遍历能力”。
- 快慢指针负责找中点，逆序负责制造右往左比较路径。
- `slow.next = null` 是防止形成环的关键。
- 做完比较再恢复链表，是更完整也更稳妥的写法。
