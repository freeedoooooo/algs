## 一句话结论
判断单链表是否有环，最直接的方法是用哈希表记录访问过的节点；更经典、更省空间的方法是快慢指针。
如果题目进一步要求“返回第一个入环节点”，那就要在快慢指针第一次相遇后，再做一次同步前进。

## 核心知识点
- 哈希表版的本质是“节点是否重复访问”。
- 快慢指针版的本质是“快指针会不会在环里追上慢指针”。
- 仅判断有无环和定位入环点，属于同一套方法的两个层次。
- 面试里通常更推荐快慢指针，因为空间复杂度能做到 `O(1)`。

## 图片转写 / 题意还原
原始笔记没有配图，只有两个提示：

- 版本 1：利用哈希表
- 版本 2：快慢指针，思路有点像追击问题

这题可以拆成两个子问题：

1. 链表有没有环
2. 如果有环，环从哪个节点开始

而仓库代码对应的是更完整、更难一点的版本：返回第一个入环节点。

## 图解
假设链表如下：

```text
1 -> 2 -> 3 -> 4 -> 5
          ^         |
          |_________|
```

其中：

```text
3 是入环点
```

### 第 1 阶段：快慢指针相遇

```text
slow: 每次走 1 步
fast: 每次走 2 步
```

进入环后，`fast` 会像跑道追击一样追上 `slow`：

```text
slow 和 fast 会在环中的某个点相遇
但这个点不一定是入环点
```

### 第 2 阶段：找入环点
把一个指针放回头部，另一个留在相遇点：

```text
head -> 1 -> 2 -> 3 -> ...
meet -> 环中的某处
```

然后两个指针都改成一次走一步：

```text
head 指针：1 -> 2 -> 3
meet 指针：... -> ... -> 3
```

再次相遇的位置就是入环点。

## 解题思路
### 1. 版本 1：哈希表判重
从头开始遍历链表，把每个节点放进 `HashSet`：

- 如果某个节点第一次出现，就加入集合
- 如果某个节点再次出现，说明有环

如果一路走到 `null`，说明无环。

优点：

- 思路最直观
- 写法简单

缺点：

- 需要 `O(N)` 额外空间

### 2. 版本 2：快慢指针判断有环
令：

- `slow` 一次走一步
- `fast` 一次走两步

如果链表无环：

- `fast` 一定会先走到 `null`

如果链表有环：

- `fast` 一旦进环，就会不断逼近 `slow`
- 最终两者一定会在环内某处相遇

这就像跑道上的追击问题。

### 3. 为什么相遇后能找到入环点
设：

- 从头到入环点距离为 `a`
- 入环点到相遇点距离为 `b`
- 相遇点再回到入环点距离为 `c`
- 环长为 `b + c`

根据快指针速度是慢指针两倍，可以推导出：

```text
a = c + (k - 1) * 环长
```

这意味着：

- 从头节点走到入环点
- 和从相遇点继续往前走到入环点

它们相差若干整圈，最终会在入环点重合。

所以：

1. 快慢指针先相遇
2. 其中一个指针回到头节点
3. 两个指针都改成一次走一步
4. 再次相遇的位置就是第一个入环节点

## 典型例子
链表：

```text
1 -> 2 -> 3 -> 4 -> 5
          ^         |
          |_________|
```

这里：

- `3` 是入环点

### 快慢指针阶段
- `slow` 每次一步
- `fast` 每次两步

经过若干轮后，它们会在环中的某个节点相遇，比如 `4` 或 `5`，不一定正好在入环点。

### 找入环点阶段
把一个指针放回头节点 `1`，另一个留在相遇点。

然后两个指针都一次走一步，再次相遇时就会在：

```text
3
```

## 复杂度
- 时间复杂度：
  - 哈希表版：`O(N)`
  - 快慢指针版：`O(N)`
- 空间复杂度：
  - 哈希表版：`O(N)`
  - 快慢指针版：`O(1)`

## 易错点
- 判断无环时，`fast == null` 或 `fast.next == null` 的边界要先处理好。
- 很多人只记得“会相遇”，但忘了“第一次相遇点不一定是入环点”。
- 如果题目只问“是否有环”，返回布尔值即可；如果问“入环节点”，还要多做第二阶段。
- 初始化 `slow`、`fast` 的位置不同，`while` 条件写法也会不同，代码风格要自洽。

## 代码 / 伪代码
仓库中的对应实现：

- [LinkedListCycleII.java](../../../solutions-java/leetcode/p142_linked_list_cycle_ii/LinkedListCycleII.java)

这份代码实现的是“返回第一个入环节点”。

第一阶段：判断是否有环

```java
ListNode slow = head.next;
ListNode fast = head.next.next;
while (slow != fast) {
    if (fast.next == null || fast.next.next == null) {
        return null;
    }
    slow = slow.next;
    fast = fast.next.next;
}
```

第二阶段：寻找入环点

```java
fast = head;
while (slow != fast) {
    slow = slow.next;
    fast = fast.next;
}
return slow;
```

如果只写哈希表版，伪代码很简单：

```java
Set<ListNode> seen = new HashSet<>();
while (head != null) {
    if (seen.contains(head)) {
        return head;
    }
    seen.add(head);
    head = head.next;
}
return null;
```

## 记忆点
- 哈希表版看“重复访问”，快慢指针版看“追上”。
- 第一次相遇点不一定是入环点。
- 找入环点的固定套路：一个回头，一个留在相遇点，然后一起一步一步走。
- 这题和快慢指针是强绑定的经典题。
