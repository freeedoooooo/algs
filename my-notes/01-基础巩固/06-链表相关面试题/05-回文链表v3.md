## 一句话结论
版本 3 是这组题里最优的一版：先用快慢指针找到中点，再把链表右半部分原地反转，然后从两端向中间比较，额外空间只需 `O(1)`。
这也是最典型的面试写法，因为它同时考察快慢指针、链表反转和边界处理。

## 核心知识点
- 右半部分原地反转，本质是在不借助栈的情况下制造“逆序访问能力”。
- 快慢指针负责切半，反转负责对称比较，恢复链表负责保证结构不被永久修改。
- 这版代码最省空间，但也是三版里最考验细节的一版。
- 形成环、起点选错、恢复失败，是最常见的三类错误。

## 图片转写 / 题意还原
原图内容可以转写为：

- 常见面试题
- 给定一个单链表的头节点 `head`，请判断该链表是否为回文结构
- 方法 1：栈方法特别简单，适合笔试
- 方法 2：改原链表的方法需要注意边界，适合面试

也就是说，这一版强调的是：

- 面试不只要写出答案
- 还要把空间优化到更低
- 但与此同时，链表边界处理会明显更复杂

## 图解
以回文链表：

```text
1 -> 2 -> 3 -> 2 -> 1
```

为例，版本 3 的整体过程可以看成 4 步：

### 第 1 步：找到中点

```text
1 -> 2 -> 3 -> 2 -> 1
          ^
         slow
```

### 第 2 步：断开并反转右半部分

```text
左半边：1 -> 2 -> 3
右半边：2 -> 1

反转后：
左半边：1 -> 2 -> 3
右半边：1 -> 2
```

### 第 3 步：两边同时比较

```text
left :  1 -> 2 -> 3
right:  1 -> 2
```

只要每一对都相等，就说明是回文。

### 第 4 步：恢复链表

```text
把右半边再反转回去
重新接回到 3 后面
```

最终又变回：

```text
1 -> 2 -> 3 -> 2 -> 1
```

## 解题思路
### 1. 用快慢指针找到中点
令：

- `slow` 一次走一步
- `fast` 一次走两步

当循环结束时：

- 奇数长度时，`slow` 在中点
- 偶数长度时，`slow` 在上中点

然后从 `slow.next` 开始处理右半段。

### 2. 先断开，再反转右半部分
反转前先执行：

```java
slow.next = null;
```

这样做是为了把左右两段断开，避免在指针反转时形成环。

然后把右半部分原地反转，最终会形成：

```text
head -> ... -> slow <- ... <- tail
```

### 3. 左右同时向中间比较
此时：

- `left` 从 `head` 出发
- `right` 从反转后右半部分头节点出发

只要有一对值不同，就返回 `false`。

### 4. 比较后把链表恢复
这一步很加分。

因为我们为了判断回文而改动了链表结构，最好在返回结果前把它恢复为原始状态。

这既体现代码完整性，也能避免后续逻辑依赖原链表时出问题。

## 典型例子
链表：

```text
1 -> 2 -> 3 -> 2 -> 1
```

找到中点 `3` 后，右半部分是：

```text
2 -> 1
```

反转为：

```text
1 -> 2
```

于是比较时就是：

- 左边 `1, 2`
- 右边 `1, 2`

全部相同，所以是回文。

如果链表是：

```text
1 -> 2 -> 2 -> 3
```

反转右半后第一对比较就可能失败，因此直接返回 `false`。

## 复杂度
- 时间复杂度：`O(N)`
- 空间复杂度：`O(1)`

这也是它比前两个版本更适合作为“最优解”的原因。

## 易错点
- `slow.next = null` 很容易漏掉，一漏就可能形成环。
- 奇数长度时，中点本身不需要参与左右比较两次。
- 反转结束后，右半部分头节点已经变了，比较起点要用新的头。
- 如果题目或业务场景要求链表保持不变，恢复步骤不能省略。

## 代码 / 伪代码
仓库中的对应实现：

- [PalindromeLinkedList.java](../../../solutions-java/leetcode/p234_palindrome_linked_list/PalindromeLinkedList.java)

这份代码几乎就是版本 3 的标准答案。主流程如下：

```java
1. 快慢指针找中点
2. 断开 slow.next
3. 反转右半部分
4. 左右两端比较
5. 再反转一次，把链表恢复
```

其中最关键的三段分别是：

```java
while (fast.next != null && fast.next.next != null) {
    slow = slow.next;
    fast = fast.next.next;
}
```

```java
slow.next = null;
while (cur != null) {
    next = cur.next;
    cur.next = pre;
    pre = cur;
    cur = next;
}
```

```java
cur = pre.next;
pre.next = null;
while (cur != null) {
    next = cur.next;
    cur.next = pre;
    pre = cur;
    cur = next;
}
```

## 记忆点
- 版本 3 的本质，是“用链表反转代替栈”。
- 空间最优，但边界最容易错。
- 找中点、断开、反转、比较、恢复，这 5 步最好按固定顺序记。
- 面试里如果能主动提恢复链表，通常会更加分。
