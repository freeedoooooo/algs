## 一句话结论
这题难的不是“复制节点值”，而是“把节点之间那套复杂的指向关系也原样复制出来”。
它真正想练的是：

```text
当链表里不只有 next
还多了一层 random 关系时
你怎么把这张关系网完整复制
```

最自然的入门写法是哈希表映射，面试里更想听到的是原地穿插版，因为它把“原节点 -> 复制节点”的映射直接长进了链表结构里，把额外空间压到了 `O(1)`。

## 这题为什么比普通链表复制难
普通单链表复制其实很简单：

- 新建一个头节点
- 按 `next` 方向一路复制过去

因为普通链表里，结构关系只有一条：

```text
当前节点 -> 下一个节点
```

但 Random 链表不一样。
每个节点除了 `next`，还可能有一条 `random` 指针，指向：

- 链表中的任意一个节点
- 或者 `null`

这就意味着你复制的已经不是一条单纯的链，而是一张“链表骨架 + 随机跳转关系”的结构网。

所以这题真正难的地方不是建节点，而是：

```text
新节点的 random
到底该指向哪个“新节点”
```

## 如果只复制值，会错在哪里
假设原链表是：

```text
A -> B -> C
```

并且：

- `A.random = C`
- `B.random = A`
- `C.random = null`

如果你只是把值复制成：

```text
A' -> B' -> C'
```

那还远远不够。
因为你还得保证：

- `A'.random = C'`
- `B'.random = A'`
- `C'.random = null`

注意这里最关键的一点：

```text
新链表的 random
不能再指回原链表
```

也就是说，复制的不是节点值，而是整套指针关系。

## 最自然的思路：先建立映射
这题最容易想到的办法，其实非常合理：

- 既然我要知道“原节点对应哪个新节点”
- 那就直接建一张映射表

这就是哈希表版。

## 版本 1：哈希表版
哈希表版的思路特别顺：

1. 第一遍遍历，为每个原节点创建一个复制节点
2. 用 `map.put(原节点, 复制节点)` 建立映射
3. 第二遍遍历，根据原节点的 `next` 和 `random` 去补新节点的指针

这样做为什么自然？

因为只要映射在，你就能随时回答两个问题：

- 原节点 `cur.next` 对应的新节点是谁
- 原节点 `cur.random` 对应的新节点是谁

于是代码层面就很好写：

```java
map.get(cur).next = map.get(cur.next);
map.get(cur).random = map.get(cur.random);
```

这个版本的优点很明显：

- 好理解
- 好实现
- 不容易写错

缺点也一样明显：

- 额外空间是 `O(N)`

所以它很适合作为第一反应，但不是面试里最亮眼的版本。

## 面试真正想听什么：不用哈希表怎么办
如果不能用哈希表，本质上还是那个问题：

```text
原节点对应的复制节点
到底怎么快速找到
```

原地版最巧的地方就在这。
它没有否定“需要映射”这件事，而是换了一种存映射的方法：

- 不把映射存在哈希表里
- 而是把映射直接塞进链表结构本身

## 版本 2：原地穿插版
原地版第一步会把每个复制节点插到原节点后面。

比如原链表：

```text
A -> B -> C
```

处理后变成：

```text
A -> A' -> B -> B' -> C -> C'
```

这一步一完成，就等于天然得到了一张映射表：

```text
原节点的复制节点 = 原节点.next
```

这就是整题最核心的等式。

一旦这条等式成立，后面很多事都会突然变简单。

## 为什么这样就能设置 random
假设当前原节点是 `cur`。

如果：

- `cur.random = R`

那么根据刚才那条映射关系：

- `cur` 的复制节点就是 `cur.next`
- `R` 的复制节点就是 `R.next`

于是复制节点的 `random` 就可以直接写成：

```java
cur.next.random = cur.random != null ? cur.random.next : null;
```

这一步非常巧，因为它本质上是在说：

```text
我不需要查哈希表
因为目标复制节点就贴在原目标节点后面
```

这也是为什么原地版常被认为是这题真正的面试亮点。

## 原地版完整在做哪三件事
整题可以固定记成 3 次遍历。

### 第一次遍历：穿插复制节点
把：

```text
A -> B -> C
```

变成：

```text
A -> A' -> B -> B' -> C -> C'
```

目的不是为了好看，而是为了把“映射关系”长到结构里。

### 第二次遍历：补复制节点的 random
这时每个原节点都能通过 `next` 找到自己的复制节点。
所以只需要顺着原链表走一遍，就能把所有复制节点的 `random` 补齐。

### 第三次遍历：拆分新旧链表
前两步做完后，原链表和新链表是交错在一起的。
最后要把它们拆开：

- 原链表恢复原状
- 新链表单独成链

这一步做完，才真正得到答案。

## 用一个例子看懂原地版
假设原链表是：

```text
A -> B -> C
```

并且：

- `A.random = C`
- `B.random = A`
- `C.random = null`

### 第 1 步：穿插

```text
A -> A' -> B -> B' -> C -> C'
```

此时天然有：

- `A` 的复制是 `A'`
- `B` 的复制是 `B'`
- `C` 的复制是 `C'`

### 第 2 步：补 random
因为：

- `A.random = C`

所以：

- `A'.random = C'`

因为：

- `B.random = A`

所以：

- `B'.random = A'`

而：

- `C.random = null`

所以：

- `C'.random = null`

### 第 3 步：拆开
最后把交错结构拆成两条链：

原链表恢复成：

```text
A -> B -> C
```

新链表变成：

```text
A' -> B' -> C'
```

并且它们的 `random` 关系完全对应。

## 这题到底在练什么能力
这题不只是复制题，它实际上在练 4 个层次的能力：

- 能不能看出“复制的核心不是值，而是关系”
- 能不能先想到哈希表这种直接映射
- 能不能进一步把映射从外部结构压回原链表
- 能不能在改动结构后，再把原链表完整恢复

所以它和这一章前面的很多题是一条主线：

- 快慢指针是在链表里找结构信息
- 回文链表是在链表里先改结构再恢复
- 分区题是在链表里拆链再拼链
- 这题是在链表里把“映射关系”临时嵌进结构

本质上都在练“能不能安全操控链表结构”。

## 复杂度
- 时间复杂度：
  - 哈希表版：`O(N)`
  - 原地穿插版：`O(N)`
- 空间复杂度：
  - 哈希表版：`O(N)`
  - 原地穿插版：`O(1)`

原地版虽然也走了三遍，但每一遍都是线性的，所以总复杂度仍然是 `O(N)`。

## 易错点
- 以为复制的是值，结果让新链表的 `random` 又指回了原链表。
- 第二轮设置 `random` 时，忘了 `cur.random` 可能是 `null`。
- 第三轮拆分时，只顾着拿新链表，忘了把原链表恢复。
- 没有真正记住 `复制节点 = 原节点.next` 这条核心关系，导致后两轮写得很乱。


其中最值得记住的两处是：

```java
cur.next = new Node(cur.val);

```

以及：

```java
copy.random = cur.random != null ? cur.random.next : null;
```

前者负责把映射关系“长进链表”，后者负责真正利用这层映射。

## 和记忆方式
- 哈希表版最好想，因为它是“显式建映射”。
- 原地版最好考，因为它是“把映射藏进结构”。
- 整题最核心的一句话是：`原节点的复制节点永远在原节点后面`。
- 固定记三步就够了：穿插、补 `random`、拆分。
