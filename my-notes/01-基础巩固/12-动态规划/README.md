## 本章结论
- Dijkstra 是单源最短路的经典贪心算法，但要求边权不能为负。
- Dijkstra 的常规写法每轮找最小未锁定点，改进版用小根堆优化选点过程。
- 暴力递归的本质是“把原问题变成规模更小的同类子问题”。
- 暴力递归必须有 base case、决策过程，而且默认不缓存子问题答案。
- 汉诺塔和逆序栈，都是训练“递归函数职责设计”的经典题。

## 图片转写 / 章节笔记还原
原始笔记里这章主要有两条主线：

### 1. Dijkstra 的核心过程
1. 必须指定一个源点
2. 生成一张源点到各点的最小距离表，一开始只有源点到自己为 `0`
3. 每次从未锁定的记录里选最小距离点，用它的出边去更新其他点距离
4. 当所有点都被处理过，最小距离表就确定了

### 2. 暴力递归是什么
1. 把问题转化为规模缩小后的同类子问题
2. 有明确的停止条件 `base case`
3. 拿到子问题答案后，要有整合决策过程
4. 默认不记录每个子问题的解

## 记忆点
- Dijkstra 是贪心，不是动态规划。
- Dijkstra 不能处理负权边。
- 暴力递归先想：子问题、停止条件、答案整合。
- 递归不是背模板，而是设计函数职责。
