## 一句话结论
派对最大快乐值题的本质，是一棵多叉树上的树形 DP：每个员工来不来，会直接影响他的所有直接下级能不能来。
最标准的做法，是让每个节点返回两个值：自己来时的最大快乐值、自己不来时的最大快乐值。

## 核心知识点
- 这是多叉树版本的递归套路题，本质和二叉树 `Info` 模型完全一样。
- 当前员工有两种状态：
  - 来
  - 不来
- 如果当前员工来，则所有直接下级都不能来。
- 如果当前员工不来，则每个直接下级都可以自由选择“来或不来”中的更优值。
- 所以每个节点只需要返回：
  - `yes`：自己来时的最大快乐值
  - `no`：自己不来时的最大快乐值

## 图片转写 / 题意还原
原图内容可以整理为：

- 派对的最大快乐值
- 这个公司现在要办 party，你可以决定哪些员工来、哪些员工不来，规则：
1. 如果某个员工来了，那么这个员工的所有直接下级都不能来
2. 派对的整体快乐值是所有到场员工快乐值的累加
3. 你的目标是让派对的整体快乐值尽量大
- 给定一棵多叉树的头节点 `boss`，请返回派对的最大快乐值

这题其实不是贪心题，而是标准树形 DP 题。

## 图解
假设组织结构如下：

```text
        A(10)
      /   |   \
   B(5)  C(20) D(1)
          |
         E(30)
```

括号里是员工快乐值。

### 情况 1：A 来
如果老板 `A` 来：

- `B/C/D` 都不能来
- 但更下一层可以继续按规则选择

所以：

```text
yes(A) = 10 + no(B) + no(C) + no(D)
```

### 情况 2：A 不来
如果老板 `A` 不来：

- `B/C/D` 每个人都可以自己决定来不来
- 对每个下级都取最优值

所以：

```text
no(A) = max(yes(B), no(B))
      + max(yes(C), no(C))
      + max(yes(D), no(D))
```

这就是整题的核心递推关系。

## 解题思路
### 1. 当前节点的状态只有两个
以当前员工 `x` 为头时，不需要想太多状态，只要考虑两件事：

```text
x 来
x 不来
```

这是这题最关键的降维。

### 2. 如果当前员工来
那么根据题意：

```text
所有直接下级都不能来
```

所以：

```text
yes = x.happy + 所有下级的 no 之和
```

### 3. 如果当前员工不来
那每个直接下级都自由：

- 可以来
- 也可以不来

于是对每个下级，只需要取它的更优解：

```text
no = Σ max(child.yes, child.no)
```

### 4. `Info` 设计
所以每个节点只需要返回：

```text
yes
no
```

定义：

```java
Info process(Employee x)
```

返回：

- `x` 来时的最大快乐值
- `x` 不来时的最大快乐值

伪代码：

```java
Info process(Employee x) {
    if (x == null) {
        return new Info(0, 0);
    }
    int yes = x.happy;
    int no = 0;
    for (Employee next : x.nexts) {
        Info nextInfo = process(next);
        yes += nextInfo.no;
        no += Math.max(nextInfo.yes, nextInfo.no);
    }
    return new Info(yes, no);
}
```

最终答案：

```java
Info all = process(boss);
return Math.max(all.yes, all.no);
```

## 典型例子
还是看这棵树：

```text
        A(10)
      /   |   \
   B(5)  C(20) D(1)
          |
         E(30)
```

先看最底层：

- `B`：`yes=5, no=0`
- `D`：`yes=1, no=0`
- `E`：`yes=30, no=0`

再看 `C`：

- `yes(C) = 20 + no(E) = 20`
- `no(C) = max(30, 0) = 30`

最后看 `A`：

- `yes(A) = 10 + no(B) + no(C) + no(D) = 10 + 0 + 30 + 0 = 40`
- `no(A) = max(5,0) + max(20,30) + max(1,0) = 5 + 30 + 1 = 36`

所以最终答案：

```text
40
```

也就是：

- 让 `A` 来
- `B/C/D` 不来
- `E` 来

## 复杂度
- 时间复杂度：`O(N)`
- 空间复杂度：`O(H)`

其中：

- `N` 是员工节点总数
- `H` 是树高

## 易错点
- 约束的是“直接下级不能来”，不是整棵子树都不能来。
- `yes` 状态下只能加所有孩子的 `no`。
- `no` 状态下是对每个孩子单独取 `max(yes, no)`，不是统一强制全来或全不来。
- 这是多叉树题，但思路和二叉树递归套路完全一致。

## 代码 / 伪代码
仓库里目前没有这一题对应的现成实现文件，这里保留标准伪代码。

```java
class Info {
    int yes;
    int no;

    Info(int yes, int no) {
        this.yes = yes;
        this.no = no;
    }
}

Info process(Employee x) {
    if (x == null) {
        return new Info(0, 0);
    }
    int yes = x.happy;
    int no = 0;
    for (Employee next : x.nexts) {
        Info nextInfo = process(next);
        yes += nextInfo.no;
        no += Math.max(nextInfo.yes, nextInfo.no);
    }
    return new Info(yes, no);
}
```

## 记忆点
- 这题只有两个状态：来 / 不来。
- `yes = 自己快乐值 + 所有孩子的 no`。
- `no = 所有孩子各自 max(yes, no)` 之和。
- 多叉树树形 DP，本质和二叉树套路完全一样。
