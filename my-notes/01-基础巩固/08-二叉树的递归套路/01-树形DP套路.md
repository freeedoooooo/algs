## 一句话结论
二叉树递归套路的本质，是把“整棵树的问题”转成“每棵子树都返回一份标准信息 `Info`”，再由父节点统一整合左右子树的信息。

一旦你学会先想“左树要什么、右树要什么、我怎么合并”，绝大多数树形 DP 题都会变成同一类题。

此套路可以解决95%以上二叉树问题。

## 核心知识点
- 递归套路不是固定模板代码，而是一套分析框架。
- 关键不是“怎么递归”，而是“每棵子树应该返回什么信息”。
- 父节点的答案，往往只依赖于：左子树 `Info`、右子树 `Info`、当前节点值。
- 所有子树都返回同一种 `Info`，递归代码就会非常统一。
- 平衡树、最大距离、最大 BST 子树、派对快乐值，本质都属于这套树形 DP 思维。

1. 假设以 `X` 节点为头，假设可以向 `X` 左树和 `X` 右树要任何信息

3. 列出所有可能性后，确定到底需要向左树和右树要什么样的信息
4. 把左树信息和右树信息求全集，就是任何一棵子树都需要返回的信息集合 `S`
5. 递归函数都返回 `S`，每一棵子树都这么要求
6. 写代码，在代码中考虑如何把左树的信息和右树信息整合出整棵树的信息

这就是整章最核心的方法论。

这篇虽然不是一道单独刷题，但它在回答一个非常具体的问题：

- 面对一棵树上的复杂问题
- 为什么很多题都能统一写成递归
- 递归时到底该让每棵子树返回什么

所以它的完整“题意”可以理解为：

```text
给定一类二叉树问题
如何把“整棵树求答案”
改写成“每棵子树返回一份 Info”
再由父节点统一合并
```

## 图解
假设有一棵树：

```text
        X
      /   \
     L     R
```

递归套路并不是直接问：

```text
X 这棵树答案是什么
```

而是先问：

```text
如果左树能告诉我一些信息
如果右树也能告诉我一些信息
我能不能把 X 这棵树的答案算出来
```

比如对不同题，左右子树可能需要返回：

### 平衡二叉树

```text
左树：是否平衡、高度
右树：是否平衡、高度
```

### 最大距离

```text
左树：高度、最大距离
右树：高度、最大距离
```

### 最大 BST 子树

```text
左树：最大值、最小值、最大BST大小、最大BST头
右树：最大值、最小值、最大BST大小、最大BST头
```

这说明：

```text
题目不同
Info 不同
套路不变
```

## 解题思路
### 1. 第一步：先假设能从左右子树拿信息
这一步很关键，也最容易被忽略。

分析树题时，不要一上来就写递归代码，而是先大胆假设：

```text
左子树已经帮我算好了
右子树也已经帮我算好了
```

那我在当前节点这里，还差什么才能得到整棵树答案？

### 2. 第二步：枚举当前节点的答案可能性
这是整套方法里最重要的一步。

以某个节点 `X` 为头时，答案往往只有有限几种情况。

比如：

- 平衡树：要么左平衡右平衡且高度差不超过 1，要么不平衡
- 最大距离：要么在左树里，要么在右树里，要么穿过当前节点
- 最大 BST 子树：要么在左树里，要么在右树里，要么整棵树本身就是 BST

只要把这几种情况列全，题基本就做出来一半了。

### 3. 第三步：反推出子树必须提供哪些信息
一旦当前节点的答案可能性列出来，就可以反推：

```text
为了在 X 这里做判断
左树和右树至少得告诉我什么
```

这就是 `Info` 的来源。

例如最大距离题里，为什么 `Info` 要有：

- `height`
- `maxDistance`

因为：

- 不知道左右高度，就算不出“穿过根”的距离
- 不知道左右最大距离，就无法比较“答案在左 / 在右 / 过根”

### 4. 第四步：统一让每棵子树返回同一种 `Info`
这一步是代码层面的统一。

既然任何一个节点都可能被当作“当前节点”分析，那么：

- 每棵子树都必须返回相同结构的 `Info`
- 递归函数签名才可以统一

形式上通常是：

```java
Info process(Node x)
```

含义是：

```text
返回以 x 为头的整棵子树所需全部信息
```

### 5. 第五步：写代码时只做三件事
标准写法基本都一样：

1. 递归拿左树 `Info`
2. 递归拿右树 `Info`
3. 根据左右 `Info` 和当前节点值，构造自己的 `Info`

伪代码骨架：

```java
Info process(Node x) {
    if (x == null) {
        return baseInfo;
    }
    Info leftInfo = process(x.left);
    Info rightInfo = process(x.right);

    // 根据 leftInfo、rightInfo、x.value
    // 组装当前节点的信息
    return new Info(...);
}
```

## 典型例子
以“判断平衡二叉树”为例。

如果当前节点是 `X`，那么只要知道：

- 左树是否平衡、左树高度
- 右树是否平衡、右树高度

就能判断：

```text
X 这棵树是否平衡
```

因为条件只有：

```text
左树平衡
右树平衡
|左高 - 右高| <= 1
```

于是 `Info` 就自然是：

```text
isBalanced
height
```

这就是递归套路最典型的形成过程。

## 复杂度
- 时间复杂度：通常是 `O(N)`
- 空间复杂度：通常是 `O(H)`

其中：

- `N` 是节点数
- `H` 是树高

因为这类题大多都只对每个节点处理一次。

## 易错点
- 递归套路不是上来就套 `Info`，而是先列“当前节点答案的可能性”。
- 不要把 `Info` 设计成一堆和当前题无关的字段，够用就行。
- `null` 节点返回什么基础信息，要根据题目 carefully 设计。
- 很多树题其实不是“遍历顺序题”，而是“子树信息整合题”。
- 这套方法不仅适用于二叉树，稍作修改也能用到多叉树。

```java
class Info {
    // 题目所需信息字段
}

Info process(Node x) {
    if (x == null) {
        return baseInfo;
    }
    Info leftInfo = process(x.left);
    Info rightInfo = process(x.right);

    // 讨论以 x 为头时的答案可能性
    // 反推出当前节点要返回的 Info
    return new Info(...);
}
```

## 记忆点
- 先假设左右子树能给你任何信息。
- 先列当前节点答案的所有可能性，再反推需要什么 `Info`。
- 每棵子树统一返回同一种 `Info`。
- 这套方法本质就是树形 DP。
