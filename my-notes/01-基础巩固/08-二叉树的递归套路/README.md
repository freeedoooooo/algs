## 章节导读
这一章不是在继续补遍历，而是在升级二叉树问题的分析方式。

如果上一章讲的是：

```text
树怎么遍历
```

那么这一章讲的是：

```text
遍历过程中
每棵子树应该返回什么信息
```

这就是所谓的“二叉树递归套路”，也常被叫作树形 DP。

整章可以概括为一条主线：

```text
先列当前节点答案的可能性
再反推左右子树需要返回什么 Info
最后统一递归返回 Info
```

## 本章结论
- 二叉树递归套路的核心，不是写递归，而是设计 `Info`。
- 当前节点的答案，通常只依赖左树 `Info`、右树 `Info` 和当前节点值。
- 平衡树、最大距离、最大 BST 子树，都是同一个分析套路。
- 这套方法不仅适用于二叉树，稍作修改也适用于多叉树。
- 一旦会用这套方法，很多树题都会从“不会下手”变成“只是 `Info` 设计不同”。

## 图片转写 / 章节笔记还原
原始章节核心信息可以整理为：

### 1. 这套递归方法的定位
- 可以解决面试中绝大多数的二叉树问题，尤其是树型 DP 问题。
- 本质是利用递归遍历树时，天然可以自底向上收集子树信息。

### 2. 标准分析步骤
1. 假设以 `X` 节点为头，可以向左树和右树要任何信息
2. 在这个假设下，讨论以 `X` 为头时答案有哪些可能性
3. 列完可能性后，反推到底需要左右子树提供哪些信息
4. 把这些信息打包成统一的 `Info`
5. 所有子树都返回同一种 `Info`
6. 代码里只做：拿左右 `Info`，整合出当前 `Info`

这一章 5 道题，其实都在反复练这 6 步。

## 记忆点
- 先列“当前节点答案的可能性”，再设计 `Info`。
- `Info` 不是越多越好，而是够当前节点判断就行。
- 每棵子树统一返回同一种 `Info`，这是代码统一的关键。
- 这章本质就是树形 DP 入门章。
