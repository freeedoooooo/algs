# 堆-移动距离小于K的排序

## 一句话结论
如果一个数组几乎有序，并且每个元素离它最终位置的距离都不超过 `K`，那么没必要做完整的 `O(N log N)` 排序。

更合适的做法是：

- 始终维护一个大小不超过 `K+1` 的小根堆
- 每次从这 `K+1` 个候选里取最小值

这样总时间复杂度可以做到 `O(N log K)`。

## 图片转写 / 题意还原
这道题描述的是一种“几乎有序”的特殊数组：

- 如果把它排好序
- 每个元素和自己最终位置之间的距离都不超过 `K`
- 并且 `K` 相对数组长度 `N` 来说比较小

要求是：

```text
利用这个“每个元素最多只会移动 K 格”的信息，
设计一个比普通排序更高效的方法。
```

## 先抓住最关键的一句话
假设现在要确定排好序后第 `i` 个位置应该放谁。

因为任何元素最多只会移动 `K` 格，所以：

```text
真正应该放在位置 i 的那个数，
原来只可能出现在 [i ... i+K] 这段范围里。
```

也就是说，当前位置的正确答案，根本不需要去整个数组里找，最多只需要在 `K+1` 个元素里找。

这就是整道题最核心的突破口。

## 为什么一定是 `K+1` 个元素
很多人第一次看这题，最容易卡住的就是这一点：

```text
为什么不是 K 个，而是 K+1 个？
```

原因很简单。

如果某个元素最终应该落在位置 `i`，而它最多只能移动 `K` 格，那么它最远也只能从 `i+K` 的位置来到 `i`。

所以它可能出现的原始范围是：

```text
i, i+1, i+2, ..., i+K
```

这里一共是：

```text
K+1 个位置
```

可以把它画成：

```text
要填结果位置 i

候选范围：
i      i+1    i+2          ...    i+K
|------|------|--------------------|
        真正答案只可能藏在这里
```

因此：

- 第 `0` 个位置的答案，只可能在原数组的 `[0 ... K]`
- 第 `1` 个位置的答案，只可能在原数组的 `[1 ... K+1]`
- 第 `2` 个位置的答案，只可能在原数组的 `[2 ... K+2]`

本质上，这就是一个不断向右滑动的候选窗口。

## 为什么用小根堆最合适
既然每一步都只需要在 `K+1` 个候选里找最小值，那么问题就变成了：

```text
如何一边滑动窗口，
一边快速拿到窗口中的最小值？
```

这正是小根堆最擅长的事：

- 插入一个新元素：`O(log K)`
- 弹出当前最小值：`O(log K)`
- 堆顶就是当前候选中的最小值

所以这题的自然解法就是：

```text
用小根堆维护“当前可能成为答案的 K+1 个候选”
```

## 整体步骤
### 1. 先把前 `K+1` 个元素放入小根堆
因为排序后第 `0` 个位置的答案，一定在这批元素里。

### 2. 弹出堆顶，写回当前结果位置
堆顶就是这批候选里的最小值，也就是当前最该放出去的数。

### 3. 把下一个新元素加入堆中
窗口向右滑动一格后，新的候选范围里会多一个元素，把它补进堆里。

### 4. 重复“弹出最小值 + 补入新元素”
这样就能一格一格地把整个数组排好。

### 5. 当原数组扫描结束后，把堆里剩下的元素依次弹出
此时已经没有新元素可补，但堆里剩余的元素本身就是最后那部分的有序答案。

## 图解：窗口是怎么滑动的
假设数组是：

```text
[3, 1, 2, 6, 4, 5, 7]
```

并且：

```text
K = 2
```

这说明每个元素离自己最终位置都不超过 2。

排好序后应该是：

```text
[1, 2, 3, 4, 5, 6, 7]
```

### 第 0 个位置
第 `0` 个位置的答案，只能在 `[0 ... 2]` 中找，也就是：

```text
[3, 1, 2]
```

放入小根堆后，堆顶是 `1`，所以：

```text
结果第 0 位 = 1
```

### 第 1 个位置
前一步弹出 `1` 之后，堆里还剩：

```text
[3, 2]
```

这时窗口向右滑一格，把新元素 `6` 补进来：

```text
[3, 2, 6]
```

最小值是 `2`，所以：

```text
结果第 1 位 = 2
```

### 第 2 个位置
再往右滑一格。

此时弹出 `2` 之后，堆里还剩：

```text
[3, 6]
```

再加入新元素 `4`。

堆中候选可以理解成：

```text
[3, 6, 4]
```

最小值是 `3`，所以：

```text
结果第 2 位 = 3
```

接下来同理。

## 完整流程表
还是以上面的数组为例：

```text
arr = [3, 1, 2, 6, 4, 5, 7]
K = 2
```

下面这张表，按“更容易理解”的流程来写：

| 当前要填的位置 | 当前新加入的元素 | 堆中的候选（示意） | 弹出的最小值 | 已确定结果 |
| --- | --- | --- | --- | --- |
| 先初始化 | 加入 `3,1,2` | `[1,2,3]` | - | `[]` |
| `0` | - | `[1,2,3]` | `1` | `[1]` |
| `1` | 加入 `6` | `[2,3,6]` | `2` | `[1,2]` |
| `2` | 加入 `4` | `[3,4,6]` | `3` | `[1,2,3]` |
| `3` | 加入 `5` | `[4,5,6]` | `4` | `[1,2,3,4]` |
| `4` | 加入 `7` | `[5,6,7]` | `5` | `[1,2,3,4,5]` |
| 扫描结束 | 无新元素 | `[6,7]` | `6` | `[1,2,3,4,5,6]` |
| 收尾 | 无新元素 | `[7]` | `7` | `[1,2,3,4,5,6,7]` |

可以把这个过程再压缩成一句话：

```text
先准备前 K+1 个候选，
之后每往右走一步，就输出一个最小值，再补进一个新元素。
```

## 代码实现时的一个小细节
从“讲思路”的角度，最顺的写法通常是：

```text
先弹出当前最小值
再补入下一个新元素
```

但仓库里的实现是另一种等价写法：

```text
先补入下一个新元素
再弹出最小值
```

为什么也对？

因为这个新补进来的元素，原始位置更靠后，它不可能跨越超过 `K` 格跑到当前待填位置之前。

也就是说：

```text
它可以先进入堆，
但不会错误地抢走当前这一位本该输出的答案。
```

所以两种写法本质是一样的，只是叙述顺序不同。

## 解题思路总结
如果只记四句话，记这四句就够了：

1. 每个元素最多移动 `K` 格。
2. 所以当前位置的答案，只可能在后面的 `K+1` 个候选里。
3. 用小根堆维护这批候选，堆顶就是当前答案。
4. 每次弹出一个最小值，再把窗口向右推进。

## 复杂度
- 时间复杂度：`O(N log K)`
- 空间复杂度：`O(K)`

原因是：

- 每个元素最多进堆一次、出堆一次
- 每次进堆和出堆的代价都是 `O(log K)`
- 堆的大小始终不超过 `K+1`

## 易错点
- 不是看 `K` 个元素，而是看 `K+1` 个元素。
- 这里应该用小根堆，不是大根堆。
- 题目成立的前提是“每个元素离最终位置都不超过 `K`”，不是“看起来差不多有序”。
- 当 `K = 0` 时，数组本来就已经有序。
- 复杂度是 `O(N log K)`，不是 `O(N log N)`。

## 代码 / 伪代码
仓库里的对应实现就在：

- [HeapSort.java](../../../solutions-java/practice/algs01_sort/HeapSort.java)

对应方法：

```java
public void sortArrDistanceLessK(int[] arr, int k)
```

核心逻辑：

```java
PriorityQueue<Integer> heap = new PriorityQueue<>();
int index = 0;
for (; index <= Math.min(k, arr.length - 1); index++) {
    heap.add(arr[index]);
}
int i = 0;
for (; index < arr.length; i++, index++) {
    heap.add(arr[index]);
    arr[i] = heap.poll();
}
while (!heap.isEmpty()) {
    arr[i++] = heap.poll();
}
```

如果改写成更容易直接理解的伪代码，可以写成：

```java
heap = 小根堆
先加入前 K+1 个元素
write = 0

for (每个后续元素 x) {
    arr[write++] = heap.poll();
    heap.add(x);
}

while (heap 非空) {
    arr[write++] = heap.poll();
}
```

## 记忆点
- 位移不超过 `K`，就只看 `K+1` 个候选。
- 当前位置的答案，本质上是在一个滑动窗口里找最小值。
- “滑动窗口最小值”这里最顺手的结构就是小根堆。
- 这类特殊排序题，关键不是硬记写法，而是先抓住候选范围为什么只有 `K+1` 个。
