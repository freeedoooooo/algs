## 一句话结论
Kruskal 算法是从“边”的角度构造最小生成树：把所有边按权值从小到大考虑，能接进来且不成环就要，成环就丢弃。
它和并查集天然搭配，因为“会不会成环”本质就是看一条边两端点当前是否已经连通。

## 核心知识点
- 适用对象是无向图。
- 先按边权从小到大排序。
- 当前边若连接了两个不同集合，就选它。
- 当前边若连接了同一集合，就会成环，不能选。
- 并查集用于快速判断连通性。

## 图片转写 / 题意还原
原图内容为：

1. 总是从权值最小的边开始考虑，依次考察权值依次变大的边
2. 当前边要么进入最小生成树集合，要么丢弃
3. 如果当前边进入集合中不会形成环，就要当前边
4. 如果会形成环，就不要当前边
5. 考察完所有边之后，最小生成树集合就得到了

原始笔记还补充了一句：

- 这是针对无向图的算法，常借助并查集实现

## 图解
假设边为：

```text
A-B 1
B-C 2
A-C 3
C-D 4
```

排序后依次看：

- 选 `A-B`
- 选 `B-C`
- `A-C` 会形成环，丢弃
- 选 `C-D`

最终得到生成树。

## 解题思路
### 1. 为什么按边权从小到大
因为最小生成树的目标就是总边权最小，所以优先考虑更便宜的边。

### 2. 为什么并查集能判断成环
如果一条边两端点已经在同一集合里，再把它接进来就一定形成环。

## 复杂度
- 时间复杂度：`O(E log E)`
- 空间复杂度：`O(V)`

## 典型例子
如果当前并查集里：

```text
{A, B, C}
{D}
```

再考虑边 `B-D`：

- 两端不在同一集合
- 说明接上后不会成环
- 就应该选它

## 易错点
- Kruskal 是无向图算法。
- 判断成环不要真的去找环，直接看并查集是否同集合。
- 选中的边数达到 `V-1` 后，就已经得到生成树。

## 代码 / 伪代码
课程标准伪代码：

```java
Set<Edge> kruskalMST(Graph graph) {
    UnionFind<Node> uf = new UnionFind<>(graph.nodes.values());
    PriorityQueue<Edge> heap = new PriorityQueue<>((a, b) -> a.weight - b.weight);
    heap.addAll(graph.edges);
    Set<Edge> result = new HashSet<>();
    while (!heap.isEmpty()) {
        Edge edge = heap.poll();
        if (!uf.isSameSet(edge.from, edge.to)) {
            result.add(edge);
            uf.union(edge.from, edge.to);
        }
    }
    return result;
}
```

## 记忆点
- Kruskal 看边。
- 小边优先。
- 会成环就丢。
- 并查集判断是否成环。
