## 一句话结论
图的 BFS 本质上就是“从源点开始，按距离一层一层往外扩”。实现工具是队列，再配一个 `visited` 集合防止图里有环时死循环。

## 核心知识点
- BFS 使用队列。
- 处理顺序是“先进先出”。
- 每弹出一个点，把它所有未进队的邻接点进队。
- 图和树不同，图里必须记录 `visited`。

## 图片转写 / 题意还原
原始章节配图对 BFS 的总结是：

1. 利用队列实现
2. 从源节点开始依次按照宽度进队列，然后弹出
3. 每弹出一个点，把该节点所有没有进过队列的邻接点放入队列
4. 直到队列变空

## 图解
如果有图：

```text
1 -> 2, 3
2 -> 4
3 -> 5
```

从 `1` 开始 BFS：

```text
队列: [1]
弹出 1，加入 2、3
队列: [2, 3]
弹出 2，加入 4
队列: [3, 4]
弹出 3，加入 5
队列: [4, 5]
...
```

遍历顺序：

```text
1, 2, 3, 4, 5
```

## 解题思路
### 1. 初始化
- 源点入队
- 源点加入 `visited`

### 2. 主循环
当队列不空时：

- 弹出队头节点 `cur`
- 遍历 `cur.nexts`
- 没访问过的邻接点全部入队并打标记

## 复杂度
- 时间复杂度：`O(V + E)`
- 空间复杂度：`O(V)`

## 典型例子
在无权图里，BFS 经常顺手就能解决：

- 最少步数
- 最短边数路径
- 层级扩散

因为 BFS 的天然顺序就是按“离起点的层数”扩展。

## 易错点
- 图的 BFS 一定要有 `visited`，否则有环就会反复入队。
- 标记访问最好在“入队时”做，而不是“出队时”做。
- BFS 遍历顺序和邻接点存放顺序有关，但分层性质不变。

## 代码 / 伪代码
课程标准伪代码：

```java
void bfs(Node start) {
    Queue<Node> queue = new LinkedList<>();
    Set<Node> visited = new HashSet<>();
    queue.add(start);
    visited.add(start);
    while (!queue.isEmpty()) {
        Node cur = queue.poll();
        for (Node next : cur.nexts) {
            if (!visited.contains(next)) {
                visited.add(next);
                queue.add(next);
            }
        }
    }
}
```

## 记忆点
- BFS 用队列。
- 一层一层向外扩。
- 图里必须防环。
- 入队就标记。
