## 一句话结论
图的 DFS 是“先沿一条路尽量走到底，走不动再回退”。实现上可以用递归，也可以显式用栈模拟。
课程里更强调非递归写法，因为它能更清楚地看见“回退现场”的维护过程。

## 核心知识点
- DFS 使用栈或递归。
- 图里同样要用 `visited` 防环。
- 非递归 DFS 的关键是：遇到一个新邻居时，要把当前点重新压回栈，保留回退现场。
- DFS 常用于连通性、路径搜索、回溯问题。

## 图片转写 / 题意还原
原始章节配图把 DFS 总结为：

1. 利用栈实现
2. 从源节点开始把节点按照深度放入栈，然后弹出
3. 每弹出一个点，把该节点下一个没有进过栈的邻接点放入栈
4. 直到栈变空

把题目补完整后，它讲的是：

- 给定图中的一个起始节点 `start`
- 按照“能继续往深处走就先往深处走”的顺序遍历整张图
- 每个节点只访问一次
- 输出访问顺序，或者把遍历框架作为后续搜索的基础模板

这里和树的 DFS 不同，图里默认可能有环，所以必须显式防止重复访问。

## 图解
图：

```text
1 -> 2, 3
2 -> 4
4 -> 5
```

从 `1` 开始：

```text
先到 2
再到 4
再到 5
5 走不动，回到 4
4 走不动，回到 2
2 走不动，回到 1
再去 3
```

这就是深度优先。

## 解题思路
### 1. 递归版
进入一个点：

- 先标记访问
- 再递归所有没访问过的邻居

### 2. 栈版
栈版为了保留“回到当前点继续找下一个邻居”的现场，常用写法是：

- 弹出当前点 `cur`
- 遇到第一个未访问邻居 `next`
- 把 `cur` 压回栈
- 再把 `next` 压栈并标记
- `break`，让流程继续向更深处走

## 复杂度
- 时间复杂度：`O(V + E)`
- 空间复杂度：`O(V)`

## 典型例子
DFS 特别适合：

- 枚举所有路径
- 搜索某种可行解
- 回溯和状态树遍历

所以它既是图算法模板，也是回溯类题目的底层骨架。

## 易错点
- 图的 DFS 也必须防止重复访问。
- 非递归写法里，把当前点压回栈是关键，否则会丢失后续分支。
- 递归版代码更短，但非递归版更能看清系统栈在做什么。

## 代码 / 伪代码
课程标准伪代码：

```java
void dfs(Node start) {
    Stack<Node> stack = new Stack<>();
    Set<Node> visited = new HashSet<>();
    stack.push(start);
    visited.add(start);
    while (!stack.isEmpty()) {
        Node cur = stack.pop();
        for (Node next : cur.nexts) {
            if (!visited.contains(next)) {
                stack.push(cur);
                stack.push(next);
                visited.add(next);
                break;
            }
        }
    }
}
```

## 记忆点
- DFS 用栈。
- 先走到底，再回退。
- 图里必须防环。
- 非递归版别忘了把 `cur` 压回去。
