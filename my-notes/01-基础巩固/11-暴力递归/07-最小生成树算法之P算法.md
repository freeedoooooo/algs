## 一句话结论
Prim 算法是从“点”的角度逐步扩展最小生成树：先随便选一个起点，把它能解锁的边放进小根堆，然后每次取一条当前最小且能把新点拉进来的边。
它和 Kruskal 都是在求最小生成树，但思维视角完全不同。

## 核心知识点
- 适用于无向图。
- Prim 的扩展单位是“当前生成树解锁出来的边”。
- 只要一条最小边能带来新点，就把它选入答案。
- 常配合小根堆和已访问点集合实现。

## 图片转写 / 题意还原
原始笔记中虽然只记录了题目名称，但这题在课程里的标准题意很明确：介绍 Prim 算法如何在带权无向图中构造最小生成树。

Prim 算法的核心可以概括为：

- 从任意一点出发
- 解锁从已访问点出发的所有边
- 每次取权值最小的可行边
- 如果这条边能带来一个新点，就把它接入最小生成树

## 图解
假设从 `A` 开始：

```text
A-B 3
A-C 1
C-D 2
B-D 5
```

过程：

- 起点 `A`
- 解锁 `A-B(3)`、`A-C(1)`
- 先取 `A-C(1)`，加入 `C`
- 再解锁 `C-D(2)`
- 此时候选边有 `A-B(3)`、`C-D(2)`
- 先取 `C-D(2)`，加入 `D`
- 再取 `A-B(3)`，加入 `B`

## 解题思路
### 1. 起点随便选
最小生成树不依赖固定起点。

### 2. 用小根堆维护“当前解锁的边”
谁最便宜先看谁。

### 3. 只接纳能带来新点的边
如果边的终点已经访问过，再选它就会形成环。

## 复杂度
- 时间复杂度：`O(E log E)` 或 `O(E log V)`
- 空间复杂度：`O(E)`

## 典型例子
Prim 特别适合你已经有：

- 一个起始区域
- 希望不断往外以最小代价扩张

它更像“已建区域不断向外铺开”的过程。

## 易错点
- Prim 和 Kruskal 都解最小生成树，但 Prim 是从点角度扩张。
- 不是堆里最小边就一定选，还得看它能不能带来新点。
- 图不连通时，要按连通分量分别启动或理解为最小生成森林。

## 代码 / 伪代码
课程标准伪代码：

```java
Set<Edge> primMST(Graph graph) {
    PriorityQueue<Edge> heap = new PriorityQueue<>((a, b) -> a.weight - b.weight);
    Set<Node> visited = new HashSet<>();
    Set<Edge> result = new HashSet<>();

    for (Node node : graph.nodes.values()) {
        if (!visited.contains(node)) {
            visited.add(node);
            for (Edge edge : node.edges) {
                heap.add(edge);
            }
            while (!heap.isEmpty()) {
                Edge edge = heap.poll();
                Node toNode = edge.to;
                if (!visited.contains(toNode)) {
                    visited.add(toNode);
                    result.add(edge);
                    for (Edge nextEdge : toNode.edges) {
                        heap.add(nextEdge);
                    }
                }
            }
        }
    }
    return result;
}
```

- 从任意点开始扩。
- 解锁边放小根堆。
- 只接纳能带来新点的边。
