## 一句话结论
项目选择问题的核心不是“选利润最大的项目”这么简单，而是两层贪心一起配合：

- 先在“当前做得起”的项目里筛选
- 再从这些可做项目里挑利润最高的

所以标准做法是：

```text
小根堆管门槛
大根堆管收益
```

## 核心知识点
- 每个项目都有花费 `cost` 和利润 `profit`。
- 你有初始资金 `M`，最多只能做 `K` 个项目。
- 做完一个项目后，立刻获得利润，资金会增长。
- 后续能做哪些项目，取决于当前资金是否达到门槛。
- 标准解法是“小根堆 + 大根堆”。

## 题意说明
给定：

- 数组 `costs`
- 数组 `profits`
- 正整数 `K`
- 初始资金 `M`

其中：

- `costs[i]` 表示做第 `i` 个项目至少需要多少本金
- `profits[i]` 表示做完第 `i` 个项目能净赚多少钱

规则是：

- 最多只能做 `K` 个项目
- 只有当前资金 `M` 足够时，项目才允许开做
- 做完后立刻拿到利润，资金增加

要求返回：

```text
最多做 K 个项目后，最后手里最多有多少钱
```

你可以把这题想成：

```text
你手里有一笔启动资金
每做完一个项目，钱包会变厚
钱包一变厚，下一批更贵的项目才会被解锁
```

所以这不是静态选择题，而是：

```text
资金会变化
候选项目集合也会跟着变化
```

## 先把这题卡点看清楚
这题最容易一开始想错的地方是：

```text
为什么不能每次都直接选利润最大的项目？
```

因为你不是“想做谁就能做谁”。

这题有一个非常关键的限制：

```text
当前资金不够
项目就根本没资格进入候选池
```

所以整个过程其实分成两步：

1. 先看哪些项目现在做得起
2. 再在这些做得起的项目里选最赚钱的

一旦把这个顺序想清楚，这题就顺了。

## 图解
先看一个最小例子：

```text
costs   = [5, 1, 3]
profits = [4, 2, 5]
K = 2
M = 1
```

开始时资金只有 `1`，所以：

- 成本 `5` 的项目做不起
- 成本 `3` 的项目做不起
- 只有成本 `1` 的项目能做

第一轮只能先做它：

```text
资金 1 -> 做成本 1、利润 2 的项目 -> 资金变成 3
```

现在资金变成 `3` 后：

- 成本 `3` 的项目解锁
- 成本 `5` 的项目还是做不起

第二轮在当前可做项目里，选利润最大的：

```text
做成本 3、利润 5 的项目
资金 3 -> 8
```

最终答案是 `8`。

## 解题思路
### 1. 为什么一个堆不够
这题同时有两个维度：

- 谁现在做得起
- 在做得起的项目里，谁最赚钱

所以只用一个堆很难同时兼顾“门槛”和“收益”。

### 2. 两个堆分别干什么
小根堆按 `cost` 排：

```text
用来快速找到所有当前资金能解锁的项目
```

大根堆按 `profit` 排：

```text
用来在已解锁项目里，快速挑出最赚钱的
```

你可以把它们记成：

```text
costHeap   = 门口保安
profitHeap = 候选名单里的利润排行榜
```

也就是说：

- 小根堆负责判断“谁有资格进场”
- 大根堆负责判断“进场后先选谁”

### 3. 主流程怎么走
每一轮都做同样的事：

1. 把所有 `cost <= 当前资金` 的项目，从小根堆转移到大根堆
2. 如果大根堆为空，说明已经没有项目能做，直接结束
3. 否则弹出利润最高的项目去做
4. 更新资金

这就是典型的“先解锁，再挑最优收益”。

## 过程示例
下面换一个更完整一点的例子，看“解锁 -> 选择 -> 资金上涨 -> 再解锁”的完整链条。

```text
costs   = [5, 1, 3, 6, 2]
profits = [4, 2, 5, 7, 3]
K = 3
M = 2
```

初始：

```text
小根堆里有全部项目，按成本排序
大根堆为空
当前资金 = 2
```

第一轮：

- 把所有成本 `<= 2` 的项目从小根堆移到大根堆
- 可做项目变成：

```text
(cost=1, profit=2)
(cost=2, profit=3)
```

- 这时不应该随便选，而是选利润更高的 `(2,3)`
- 做完后，资金从 `2` 变成 `5`

第二轮：

- 现在资金变成 `5`，于是更多项目被解锁：

```text
(cost=1, profit=2)
(cost=3, profit=5)
(cost=5, profit=4)
```

- 当前利润最高的是 `(3,5)`
- 做完后，资金从 `5` 变成 `10`

第三轮：

- 现在成本 `6` 的项目也被解锁
- 候选项目大致变成：

```text
(cost=1, profit=2)
(cost=5, profit=4)
(cost=6, profit=7)
```

- 选利润最高的 `(6,7)`
- 做完后，资金从 `10` 变成 `17`

做满 `K = 3` 轮，结束。

最终答案是：

```text
17
```

这个例子最重要的是让你看到：

```text
不是一开始就能看到最终最优路径
而是每做完一步，新的候选项目才会出现
```

## 复杂度
- 时间复杂度：`O(N log N + K log N)`
- 空间复杂度：`O(N)`

## 易错点
- 不是全局直接选利润最大项目，得先看当前做不做得起。
- 不是每轮都重新遍历全部项目，应该用堆维护未解锁和已解锁项目。
- 利润是做完后立刻加到资金里的，所以会影响下一轮可选项目集合。
- 如果某一轮大根堆为空，说明当前资金已经无法继续解锁任何项目，应立即结束。

## 代码 / 伪代码
下面这版代码更适合直接对应前面的思路去读：

```java
int findMaximizedCapital(int K, int M, int[] costs, int[] profits) {
    if (costs == null || profits == null || costs.length != profits.length) {
        return M;
    }

    PriorityQueue<Project> costHeap = new PriorityQueue<>((a, b) -> a.cost - b.cost);
    PriorityQueue<Project> profitHeap = new PriorityQueue<>((a, b) -> b.profit - a.profit);

    for (int i = 0; i < costs.length; i++) {
        costHeap.add(new Project(costs[i], profits[i]));
    }

    for (int i = 0; i < K; i++) {
        // 先把当前资金能做的项目全部解锁
        while (!costHeap.isEmpty() && costHeap.peek().cost <= M) {
            profitHeap.add(costHeap.poll());
        }

        // 当前没有任何可做项目，说明流程提前结束
        if (profitHeap.isEmpty()) {
            return M;
        }

        // 在已解锁项目里，优先做利润最高的
        Project best = profitHeap.poll();
        M += best.profit;
    }
    return M;
}
```

如果你把这段代码压成一句流程话术，就是：

```text
先解锁所有做得起的项目
再从已解锁项目里挑最赚钱的那个
```

## 记忆点
- 小根堆负责“谁能做”，大根堆负责“谁最赚”。
- 每轮先解锁，再选利润最高的项目。
- 资金会动态增长，所以可选项目集合也会动态变化。
- 这是“堆 + 贪心”非常典型的一道题。
