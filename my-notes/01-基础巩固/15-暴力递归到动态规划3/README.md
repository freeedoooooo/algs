## 章节导读
这一章是“把前面已经会写的递归题，系统改成 DP”的练习章。题型覆盖：

- 从左往右模型：背包、数字转字母
- 范围模型：纸牌博弈
- 完全背包计数：货币组成方式

## 本章结论
- 递归改 DP，第一步永远是找状态。
- 状态参数的所有组合数量，就是 DP 表大小。
- 先做记忆化搜索，再写经典表格，通常更稳。
- 同一题往往不止一种 DP 写法，取决于状态设计和枚举方式。

## 图片转写 / 章节笔记还原
原始笔记这一章的核心意思可以压缩为：

- 先找到暴力递归
- 再分析有没有重复解
- 有重复解时，用参数做缓存就是记忆化搜索
- 进一步精细组织缓存结构，就是经典动态规划

另外，本章最后一道题明确给了币值组成问题：

- 给定正数数组，表示不同面值货币，每种面值可用任意张
- 给定 `aim`
- 返回组成 `aim` 的方法数

## 记忆点
- 先递归，后 DP。
- 状态组合数量 = 表大小。
- 从左往右常是一维 / 二维位置表。
- 范围模型常是区间表。
