## 一句话结论
这题是经典“无限张货币凑金额”的方法数问题，属于完全背包计数模型。暴力递归会枚举每种面值拿几张，DP 则把这种枚举系统化。

## 核心知识点
- `arr` 中面值互不重复，且每种可用任意张。
- 目标是组成 `aim` 的方法数，不是最少张数。
- 状态常用：来到第 `index` 种货币，剩余金额 `rest`。

## 图片转写 / 题意还原
原图内容为：

- 给定数组 `arr`，其中所有值都为正数且不重复
- 每个值代表一种面值货币，每种货币可以使用任意张
- 给定整数 `aim`
- 求组成 `aim` 的方法数

把题目补完整后，它的要求是：

- `arr` 中每个数都是一种货币面值
- 每种面值的货币数量无限
- 可以选若干张不同面值或相同面值的货币
- 只要最后总和恰好等于 `aim`，就算一种合法方案
- 问总共有多少种不同组成方法

这里通常默认：

- `arr` 中面值互不重复
- 顺序不同不算新方法，例如 `2+3` 和 `3+2` 算同一种

所以这题是“完全背包计数”，不是最少硬币数，也不是排列数问题。

## 解题思路
递归：

```text
process(index, rest)
枚举当前面值拿 0 张、1 张、2 张...
```

DP：

- `dp[index][rest]` 表示从 `index` 往后组成 `rest` 的方法数
- 可以直接按枚举版转表
- 也能进一步优化成：

```text
dp[index][rest] = dp[index + 1][rest] + dp[index][rest - arr[index]]
```

## 代码 / 伪代码
```java
for (int index = N - 1; index >= 0; index--) {
    for (int rest = 0; rest <= aim; rest++) {
        dp[index][rest] = dp[index + 1][rest];
        if (rest - arr[index] >= 0) {
            dp[index][rest] += dp[index][rest - arr[index]];
        }
    }
}
return dp[0][aim];
```

## 记忆点
- 这是完全背包计数。
- 每种面值可用无限次。
- 不是求最少张数，是求方法数。
