## 一句话结论
数字转字母题的递归状态只有一个：当前位置 `index`，因此它很适合改成一维 DP。

## 图片转写 / 题意还原
这篇对应的原题是“数字字符串转字母方法数”：

- 规定 `1 -> A, 2 -> B, ..., 26 -> Z`
- 给定一个只由数字字符组成的字符串 `str`
- 你可以把它切分成若干段，每一段必须能合法映射到一个字母
- 问一共有多少种不同转法

例如：

- `"111"` 可以转成 `"AAA"`、`"KA"`、`"AK"`
- `"0"` 不能单独转
- `"30"` 也不是合法转法

这篇笔记的重点是把原来的递归：

```text
process(index)
```

改成一维动态规划数组 `dp[index]`。

## 解题思路
- `dp[i]` 表示从 `i` 位置开始转，有多少种方法
- `dp[N] = 1`
- 从右往左填表
- 遇到 `'0'`，`dp[i] = 0`

## 代码 / 伪代码
```java
int[] dp = new int[N + 1];
dp[N] = 1;
for (int i = N - 1; i >= 0; i--) {
    if (str[i] != '0') {
        dp[i] = dp[i + 1];
        if (i + 1 < N) {
            int num = (str[i] - '0') * 10 + (str[i + 1] - '0');
            if (num <= 26) {
                dp[i] += dp[i + 2];
            }
        }
    }
}
return dp[0];
```

## 记忆点
- 状态只有一个位置参数。
- 从右往左填。
- `'0'` 不能单独转。
