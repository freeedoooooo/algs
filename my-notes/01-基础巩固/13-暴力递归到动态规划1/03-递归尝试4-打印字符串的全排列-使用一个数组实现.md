## 一句话结论
全排列题的本质是：当前位置该放谁。常见写法是把当前位置和后面每个位置依次交换，然后递归处理下一个位置。

## 核心知识点
- 参数 `index` 表示当前该决定哪一位。
- `[index...end]` 中任意字符都可以来到当前位置。
- 交换、递归、恢复现场，是经典套路。

## 图片转写 / 题意还原
原始笔记只保留了题目名称，本节按课程标准内容补全：

- 给定字符串，打印它的全部排列。

补成完整题意后，这题的要求是：

- 输入一个字符串，例如 `"abc"`
- 需要把其中所有字符重新排列
- 每一种不同的位置安排都算一种排列
- 要求输出全部排列结果

这里和“子序列”完全不同：

- 子序列是“选哪些字符”
- 全排列是“所有字符都必须用上，只是顺序重新安排”

因此当字符串长度为 `N` 且字符互不相同时，答案数量是 `N!`。

## 图解
对 `abc`：

```text
第 0 位可放 a / b / c
第 1 位再从剩余字符里选
...
```

## 解题思路
- 如果 `index == str.length`，收集当前排列
- 否则让 `i` 从 `index` 枚举到末尾
- 交换 `str[index]` 和 `str[i]`
- 递归处理 `index + 1`
- 再交换回来恢复现场

## 复杂度
- 时间复杂度：`O(N * N!)`
- 空间复杂度：`O(N)`

## 代码 / 伪代码
```java
void process(char[] str, int index, List<String> ans) {
    if (index == str.length) {
        ans.add(String.valueOf(str));
        return;
    }
    for (int i = index; i < str.length; i++) {
        swap(str, index, i);
        process(str, index + 1, ans);
        swap(str, index, i);
    }
}
```

## 记忆点
- 排列题问的是“这一位放谁”。
- 交换 + 递归 + 恢复。
- 当前位置之后的每个字符都要试。
