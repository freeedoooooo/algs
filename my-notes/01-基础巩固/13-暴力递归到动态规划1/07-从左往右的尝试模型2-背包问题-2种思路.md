## 一句话结论
0/1 背包是从左往右尝试模型的另一个代表：来到第 `index` 个物品时，只需要决定“要”还是“不要”。
递归参数常见有两种设计：剩余容量，或者已用容量。

## 核心知识点
- 每个物品只能选一次。
- 状态通常是：当前来到哪件物品、当前背包还剩多少容量。
- 当前层决策：不要当前物品 / 要当前物品。

## 图片转写 / 题意还原
原图内容为：

- 给定两个长度都为 `N` 的数组 `weights` 和 `values`
- `weights[i]` 和 `values[i]` 分别表示第 `i` 号物品重量和价值
- 给定一个正数 `bag`，表示载重
- 返回能装下的最大价值

把题目完整展开，就是标准 0/1 背包：

- 有 `N` 件物品
- 第 `i` 件物品重量为 `weights[i]`，价值为 `values[i]`
- 背包最多只能承受 `bag` 的总重量
- 每件物品最多只能拿一次，不能切开，也不能重复拿
- 目标是在不超重的前提下，让总价值尽可能大

这题要求输出的是“最大价值”，不是具体拿法；但递归和 DP 设计时，本质上都在枚举“当前物品拿还是不拿”。

## 解题思路
定义：

```text
process(index, rest)
表示从 index 往后选，背包还剩 rest 容量时的最大价值
```

则：

- 不要当前物品：`p1 = process(index + 1, rest)`
- 要当前物品：
  - 如果 `weights[index] > rest`，这条路非法
  - 否则 `p2 = values[index] + process(index + 1, rest - weights[index])`

答案是 `max(p1, p2)`。

## 复杂度
- 暴力递归时间复杂度：`O(2^N)`
- 空间复杂度：`O(N)`

## 代码 / 伪代码
```java
int process(int[] w, int[] v, int index, int rest) {
    if (rest < 0) return -1;
    if (index == w.length) return 0;
    int p1 = process(w, v, index + 1, rest);
    int p2 = -1;
    int next = process(w, v, index + 1, rest - w[index]);
    if (next != -1) {
        p2 = v[index] + next;
    }
    return Math.max(p1, p2);
}
```

## 记忆点
- 背包是从左往右模型。
- 每件物品要 / 不要。
- 参数常用 `index + rest`。
