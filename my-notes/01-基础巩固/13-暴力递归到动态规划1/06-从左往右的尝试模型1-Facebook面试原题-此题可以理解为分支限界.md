## 一句话结论
这题的标准模型是“从左往右”：来到字符串某个位置时，只关心当前这位怎么转，或当前两位能不能一起转。
经典例子是数字字符串转字母方案数，和力扣的 `Decode Ways` 是同一题型。

## 核心知识点
- `1 -> A, 2 -> B, ..., 26 -> Z`
- 递归参数是当前来到的位置 `index`
- 每次尝试 1 位或 2 位
- `0` 不能单独转

## 图片转写 / 题意还原
原图内容为：

- 规定 `1` 和 `A` 对应，`2` 和 `B` 对应，`3` 和 `C` 对应...
- 数字字符串比如 `"111"` 可以转化为：
  - `"AAA"`
  - `"KA"`
  - `"AK"`
- 给定一个只由数字字符组成的字符串 `str`
- 返回有多少种转化结果

## 解题思路
- 如果来到末尾，说明前面转化合法，返回 `1`
- 如果当前字符是 `'0'`，非法，返回 `0`
- 先尝试单独转当前一位
- 如果当前位置和下一位组成的数字 `<= 26`，再尝试两位一起转

## 复杂度
- 暴力递归时间复杂度：`O(2^N)`
- 空间复杂度：`O(N)`

## 代码 / 伪代码
```java
int process(char[] str, int index) {
    if (index == str.length) return 1;
    if (str[index] == '0') return 0;
    int ways = process(str, index + 1);
    if (index + 1 < str.length) {
        int num = (str[index] - '0') * 10 + (str[index + 1] - '0');
        if (num <= 26) {
            ways += process(str, index + 2);
        }
    }
    return ways;
}
```

## 记忆点
- 从左往右，位置参数最关键。
- 当前看 1 位，也可能看 2 位。
- `0` 单独出现直接死路。
