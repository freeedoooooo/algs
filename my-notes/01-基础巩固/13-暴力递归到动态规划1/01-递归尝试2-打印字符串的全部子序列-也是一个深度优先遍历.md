## 一句话结论
打印字符串全部子序列，是最基础的递归树题：每到一个字符，都只有“要”或“不要”两种选择。
因此整题天然是一棵二叉决策树，也可以看成最简单的 DFS。

## 核心知识点
- 每个位置有两种决策：选 / 不选。
- 递归参数通常是当前位置 `index` 和当前路径 `path`。
- 当 `index` 走到末尾时，当前路径就是一个子序列。

## 图片转写 / 题意还原
原始笔记只保留了题目名称，本节按课程标准内容补全：

- 给定字符串 `str`
- 打印它的全部子序列
- 子序列保持相对顺序，但可以跳过字符

把题目说完整一点，就是：

- 输入一个字符串，例如 `"abc"`
- 你可以从左到右决定每个字符“保留”还是“跳过”
- 只要保留下来的字符相对先后顺序不变，就算一个合法子序列
- 要求枚举并打印所有可能的子序列
- 空串通常也算一个合法子序列

这里要和“子串”区分开：

- 子串必须连续
- 子序列不要求连续，只要求相对顺序不乱

## 图解
对 `abc`：

```text
index=0，a 要 / 不要
-> index=1，b 要 / 不要
-> index=2，c 要 / 不要
```

最终得到：

```text
""
"a" "b" "c"
"ab" "ac" "bc"
"abc"
```

## 解题思路
- 到某个位置时，做两种选择
- 递归到下一位
- 走到末尾时收集答案

## 复杂度
- 时间复杂度：`O(2^N)`
- 空间复杂度：`O(N)`

## 代码 / 伪代码
```java
void process(char[] str, int index, String path, List<String> ans) {
    if (index == str.length) {
        ans.add(path);
        return;
    }
    process(str, index + 1, path, ans);
    process(str, index + 1, path + str[index], ans);
}
```

## 记忆点
- 子序列模型就是“要 / 不要”。
- 递归树是二叉树。
- 保序，但可跳过。
