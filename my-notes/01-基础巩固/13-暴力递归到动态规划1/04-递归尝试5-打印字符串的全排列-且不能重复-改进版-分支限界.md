## 一句话结论
有重复字符时，普通全排列会生成重复答案。改进版的关键是“同一层递归里，相同字符只让它上场一次”，这就是典型的分支限界。

## 核心知识点
- 去重发生在“同层决策”。
- 常见做法是本层准备一个 `visited[256]` 或 `HashSet<Character>`。
- 本层如果某字符已经试过，就跳过后续同字符分支。

## 图片转写 / 题意还原
原始笔记只保留了题目名称，本节按课程标准内容补全。

完整题意是：

- 给定一个可能包含重复字符的字符串
- 要求输出它的所有不同排列
- 如果两个排列最终字符串完全一样，只能保留一次

例如 `acc`：

- 普通交换法会把两个 `c` 分别拿来放到同一层的某个位置
- 这样会生成重复答案
- 正确结果应该只有：`acc`、`cac`、`cca`

所以这题不是“生成后再整体去重”这么简单，而是更强调：

```text
在递归展开时就剪掉重复分支
让同一层相同字符只尝试一次
```

## 解题思路
- 外层框架仍然是交换法全排列
- 但在每层开始前，开一个本层去重结构
- 只有本层第一次遇到某个字符时才展开分支

## 复杂度
- 最坏时间复杂度仍接近 `O(N * N!)`
- 空间复杂度：`O(N)`

## 代码 / 伪代码
```java
void process(char[] str, int index, List<String> ans) {
    if (index == str.length) {
        ans.add(String.valueOf(str));
        return;
    }
    boolean[] used = new boolean[256];
    for (int i = index; i < str.length; i++) {
        if (!used[str[i]]) {
            used[str[i]] = true;
            swap(str, index, i);
            process(str, index + 1, ans);
            swap(str, index, i);
        }
    }
}
```
