## 一句话结论
范围模型的特点是：状态不是“来到哪个位置”，而是“当前只剩哪个区间”。最经典代表就是两个人从数组两端拿纸牌的博弈题。

## 核心知识点
- 状态通常是区间 `[L, R]`
- 常搭配博弈论里的“先手 / 后手”两个函数
- 当前层决策往往是拿左还是拿右

## 图片转写 / 题意还原
原图内容为：

- 给定一个整型数组 `arr`
- 玩家 A 和玩家 B 依次拿走每张纸牌
- 规定玩家 A 先拿，玩家 B 后拿
- 每个玩家每次只能拿最左或最右的纸牌
- 两个玩家都绝顶聪明
- 请返回最后获胜者的分数

原始笔记还补充了“绝顶聪明”对应的博弈论含义：

- 双方都不会做出让自己更差的决策

把题目改写成完整说明，就是：

- 给定一个整型数组 `arr`，每个位置表示一张纸牌上的分数
- 一排纸牌从左到右排好
- 两名玩家轮流拿牌，A 先手，B 后手
- 每次只能从最左端或最右端拿走一张
- 两名玩家都足够聪明，都会让自己的最终得分最大化
- 当牌被拿完后，比较双方总分
- 题目要求返回最终获胜者的分数

这题表面是拿牌游戏，实质是在问：

```text
面对区间 [L, R]
当前轮到谁拿
他在最优决策下最终能拿到多少分
```

## 解题思路
定义两个函数：

- `f(L, R)`：当前轮到先手在区间 `[L, R]` 上拿，先手最终最好分数
- `s(L, R)`：当前轮到后手在区间 `[L, R]` 上拿，后手最终最好分数

则：

- `f(L, R) = max(arr[L] + s(L+1,R), arr[R] + s(L,R-1))`
- `s(L, R) = min(f(L+1,R), f(L,R-1))`

## 复杂度
- 暴力递归复杂度较高，后续可改 DP

## 代码 / 伪代码
```java
int f(int[] arr, int L, int R) {
    if (L == R) return arr[L];
    return Math.max(arr[L] + s(arr, L + 1, R),
                    arr[R] + s(arr, L, R - 1));
}

int s(int[] arr, int L, int R) {
    if (L == R) return 0;
    return Math.min(f(arr, L + 1, R),
                    f(arr, L, R - 1));
}
```

## 记忆点
- 范围模型看 `[L, R]`。
- 常见于博弈题。
- 先手取最大，后手给对方留最差。
