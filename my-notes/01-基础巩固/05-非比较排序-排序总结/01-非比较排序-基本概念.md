## 一句话结论
非比较排序的核心，不是通过“比较两个元素谁大谁小”来确定顺序，而是利用数据本身的结构信息直接分配位置。  
它们之所以可能突破比较排序的 `O(N log N)` 下界，是因为走的不是“比较决策树”那条路，但代价是适用条件通常更苛刻。

## 先记结论
第一次看这章，建议先记住下面 4 句话：

```text
1. 非比较排序不靠元素两两比较来决定顺序
2. 它可能做到 O(N)、O(N + K)、O(N * d) 这类复杂度
3. 常见代表：桶排序、计数排序、基数排序
4. 速度更快，不等于更通用
```

## 核心知识点
- 比较排序的理论下界是 `O(N log N)`，前提是排序过程必须依赖比较结果。
- 非比较排序能突破这个下界，是因为它直接利用值域、位数、分布等数据特征。
- 桶排序、计数排序、基数排序都属于非比较排序，但不是同一个算法。
- 这类算法通常更依赖额外前提，因此“更快”和“更通用”很难同时兼得。
- 很多非比较排序不是原地排序，而且有些还必须依赖“稳定性”。

## 图解
可以先把两大类排序放到一张图里：

```text
排序
├─ 比较排序
│  ├─ 选择 / 冒泡 / 插入
│  ├─ 归并
│  ├─ 快速排序
│  └─ 堆排序
└─ 非比较排序
   ├─ 桶排序
   ├─ 计数排序
   └─ 基数排序
```

它们的核心差别可以粗略理解成：

```text
比较排序：靠“谁大谁小”一步步逼近最终顺序
非比较排序：靠“值域 / 位数 / 分布”直接把元素送去更接近目标的位置
```

再进一步，可以把这三类非比较排序的关系理解成：

```text
桶排序   ：按区间或范围分桶，桶内再处理
计数排序 ：值本身就能直接映射为桶下标
基数排序 ：按个位、十位、百位一轮轮分桶
```

## 解题思路
### 1. 为什么比较排序有 `O(N log N)` 下界
像选择、冒泡、快排、归并、堆排，本质上都要依赖比较结果来缩小可能的排列范围。  
如果只允许靠比较来判断顺序，就会落到判定树模型，最终得到：

```text
O(N log N)
```

这也是为什么“只靠比较”时，很难再本质突破这个级别。

### 2. 非比较排序为什么有机会更快
因为它绕开了“必须比较出全序”的过程，转而直接利用数据特征：

- 计数排序：统计每个值出现了几次
- 基数排序：按位稳定分发
- 桶排序：按区间把元素先分进不同桶

也就是说，它不是在问：

```text
A 和 B 谁大？
```

而是在问：

```text
这个元素应该先被送到哪个桶 / 哪一位 / 哪个范围？
```

### 3. 为什么又不能随便用
因为它必须满足额外前提：

- 计数排序要求值域不大
- 基数排序要求位数结构清楚，通常处理非负整数
- 桶排序要求你能合理分桶，而且数据分布最好不要太极端

如果这些条件不满足：

- 空间可能爆炸
- 桶内负担可能很重
- 实现成本可能反而高于比较排序

## 典型例子
### 计数排序的感觉

```text
[3, 1, 2, 1, 0, 3]
```

不是比较大小，而是直接统计：

```text
0 -> 1 次
1 -> 2 次
2 -> 1 次
3 -> 2 次
```

然后按统计结果回填。

### 基数排序的感觉

```text
[53, 3, 542, 748, 14, 214]
```

不是直接整体比较，而是：

```text
先按个位分桶
再按十位分桶
再按百位分桶
```

每一轮都保持稳定，最后整体有序。

### 桶排序的感觉

```text
[0.42, 0.17, 0.89, 0.33, 0.58]
```

先按区间分桶：

```text
[0.0, 0.2) -> 0.17
[0.2, 0.4) -> 0.33
[0.4, 0.6) -> 0.42, 0.58
[0.8, 1.0) -> 0.89
```

再桶内各自排序，最后按桶顺序拼起来。

## 复杂度
- 时间复杂度：通常是 `O(N + K)`、`O(N * d)` 这类线性或近线性复杂度
- 空间复杂度：通常需要额外桶、计数数组或辅助数组，因此一般不是原地排序

其中：

- `K` 可能是值域大小、桶数量等
- `d` 可能是最大位数

## 易错点
- 非比较排序不是一个具体算法名，而是一大类排序方法。
- 桶排序、计数排序、基数排序有关联，但通常应当并列理解，而不是完全等同。
- “更快”依赖前提，不要脱离数据范围、位数、分布去谈复杂度。
- 很多非比较排序要依赖稳定性，尤其是基数排序。
