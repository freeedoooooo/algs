## 一句话结论
狭义桶排序的核心是：

```text
先按数值范围把元素分到不同桶里
再在每个桶内部各自排序
最后按桶顺序依次收集
```

它适合“数据分布较均匀、容易设计桶范围”的场景；如果分布很差，桶排序的优势会明显下降。

## 先记结论
如果只想先抓住桶排序的轮廓，可以先记这 4 句：

```text
1. 桶排序是非比较排序里的“按区间分桶”方法
2. 桶内通常还要再排序
3. 数据越均匀，桶排序越好用
4. 分桶设计得不好，桶排序可能退化
```

另外要特别区分：

- 这篇讲的是狭义“桶排序”
- 后面的计数排序、基数排序虽然也带有桶思想，但通常作为并列算法单独讲

## 核心知识点
- 桶排序适合元素值能自然落入某些范围区间的场景。
- 桶排序的关键不是“分多少桶”，而是“怎么让元素尽量均匀地落到桶里”。
- 如果每个桶都很小，桶内排序的总成本就会比较低。
- 如果大量元素都挤进同一个桶，桶排序的优势就会消失。
- 桶排序的性能高度依赖数据分布和分桶策略。

## 图片转写 / 题意还原
如果把原始标题真正落成一篇完整题意，这篇要回答的是：

```text
什么是狭义桶排序
它为什么有机会很快
以及它为什么强依赖数据分布
```

也就是：

1. 先把数据按范围扔进多个桶
2. 再处理每个桶内部
3. 最后按桶顺序拼接结果

## 图解
假设我们要排序下面这些落在 `[0, 1)` 区间的小数：

```text
[0.42, 0.17, 0.89, 0.33, 0.58, 0.21, 0.76]
```

如果准备 5 个桶，可以按区间分成：

```text
桶 0: [0.0, 0.2)
桶 1: [0.2, 0.4)
桶 2: [0.4, 0.6)
桶 3: [0.6, 0.8)
桶 4: [0.8, 1.0)
```

把元素放进去后：

```text
桶 0 -> 0.17
桶 1 -> 0.33, 0.21
桶 2 -> 0.42, 0.58
桶 3 -> 0.76
桶 4 -> 0.89
```

然后桶内各自排序：

```text
桶 0 -> 0.17
桶 1 -> 0.21, 0.33
桶 2 -> 0.42, 0.58
桶 3 -> 0.76
桶 4 -> 0.89
```

最后按桶编号从小到大收集：

```text
[0.17, 0.21, 0.33, 0.42, 0.58, 0.76, 0.89]
```

这个过程最值得记住的感觉是：

```text
桶与桶之间天然有序
真正需要细排的，只剩桶内
```

## 解题思路
### 1. 先设计桶
桶排序第一步不是开始排序，而是先决定：

- 用多少个桶
- 每个桶负责哪个数值区间

例如对 `[0, 1)` 的小数，可以平均分桶；  
对整数，也可以按范围分成：

```text
[0, 9], [10, 19], [20, 29] ...
```

### 2. 把元素分配到对应桶中
每个元素根据自己的值，计算它属于哪个桶。

本质上是在做：

```text
值 -> 桶编号
```

这一步如果分布均匀，元素就会比较平均地落到各个桶里。

### 3. 桶内各自排序
桶与桶之间的顺序已经确定了，但桶内部仍然可能是乱的。  
所以需要对每个桶内部继续排序。

桶内排序可以用很多方法：

- 插入排序
- 快速排序
- 内置排序

当每个桶里的元素都很少时，桶内排序的成本就不高。

### 4. 按桶顺序收集结果
最后从桶 0 到桶 1、桶 2……依次收集，就能得到整体有序序列。

因为：

```text
前一个桶里的元素一定不大于后一个桶里的元素
```

## 为什么桶排序有时很快
桶排序快的原因，不是桶内排序 magically 变快了，而是：

```text
原本一个大排序问题
被拆成了很多个小排序问题
```

如果数据分布均匀：

- 每个桶都不大
- 桶内排序开销低
- 总体效果很好

这时桶排序常常可以接近线性时间。

## 为什么桶排序又可能退化
桶排序最怕这种情况：

```text
分桶设计不合理
或者数据分布极不均匀
```

例如所有数据都落到同一个桶里，那么就会变成：

```text
前面分桶白做了
最后几乎等于只排了一个大桶
```

这时桶排序的优势就基本没了。

所以桶排序不是“无脑快”，它很依赖：

- 桶的设计
- 数据分布

## 典型例子
### 适合桶排序的例子

```text
[0.42, 0.17, 0.89, 0.33, 0.58, 0.21, 0.76]
```

值域明确、分布比较均匀，分桶后每个桶都不大，很适合桶排序。

### 不适合桶排序的例子

```text
[0.91, 0.92, 0.93, 0.94, 0.95]
```

如果桶设计还是均匀分在 `[0, 1)`，那大部分元素可能都挤在最后一个桶里。  
这时桶排序就失去了“分散压力”的意义。

## 复杂度
桶排序的复杂度通常不写成一个固定死板的公式，因为它很依赖分布。  
粗略理解可以写成：

- 分桶与收集：`O(N)`
- 桶内排序：取决于每个桶里元素数量和桶内排序方法

理想情况下：

```text
接近 O(N)
```

较差情况下：

```text
可能明显退化
```

空间复杂度方面，通常需要：

- 多个桶
- 桶内存储结构

所以一般不是原地排序。

## 易错点
- 桶排序不是“任何数据都快”，它强依赖数据分布。
- 桶排序的重点不是桶多，而是桶分得是否合理。
- 桶与桶之间天然有序，不代表桶内已经有序。
- 计数排序、基数排序虽然也带桶思想，但通常和狭义桶排序并列讲。

## 记忆点
- 桶排序 = 先按区间分桶，再桶内排序。
- 桶与桶之间天然有序，真正细排的是桶内。
- 数据越均匀，桶排序越好发挥。
- 分桶不好或分布太偏，桶排序就可能退化。
