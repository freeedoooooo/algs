## 章节导读
这一章是二叉树部分真正的起点，核心任务不是上来就做复杂树形 DP，而是先把“树到底怎么遍历、怎么观察、怎么落成代码工具”这套基础打扎实。

整章可以分成三条主线：

- 遍历认知
- 按层处理
- 基于遍历的应用与工具

如果压成一张结构图，可以这样看：

```text
二叉树基本用法
├─ 遍历认知
│  ├─ 理解递归序
│  ├─ 先中后序
│  └─ 非递归遍历
├─ 按层处理
│  ├─ 宽度优先遍历
│  ├─ 最大宽度
│  └─ 最宽层节点数
└─ 遍历的应用与工具
   ├─ 序列化 / 反序列化
   ├─ 打印整棵树
   ├─ 中序前驱 / 后继
   └─ 折纸问题
```

## 本章结论
- 二叉树题最底层的能力，是把递归序、先序、中序、后序、层序之间的关系看懂。
- 先中后序是深度优先遍历的不同落点，按层遍历是宽度优先遍历。
- 很多看似不同的树题，本质都是“在某种遍历顺序上加信息处理”。
- 序列化、打印树、找前驱后继、折纸问题，都不是孤立技巧，它们都建立在遍历理解之上。
- 这一章学完之后，后面的“二叉树递归套路”才会真正顺起来。

## 图片转写 / 章节笔记还原
原始章节页本身没有额外配图，核心信息主要体现在这 10 道题的排列顺序里。

这个顺序其实很有讲究，可以还原成下面这条学习路径：

### 1. 先理解递归，再理解三序遍历
前两题不是让你死记模板，而是先建立一个底层认识：

- 每个节点在递归里会经历多个逻辑时机
- 先序 / 中序 / 后序只是“在不同时间点处理节点”

这一步没看懂，后面的树题很容易只剩背模板。

### 2. 再把递归遍历改写成非递归
第三题紧接着把系统栈换成手动栈，目的是让你真正理解：

```text
递归不是魔法
只是系统帮你维护了现场
```

一旦这一步打通，很多树题会从“死记递归”变成“知道为什么这样写”。

### 3. 再补齐层序遍历这一条线
第四题开始切到 BFS，也就是按层遍历。

这一步很重要，因为从这里开始你会看到：

- 不是所有树题都适合 DFS
- 只要题目和“层”“宽度”“逐层统计”有关，就应该先想到队列

### 4. 然后把遍历能力变成具体应用
后面几题其实都是“遍历 + 一点结构信息”的应用题：

- 宽度统计
- 序列化与反序列化
- 打印树结构
- 找中序后继 / 前驱
- 折纸问题映射成中序遍历

也就是说，本章后半部分真正想传达的是：

```text
遍历不是目的
遍历是工具
```

## 记忆点
- 树题第一步先想清楚：这是 DFS 题还是 BFS 题。
- 先中后序是“递归序的不同处理时机”，不是三套互不相关的模板。
- 层序、宽度、逐层统计，优先想到队列。
- 序列化、打印树、折纸题，本质都还是在用遍历。
