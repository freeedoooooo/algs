## 一句话结论
一个节点在中序遍历中的后继，就是“中序序列里它后面的那个节点”；前驱则是“中序序列里它前面的那个节点”。
如果节点带有父指针，那么后继和前驱都可以在 `O(H)` 时间内直接找到，不需要真的把整棵树中序遍历一遍。

## 核心知识点
- 这题讨论的是普通二叉树的中序后继 / 前驱，不要求是二叉搜索树。
- 找后继最重要的两种情况：
  - 有右子树：后继就是右子树的最左节点
  - 无右子树：往上找，直到当前节点第一次是某个祖先的左孩子
- 找前驱和后继完全对称：
  - 有左子树：前驱就是左子树的最右节点
  - 无左子树：往上找，直到当前节点第一次是某个祖先的右孩子
- 如果没有父指针，也能做，但通常要从根开始辅助中序遍历。

## 图片转写 / 题意还原
原始笔记虽然只有题目名称，但题意其实很明确：给定二叉树中的某个节点，要求找到它在中序遍历序列里的后继节点或前驱节点。

这题真正要解决的是：

- 给你树上的一个节点 `x`
- 不展开整棵树的完整中序结果
- 直接找到它的中序后继或前驱

中序遍历顺序是：

```text
左 -> 根 -> 右
```

所以所谓：

- 后继：中序里刚好排在它后面的节点
- 前驱：中序里刚好排在它前面的节点

## 图解
假设二叉树如下：

```text
        6
      /   \
     3     9
    / \   / \
   1   4 8  10
        \
         5
```

其中序遍历结果是：

```text
1 3 4 5 6 8 9 10
```

所以：

- `4` 的后继是 `5`
- `4` 的前驱是 `3`
- `6` 的后继是 `8`
- `6` 的前驱是 `5`
- `10` 没有后继
- `1` 没有前驱

### 1. 后继：有右子树时
如果节点 `x` 有右子树，那么后继一定在它的右子树里，而且是：

```text
右子树中最左的那个节点
```

例如 `3`：

```text
    3
   / \
  1   4
       \
        5
```

`3` 的右子树根是 `4`，右子树最左节点也是 `4`，所以后继就是 `4`。

### 2. 后继：没有右子树时
如果节点没有右子树，就要往上找祖先：

```text
一直往上走
直到当前节点第一次成为某个父节点的左孩子
那个父节点就是后继
```

例如 `5`：

```text
5 是 4 的右孩子
4 是 3 的右孩子
3 是 6 的左孩子
```

所以 `5` 的后继就是 `6`。

### 3. 前驱完全对称
前驱的找法和后继是镜像关系：

- 有左子树：找左子树最右节点
- 没左子树：往上找，直到当前节点第一次成为某个父节点的右孩子

## 解题思路
### 1. 找中序后继
假设当前节点为 `node`。

情况一：`node.right != null`

```text
后继 = node.right 这棵树里的最左节点
```

情况二：`node.right == null`

```text
不断往父节点方向走
直到第一次发现：当前节点是父节点的左孩子
这个父节点就是后继
```

如果一路走到根都没找到，说明它已经是中序最后一个节点，没有后继。

伪代码：

```java
Node getSuccessorNode(Node node) {
    if (node == null) {
        return null;
    }
    if (node.right != null) {
        return getLeftMost(node.right);
    } else {
        Node parent = node.parent;
        while (parent != null && parent.right == node) {
            node = parent;
            parent = node.parent;
        }
        return parent;
    }
}
```

### 2. 找中序前驱
完全镜像。

情况一：`node.left != null`

```text
前驱 = node.left 这棵树里的最右节点
```

情况二：`node.left == null`

```text
不断往父节点方向走
直到第一次发现：当前节点是父节点的右孩子
这个父节点就是前驱
```

伪代码：

```java
Node getPredecessorNode(Node node) {
    if (node == null) {
        return null;
    }
    if (node.left != null) {
        return getRightMost(node.left);
    } else {
        Node parent = node.parent;
        while (parent != null && parent.left == node) {
            node = parent;
            parent = node.parent;
        }
        return parent;
    }
}
```

### 3. 为什么这个结论成立
因为中序顺序固定是：

```text
左 -> 根 -> 右
```

所以：

- 如果当前节点有右子树，那它之后最先访问的一定是右子树最左侧路径
- 如果没有右子树，就只能回到祖先中，寻找第一个“还没被访问过自己”的祖先

前驱同理，只不过方向全部反过来。

## 典型例子
### 例 1：找后继
树：

```text
    6
   / \
  3   9
     /
    8
```

找 `6` 的后继：

- `6` 有右子树
- 右子树最左节点是 `8`

所以：

```text
successor(6) = 8
```

### 例 2：找后继
树：

```text
    6
   /
  3
   \
    5
```

找 `5` 的后继：

- `5` 没有右子树
- 往上走：`5` 是 `3` 的右孩子
- 再往上走：`3` 是 `6` 的左孩子

所以：

```text
successor(5) = 6
```

### 例 3：找前驱
找 `6` 的前驱：

- `6` 有左子树
- 左子树最右节点是 `5`

所以：

```text
predecessor(6) = 5
```

## 复杂度
- 时间复杂度：`O(H)`
- 空间复杂度：`O(1)`

其中：

- `H` 是树高

这是在“节点带父指针”的前提下成立的。

## 易错点
- 这题讨论的是中序后继 / 前驱，不是二叉搜索树里的大小关系。
- 有右子树时，后继不是“右孩子本身”，而是“右子树最左节点”。
- 有左子树时，前驱不是“左孩子本身”，而是“左子树最右节点”。
- 往上找祖先时，判断的是“当前节点是父节点的左孩子还是右孩子”，方向别写反。
- 如果节点没有后继或前驱，要返回 `null`。

## 记忆点
- 后继：右树有就取右树最左，没有就往上找第一个把自己当左孩子的祖先。
- 前驱：左树有就取左树最右，没有就往上找第一个把自己当右孩子的祖先。
- 这是中序关系题，不是 BST 大小题。
- 有父指针时可以做到 `O(H)`。
