## 一句话结论
设计打印整棵二叉树的函数，本质上不是算法题，而是“如何把树结构稳定、直观地投影到文本里”的工程题。
最常见也最实用的做法，是把整棵树顺时针旋转 90 度后打印，用“右子树在上、根在中、左子树在下”的顺序输出。

## 核心知识点
- 打印函数的目标不是花哨，而是让节点层次关系一眼能看出来。
- 最经典的文本打印方案，是把树旋转 90 度后按“右-根-左”递归输出。
- 通常会给节点值外面包一层标记，例如根节点用 `H`，右孩子用 `v`，左孩子用 `^`。
- 每个节点打印时占固定宽度，才能在多层结构下保持整体对齐。
- 这题虽然像工具题，但很适合训练树递归和格式化输出的思维。

## 图片转写 / 题意还原
原始笔记只留下了一句：

```text
（作为思考题）
```

这其实很符合这题定位。

它不是在考一个标准面试套路，而是在问：

- 如果调试树结构时只打印中序或先序结果，往往看不出树形状
- 那有没有一种文本方式，可以把整棵树的结构直观打印出来

所以这题真正想解决的是“可视化问题”。

## 图解
假设二叉树如下：

```text
        1
      /   \
     2     3
    / \   / \
   4   5 6   7
```

如果直接旋转 90 度来看，它会变成：

```text
        7
    3
        6
1
        5
    2
        4
```

为了让方向更清楚，通常会加标记：

- 根节点：`H1H`
- 右孩子：`v3v`
- 左孩子：`^2^`

打印效果可以理解为：

```text
                v7v
        v3v
                ^6^
H1H
                v5v
        ^2^
                ^4^
```

读法是：

- 越靠上越偏向原树的右边
- 越靠下越偏向原树的左边
- 缩进越多，层数越深

## 解题思路
### 1. 为什么要按“右-根-左”打印
如果想让树横着长出来，根节点在左边，右子树往上，左子树往下，那么打印顺序就应该是：

```text
先打印右子树
再打印当前节点
最后打印左子树
```

因为：

- 右子树要出现在上面，所以必须先输出
- 根节点居中
- 左子树要出现在下面，所以最后输出

这其实就是一种“旋转后的中序遍历”。

### 2. 为什么要固定每个节点宽度
如果节点值直接裸打，比如有的节点是一位数，有的是三位数，那么整棵树会左右错位。

所以常见做法是：

- 给每个节点字符串一个固定宽度 `len`
- 节点值左右补空格
- 这样每层缩进后，整体排版才稳定

例如固定宽度为 `17`：

```text
       H1H
```

不管是 `1`、`23` 还是 `456`，最终都占相同打印宽度。

### 3. 递归函数怎么设计
递归函数通常需要这几个参数：

- 当前节点 `head`
- 当前层深度 `height`
- 节点标记字符串 `to`
- 固定打印宽度 `len`

递归流程：

1. 先递归打印右子树，层数 `height + 1`
2. 再打印当前节点，前面补 `height * len` 个空格
3. 节点值再用固定宽度居中
4. 最后递归打印左子树

伪代码：

```java
void printInOrder(Node head, int height, String to, int len) {
    if (head == null) {
        return;
    }
    printInOrder(head.right, height + 1, "v", len);
    String val = to + head.value + to;
    val = center(val, len);
    printSpaces(height * len);
    print(val);
    printInOrder(head.left, height + 1, "^", len);
}
```

### 4. 为什么这题值得掌握
因为很多树题调试时最痛苦的点是：

```text
结果不对
但你看不出来树长啥样
```

如果你有这样一个打印函数，就能很快看出：

- 左右孩子挂反没
- 某一层结构是不是断了
- 序列化反序列化后的树是否恢复正确

所以它虽然不一定是面试主问题，但非常实用。

## 典型例子
还是看这棵树：

```text
    1
   / \
  2   3
```

递归打印顺序是：

1. 先打印右子树 `3`
2. 再打印根 `1`
3. 最后打印左子树 `2`

最终文本效果类似：

```text
        v3v
H1H
        ^2^
```

如果再加一层：

```text
      1
     / \
    2   3
       /
      6
```

打印效果类似：

```text
        v3v
                ^6^
H1H
        ^2^
```

从这个输出里，能直接看出：

- `3` 是根的右孩子
- `6` 是 `3` 的左孩子

## 复杂度
- 时间复杂度：`O(N)`
- 空间复杂度：`O(H)`

其中：

- `N` 是节点数
- `H` 是树高

如果把字符串拼接和补空格细算进去，实际输出成本还和每个节点的打印宽度有关，但主过程仍然是每个节点访问一次。

## 易错点
- 打印顺序一定是“右-根-左”，不要写成普通中序“左-根-右”。
- 每层前面的缩进应该和当前层数成正比，否则结构会挤在一起。
- 节点最好固定打印宽度，否则不同位数会把整棵树排乱。
- 根节点、左孩子、右孩子最好加不同标记，不然方向感会差很多。
- 这题是调试工具题，不要过度追求绝对对齐到每个字符都完美，先保证结构清晰。

## 代码 / 伪代码
仓库里目前没有单独命名为“打印整棵二叉树”的现成实现文件，这里保留课程常见的标准伪代码框架。

核心框架：

```java
void printTree(Node head) {
    printInOrder(head, 0, "H", 17);
}

void printInOrder(Node head, int height, String to, int len) {
    if (head == null) {
        return;
    }
    printInOrder(head.right, height + 1, "v", len);
    String val = to + head.value + to;
    val = center(val, len);
    printSpaces(height * len);
    System.out.println(val);
    printInOrder(head.left, height + 1, "^", len);
}
```

辅助函数通常还需要：

- `printSpaces(int num)`：打印若干空格
- `center(String val, int len)`：把节点字符串补到固定宽度

## 记忆点
- 打印树最常见方案：整棵树旋转 90 度。
- 递归顺序：右 -> 根 -> 左。
- 根常用 `H` 标记，右孩子 `v`，左孩子 `^`。
- 这题本质是“树结构可视化工具题”。
