## 一句话结论
非递归遍历二叉树，本质上就是用栈来手动模拟递归调用过程。
先序最简单，中序要学会“一路压左链”，后序最难，常见有双栈法和单栈法两种写法。

## 核心知识点
- 递归遍历依赖的是系统调用栈，非递归遍历依赖的是我们自己维护的栈。
- 先序遍历的关键是：弹出即处理，然后先压右、再压左。
- 中序遍历的关键是：当前节点不为空就一路向左压栈，空了再弹出处理并转向右树。
- 后序遍历的关键是：必须保证一个节点在左右子树都处理完之后才能处理自己。
- 后序遍历双栈法更好理解，单栈法更像真正模拟递归返回过程。


## 图解
假设二叉树如下：

```text
        1
      /   \
     2     3
    / \   / \
   4   5 6   7
```

### 1. 先序遍历为什么要先压右再压左
先序顺序是：

```text
根 -> 左 -> 右
```

栈是“后进先出”，所以想让左孩子先处理，就必须右孩子先入栈：

```text
初始：栈=[1]
弹出 1，打印 1，压 3，再压 2
栈=[3, 2]

弹出 2，打印 2，压 5，再压 4
栈=[3, 5, 4]

接下来依次得到：4 5 3 6 7
```

最终结果：

```text
1 2 4 5 3 6 7
```

### 2. 中序遍历为什么要一路压左链
中序顺序是：

```text
左 -> 根 -> 右
```

所以不能像先序那样“弹出就打印”，而是要先把一条左边界走到底：

```text
head=1，压1，去左边
head=2，压2，去左边
head=4，压4，去左边
head=null，停

弹出 4，打印 4，转向 4 的右树
右树为空，再弹出 2，打印 2，转向 5
...
```

可以把它理解为：

```text
谁的左边还没处理完，谁就不能打印
```

### 3. 后序遍历为什么最难
后序顺序是：

```text
左 -> 右 -> 根
```

也就是说，一个节点必须等左右都结束后，才能最后处理自己。
这比先序、中序多了一层“是否已经处理过某个子树”的判断，所以写法最绕。

常见有两种办法：

- 双栈法：先得到“根 -> 右 -> 左”，再逆过来。
- 单栈法：配合“上一次打印的节点”判断左右子树是否已经处理完。

## 解题思路
### 1. 先序遍历：弹出就处理
先序和递归版的对应关系是：

```text
来到当前节点就处理
然后先递归左，再递归右
```

改成栈之后：

1. 根节点先入栈
2. 每次弹出一个节点就立刻处理
3. 为了让左边先处理，先压右孩子，再压左孩子

伪代码：

```java
stack.push(head);
while (!stack.isEmpty()) {
    cur = stack.pop();
    print(cur);
    if (cur.right != null) stack.push(cur.right);
    if (cur.left != null) stack.push(cur.left);
}
```

### 2. 中序遍历：左边界全部入栈
递归版中序是：

```text
左处理完 -> 再处理根 -> 再处理右
```

非递归时就要显式保留“回来的路”：

1. 只要当前节点不为空，就一直压栈并向左走
2. 走到空后，弹出栈顶节点并处理
3. 然后转向它的右子树
4. 重复这个过程

伪代码：

```java
while (!stack.isEmpty() || head != null) {
    if (head != null) {
        stack.push(head);
        head = head.left;
    } else {
        head = stack.pop();
        print(head);
        head = head.right;
    }
}
```

### 3. 后序遍历双栈法：先造反序，再逆回来
后序是：

```text
左 -> 右 -> 根
```

如果我们故意做出：

```text
根 -> 右 -> 左
```

再把结果逆序，就正好得到后序。

做法是：

1. 第一个栈 `stack` 用来做遍历
2. 第二个栈 `collect` 用来收集“根 -> 右 -> 左”的顺序
3. `stack` 每次弹出节点后，先把它放进 `collect`
4. 然后先压左、再压右，这样 `stack` 实际弹出时就是“根 -> 右 -> 左”
5. 最后把 `collect` 依次弹出，就是“左 -> 右 -> 根”

伪代码：

```java
stack.push(head);
while (!stack.isEmpty()) {
    cur = stack.pop();
    collect.push(cur);
    if (cur.left != null) stack.push(cur.left);
    if (cur.right != null) stack.push(cur.right);
}
while (!collect.isEmpty()) {
    print(collect.pop());
}
```

### 4. 后序遍历单栈法：判断子树是否处理完成
单栈法更贴近递归本质，但更难想。

核心思想是：

- 栈顶节点 `cur` 代表“当前准备处理的节点”
- 如果它的左树还没处理过，就先处理左树
- 否则如果它的右树还没处理过，就先处理右树
- 否则说明左右都结束了，当前节点可以打印并弹出

为了知道“某棵子树处理过没有”，通常额外记录一个变量：

```text
lastVisited = 上一次真正打印并弹出的节点
```

仓库里的实现使用的是同样的思想，只是变量名沿用了 `head`。

## 典型例子
还是用这棵树：

```text
    1
   / \
  2   3
```

### 先序非递归

```text
栈变化：
[1]
弹1，打印1，压3、2
[3, 2]
弹2，打印2
[3]
弹3，打印3
```

结果：

```text
1 2 3
```

### 中序非递归

```text
一路压左：1 -> 2
弹2打印
回到1打印
转向3打印
```

结果：

```text
2 1 3
```

### 后序非递归
双栈法先得到：

```text
1 3 2
```

再逆序得到：

```text
2 3 1
```

## 复杂度
### 先序 / 中序 / 后序单栈
- 时间复杂度：`O(N)`
- 空间复杂度：`O(H)`，最坏为 `O(N)`

### 后序双栈
- 时间复杂度：`O(N)`
- 空间复杂度：`O(N)`

其中：

- `N` 是节点数
- `H` 是树高

## 易错点
- 先序非递归里一定是“先压右、再压左”，否则打印顺序会反。
- 中序非递归的循环条件必须是 `stack` 不空或者 `head` 不空，少任意一个都会漏节点。
- 中序遍历不是“弹出就往左右都走”，而是“弹出后处理，再只转向右子树”。
- 后序双栈法里，第一层栈中一定是先压左、再压右，这样弹出顺序才是“根 -> 右 -> 左”。
- 后序单栈法最容易错在“某个右子树是否已经处理过”的判断条件。
- 面试里如果只要求写出后序非递归，优先写双栈法，更稳、更不容易出错。

## 代码 / 伪代码

这份代码里包含了四种写法：

- 先序非递归：`preOrder`
- 中序非递归：`inOrder`
- 后序非递归双栈版：`posOrderTwoStacks`
- 后序非递归单栈版：`posOrderOneStack`

先序核心代码：

```java
Stack<TreeNode> stack = new Stack<>();
stack.push(head);
while (!stack.isEmpty()) {
    head = stack.pop();
    System.out.print(head.val + " ");
    if (head.right != null) {
        stack.push(head.right);
    }
    if (head.left != null) {
        stack.push(head.left);
    }
}
```

中序核心代码：

```java
Stack<TreeNode> stack = new Stack<>();
while (!stack.isEmpty() || head != null) {
    if (head != null) {
        stack.push(head);
        head = head.left;
    } else {
        head = stack.pop();
        System.out.print(head.val + " ");
        head = head.right;
    }
}
```

后序双栈核心代码：

```java
Stack<TreeNode> stack = new Stack<>();
Stack<TreeNode> collect = new Stack<>();
stack.push(head);
while (!stack.isEmpty()) {
    head = stack.pop();
    collect.push(head);
    if (head.left != null) {
        stack.push(head.left);
    }
    if (head.right != null) {
        stack.push(head.right);
    }
}
while (!collect.isEmpty()) {
    System.out.print(collect.pop().val + " ");
}
```

## 记忆点
- 非递归遍历 = 用自己维护的栈，替代系统递归栈。
- 先序：弹出就处理，先压右再压左。
- 中序：一路压左，弹出后处理，再转右。
- 后序：最难，双栈最稳，单栈最像递归。
