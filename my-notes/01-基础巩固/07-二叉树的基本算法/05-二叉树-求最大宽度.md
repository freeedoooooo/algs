## 一句话结论
这题要求的是二叉树每一层节点数的最大值，本质上仍然是按层遍历，只是在遍历过程中多做一层“当前层节点数统计”。
常见有两种写法：一种用 `Map` 记录节点所在层数，另一种不用 `Map`，而是直接用“当前层结束节点”和“下一层结束节点”来分层。

## 核心知识点
- 这题的基础模版仍然是二叉树的 BFS，也就是队列按层遍历。
- 课程语境下的“最大宽度”指某一层真实存在的节点个数，不计算中间缺失的 `null` 位置。
- 用 `Map` 的写法更直观，因为每个节点都带着“自己在哪一层”的信息。
- 不用 `Map` 的写法更省辅助空间，面试里通常更推荐。
- 只要能准确知道“当前层什么时候结束”，就能统计这一层有多少个节点。

## 图片转写 / 题意还原
原始笔记里只留下了两个提示：

```text
方法1，使用Map
方法2，不使用Map
```

这两个提示已经把这题的核心讲完了。

真正的题意是：

- 给定一棵二叉树
- 统计哪一层节点最多
- 返回这个最多的节点数

例如：

```text
        1
      /   \
     2     3
    / \     \
   4   5     6
```

三层的节点数分别是：

- 第 1 层：1 个
- 第 2 层：2 个
- 第 3 层：3 个

所以最大宽度是：

```text
3
```

注意这题和某些 LeetCode 版本不一样：

- 这里不统计中间空位
- 只统计这一层实际存在的节点数量

## 图解
还是看这棵树：

```text
        1
      /   \
     2     3
    / \     \
   4   5     6
```

按层遍历时，队列变化如下：

```text
初始：[1]
```

### 第 1 层

```text
弹出 1
加入 2, 3
当前层节点数 = 1
```

### 第 2 层

```text
弹出 2，加入 4, 5
弹出 3，加入 6
当前层节点数 = 2
```

### 第 3 层

```text
弹出 4
弹出 5
弹出 6
当前层节点数 = 3
```

因此：

```text
最大宽度 = max(1, 2, 3) = 3
```

## 解题思路
### 1. 方法一：使用 `Map`
思路最直接：

- 队列里照常做 BFS
- 额外准备一个 `Map<Node, Integer>`
- 记录每个节点属于第几层
- 每弹出一个节点，就看它是否还在当前层

具体做法：

1. 根节点入队，且 `levelMap.put(root, 1)`
2. 维护当前正在统计的层号 `curLevel`
3. 维护当前层节点数 `curLevelNodes`
4. 弹出节点时，查看它的层号：
5. 如果它还属于 `curLevel`，当前层节点数加一
6. 如果它已经来到下一层，说明上一层结束了：
7. 先更新最大宽度，再切换到新层重新计数

伪代码：

```java
queue.offer(head);
levelMap.put(head, 1);
int curLevel = 1;
int curLevelNodes = 0;
int max = 0;

while (!queue.isEmpty()) {
    Node cur = queue.poll();
    int curNodeLevel = levelMap.get(cur);

    if (cur.left != null) {
        levelMap.put(cur.left, curNodeLevel + 1);
        queue.offer(cur.left);
    }
    if (cur.right != null) {
        levelMap.put(cur.right, curNodeLevel + 1);
        queue.offer(cur.right);
    }

    if (curNodeLevel == curLevel) {
        curLevelNodes++;
    } else {
        max = Math.max(max, curLevelNodes);
        curLevel++;
        curLevelNodes = 1;
    }
}
max = Math.max(max, curLevelNodes);
```

这种写法好处是思路非常清楚，坏处是多用了一个 `Map`。

### 2. 方法二：不使用 `Map`
不用 `Map` 的关键点是：

```text
不用记录“每个节点在哪一层”
改为记录“当前层最后一个节点是谁”
```

需要两个指针：

- `curEnd`：当前层最右侧、最后处理完的那个节点
- `nextEnd`：下一层目前已经发现的最右侧节点

处理流程：

1. 根节点入队
2. 一开始 `curEnd = head`
3. 每弹出一个节点，当前层节点数加一
4. 如果它有左孩子，就入队，同时更新 `nextEnd = cur.left`
5. 如果它有右孩子，就入队，同时更新 `nextEnd = cur.right`
6. 一旦当前弹出的节点就是 `curEnd`，说明当前层结束了
7. 用当前层节点数更新答案
8. 然后把 `curEnd = nextEnd`，开始统计下一层

伪代码：

```java
queue.offer(head);
Node curEnd = head;
Node nextEnd = null;
int curLevelNodes = 0;
int max = 0;

while (!queue.isEmpty()) {
    Node cur = queue.poll();
    if (cur.left != null) {
        queue.offer(cur.left);
        nextEnd = cur.left;
    }
    if (cur.right != null) {
        queue.offer(cur.right);
        nextEnd = cur.right;
    }
    curLevelNodes++;
    if (cur == curEnd) {
        max = Math.max(max, curLevelNodes);
        curLevelNodes = 0;
        curEnd = nextEnd;
    }
}
```

这就是原始笔记中“不使用 Map”的真正含义。

### 3. 为什么这两种方法都对
无论是：

- 直接记录节点层号

还是：

- 记录当前层的结束边界

本质上都是在回答同一个问题：

```text
我怎么知道“这一层已经处理完了”
```

只要这个问题解决了，每层节点数就能统计出来，最大宽度也就出来了。

### 4. 两种方法怎么选
- 刚学时：先写 `Map` 版，更容易理解分层逻辑。
- 面试时：优先写不用 `Map` 版，空间更省，也更体现你对 BFS 分层边界的掌握。
- 如果只是像仓库代码那样直接在每轮开始用 `queue.size()`，其实也是“不额外用 Map”的做法，而且更简洁。

## 典型例子
还是用这棵树：

```text
    1
   / \
  2   3
 / \   \
4   5   6
```

### 用 `Map` 理解

```text
1 在第 1 层
2、3 在第 2 层
4、5、6 在第 3 层
```

统计结果：

```text
第 1 层：1
第 2 层：2
第 3 层：3
最大值：3
```

### 用层结束节点理解

```text
开始：curEnd = 1
处理完 1 后，当前层结束，下一层最后节点变成 3

开始第二层：curEnd = 3
处理完 2、3 后，当前层结束，下一层最后节点变成 6

开始第三层：curEnd = 6
处理完 4、5、6 后，当前层结束
```

所以每层大小依次是：

```text
1, 2, 3
```

答案仍然是：

```text
3
```

## 复杂度
### 使用 `Map`
- 时间复杂度：`O(N)`
- 空间复杂度：`O(N)`

### 不使用 `Map`
- 时间复杂度：`O(N)`
- 空间复杂度：`O(W)`，最坏为 `O(N)`

其中：

- `N` 是节点数
- `W` 是某一层上的最大节点数，也就是队列最大长度

## 易错点
- 不要把这题和“带空位的最大宽度”混掉，这里统计的是每层真实节点数。
- `Map` 版最后循环结束后，还要再用最后一层的节点数更新一次答案。
- 不用 `Map` 版里，`nextEnd` 应该始终更新为当前已加入队列的最后一个孩子节点。
- 不用 `Map` 版只有在 `cur == curEnd` 时，才能宣布当前层结束。
- 空树要返回 `0`。

## 代码 / 伪代码
仓库中的对应实现：

- [BinaryTreeTravelBF.java](../../../solutions-java/practice/algs03_tree/BinaryTreeTravelBF.java)

仓库代码使用的是最简洁的“不额外用 `Map`”思路：每轮开始先记当前层大小 `levelSize`，然后恰好处理这么多个节点。

核心代码如下：

```java
Queue<TreeNode> queue = new LinkedList<>();
queue.offer(root);
int maxWidth = 0;

while (!queue.isEmpty()) {
    int levelSize = queue.size();
    maxWidth = Math.max(maxWidth, levelSize);

    for (int i = 0; i < levelSize; i++) {
        TreeNode currentNode = queue.poll();
        if (currentNode.left != null) {
            queue.offer(currentNode.left);
        }
        if (currentNode.right != null) {
            queue.offer(currentNode.right);
        }
    }
}
```

虽然原始笔记里专门提了 `Map` 版和不用 `Map` 版，但仓库保留的是更简洁的按层统计版，本质上和“不使用 `Map`”是同一路思路。

## 记忆点
- 最大宽度题 = 按层遍历题上加一个“每层计数”。
- `Map` 版记“节点在哪层”，不用 `Map` 版记“这一层到哪结束”。
- 课程版宽度只算真实节点，不算中间空位。
- 一旦能判断层结束，宽度题就做完一半了。
