## 一句话结论
二叉树的非递归遍历，本质上就是：

```text
不用系统递归栈
改成自己手动用栈保存“还没处理完的现场”
```

先序最好理解，中序最值得掌握，后序最容易绕。  
如果面试现场紧张，后序优先写“双栈版”，更稳。

## 先记结论
第一次看这篇，建议先把下面这几句记住：

```text
先序：弹出就处理，先压右再压左
中序：一路压左，弹出后处理，再转向右
后序：左、右都处理完，最后才处理自己
```

再换一种更容易记的说法：

```text
先序 = 根先出
中序 = 根夹在中间
后序 = 根最后出
```

## 核心知识点
- 递归遍历依赖系统调用栈，非递归遍历依赖我们自己维护的栈。
- 先序的关键是：为什么要“先压右，再压左”。
- 中序的关键是：为什么必须“一路向左压栈到底”。
- 后序的关键是：节点必须在左右子树都处理完之后才能处理自己。
- 后序有双栈法和单栈法：双栈更好理解，单栈更像真的在模拟递归返回。

## 图片转写 / 题意还原
这题真正要解决的是：

```text
不使用递归函数
仍然完成二叉树的先序 / 中序 / 后序遍历
```

也就是说，我们不能再依赖系统自动帮我们保存这些信息：

- 当前走到哪个节点
- 左子树处理到哪了
- 右子树处理到哪了
- 回来之后下一步该干什么

非递归写法的核心，就是把这些“系统自动保存的现场”改成自己用栈来保存。

如果说上一题讲的是：

```text
递归序是怎么来的
```

那这一题讲的就是：

```text
递归能做的事
不用递归，其实也能做
只不过要自己保管现场
```

## 图解
假设二叉树如下：

```text
        1
      /   \
     2     3
    / \   / \
   4   5 6   7
```

为了让过程更顺，下面都统一用这套记号：

```text
栈：[底 ... 顶]
输出：已经打印出来的结果
```

### 1. 先序图解：为什么一定是“先压右，再压左”
先序要求：

```text
根 -> 左 -> 右
```

而栈是后进先出，所以想让左孩子先处理，就必须让左孩子后进栈。  
也就是：

```text
先压右，再压左
```

下面直接顺着过程看：

```text
初始：
栈：[1]
输出：

第 1 步：
弹出 1，打印 1
把 3 压栈，再把 2 压栈
栈：[3, 2]
输出：1

第 2 步：
弹出 2，打印 2
把 5 压栈，再把 4 压栈
栈：[3, 5, 4]
输出：1 2

第 3 步：
弹出 4，打印 4
4 没有孩子
栈：[3, 5]
输出：1 2 4

第 4 步：
弹出 5，打印 5
5 没有孩子
栈：[3]
输出：1 2 4 5

第 5 步：
弹出 3，打印 3
把 7 压栈，再把 6 压栈
栈：[7, 6]
输出：1 2 4 5 3

第 6 步：
弹出 6，打印 6
栈：[7]
输出：1 2 4 5 3 6

第 7 步：
弹出 7，打印 7
栈：[]
输出：1 2 4 5 3 6 7
```

所以先序最该记住的不是整段代码，而是这句：

```text
弹出就处理
先压右，再压左
```

### 2. 中序图解：为什么必须“一路压左”
中序要求：

```text
左 -> 根 -> 右
```

所以一个节点什么时候能打印？

```text
只有它左边全部处理完了，才能打印它自己
```

于是过程一定是：

```text
能往左就一直往左
左到底了，再回头打印
打印完再转向右边
```

下面直接跟过程：

```text
初始：
head = 1
栈：[]
输出：

第 1 步：一路压左
压 1，head -> 2
栈：[1]

压 2，head -> 4
栈：[1, 2]

压 4，head -> null
栈：[1, 2, 4]

第 2 步：左边到底，开始回退
弹出 4，打印 4
head 转向 4.right = null
栈：[1, 2]
输出：4

第 3 步：
head 为 null，继续弹栈
弹出 2，打印 2
head 转向 2.right = 5
栈：[1]
输出：4 2

第 4 步：处理 5 这棵右子树
压 5，head -> null
栈：[1, 5]

弹出 5，打印 5
head -> 5.right = null
栈：[1]
输出：4 2 5

第 5 步：
弹出 1，打印 1
head 转向 1.right = 3
栈：[]
输出：4 2 5 1

第 6 步：进入右子树 3
压 3，head -> 6
栈：[3]

压 6，head -> null
栈：[3, 6]

弹出 6，打印 6
栈：[3]
输出：4 2 5 1 6

弹出 3，打印 3
head -> 7
栈：[]
输出：4 2 5 1 6 3

压 7，head -> null
栈：[7]

弹出 7，打印 7
栈：[]
输出：4 2 5 1 6 3 7
```

所以中序最该记住的是这句：

```text
一路压左
弹出后处理
再转向右
```

### 3. 后序图解：为什么双栈最容易理解
后序要求：

```text
左 -> 右 -> 根
```

直接做比较绕，所以最稳的想法是：

```text
先故意做出：根 -> 右 -> 左
再把结果整体反过来
```

这里我们用两个栈：

- `stack`：正常遍历用
- `collect`：收集“根 -> 右 -> 左”

注意这里的关键操作是：

```text
从 stack 弹出一个节点后
先压左，再压右
```

因为栈是后进先出，所以这样下次会先弹出右边，得到“根 -> 右 -> 左”。

完整过程如下：

```text
初始：
stack：[1]
collect：[]

第 1 步：
弹出 1，放入 collect
压 2，再压 3
stack：[2, 3]
collect：[1]

第 2 步：
弹出 3，放入 collect
压 6，再压 7
stack：[2, 6, 7]
collect：[1, 3]

第 3 步：
弹出 7，放入 collect
stack：[2, 6]
collect：[1, 3, 7]

第 4 步：
弹出 6，放入 collect
stack：[2]
collect：[1, 3, 7, 6]

第 5 步：
弹出 2，放入 collect
压 4，再压 5
stack：[4, 5]
collect：[1, 3, 7, 6, 2]

第 6 步：
弹出 5，放入 collect
stack：[4]
collect：[1, 3, 7, 6, 2, 5]

第 7 步：
弹出 4，放入 collect
stack：[]
collect：[1, 3, 7, 6, 2, 5, 4]
```

这时 `collect` 从栈顶往外弹出的顺序就是：

```text
4 5 2 6 7 3 1
```

也就是标准后序：

```text
左 -> 右 -> 根
```

所以后序双栈法最值得记住的是：

```text
先造出“根 -> 右 -> 左”
再整体逆过来
```

### 4. 三种遍历放在一起看
如果你老觉得三个模板容易混，可以直接对照记：

```text
先序：
弹出就打印
先压右，再压左

中序：
一路压左
弹出后打印
再转向右

后序（双栈）：
第一栈先得到 根 -> 右 -> 左
第二栈倒过来变成 左 -> 右 -> 根
```

## 解题思路
### 1. 先序：弹出就处理
先序和递归版的关系是：

```text
来到当前节点就处理
然后再去左、去右
```

改成栈以后：

1. 根节点入栈
2. 每次弹出一个节点就立刻处理
3. 先压右孩子，再压左孩子

伪代码：

```java
stack.push(head);
while (!stack.isEmpty()) {
    cur = stack.pop();
    print(cur);
    if (cur.right != null) stack.push(cur.right);
    if (cur.left != null) stack.push(cur.left);
}
```

### 2. 中序：左边界全部入栈
中序递归的意思是：

```text
左边处理完 -> 才轮到根 -> 再去右边
```

所以非递归时要显式保存“回来的路”：

1. 只要 `head != null`，就一路压栈并向左走
2. 走到空后，弹栈顶并处理
3. 再转向这个节点的右子树

伪代码：

```java
while (!stack.isEmpty() || head != null) {
    if (head != null) {
        stack.push(head);
        head = head.left;
    } else {
        head = stack.pop();
        print(head);
        head = head.right;
    }
}
```

### 3. 后序双栈：先造一个“反后序”
后序是：

```text
左 -> 右 -> 根
```

如果我们先故意得到：

```text
根 -> 右 -> 左
```

再整体逆过来，就正好是后序。

具体做法：

1. 第一个栈 `stack` 负责遍历
2. 第二个栈 `collect` 负责收集
3. `stack` 每弹出一个节点，就放进 `collect`
4. 然后先压左、再压右
5. 这样 `stack` 实际弹出顺序就是“根 -> 右 -> 左”
6. 最后 `collect` 再弹出，就是“左 -> 右 -> 根”

伪代码：

```java
stack.push(head);
while (!stack.isEmpty()) {
    cur = stack.pop();
    collect.push(cur);
    if (cur.left != null) stack.push(cur.left);
    if (cur.right != null) stack.push(cur.right);
}
while (!collect.isEmpty()) {
    print(collect.pop());
}
```

### 4. 后序单栈：判断左右子树是否都处理完
单栈法更像真的在模拟递归返回。

核心思想是：

- 栈顶节点 `cur` 表示“当前准备处理的节点”
- 如果它的左树还没处理，就先处理左树
- 否则如果右树还没处理，就先处理右树
- 否则说明左右都结束了，当前节点才可以弹出并处理

为了知道某棵子树是不是已经处理完，通常要额外记录：

```text
lastVisited = 上一次真正处理并弹出的节点
```

这个写法更难，但一旦想明白，会更接近你对递归本质的理解。

## 完整过程示例
还是用这棵树：

```text
    1
   / \
  2   3
```

### 先序非递归

```text
栈：[1]
弹 1，打印 1，压 3，再压 2
栈：[3, 2]
弹 2，打印 2
栈：[3]
弹 3，打印 3
```

结果：

```text
1 2 3
```

### 中序非递归

```text
一路压左：1 -> 2
弹 2，打印 2
回到 1，打印 1
转向 3，打印 3
```

结果：

```text
2 1 3
```

### 后序非递归
双栈法先得到：

```text
1 3 2
```

再逆序得到：

```text
2 3 1
```

## 为什么这题容易绕
很多人不是不会写代码，而是脑子里没把这三件事分开：

```text
先序：什么时候打印？
中序：什么时候有资格打印？
后序：什么时候左右都算处理完了？
```

只要把这三件事分清楚，模板就不再是死背，而是顺着逻辑能推出来。

## 复杂度
### 先序 / 中序 / 后序单栈
- 时间复杂度：`O(N)`
- 空间复杂度：`O(H)`，最坏为 `O(N)`

### 后序双栈
- 时间复杂度：`O(N)`
- 空间复杂度：`O(N)`

其中：

- `N` 是节点数
- `H` 是树高

## 易错点
- 先序非递归一定是“先压右，再压左”，否则顺序会反。
- 中序非递归的循环条件必须是 `stack` 不空或 `head` 不空，少一个都会漏节点。
- 中序不是“弹出就往左右都走”，而是“弹出后处理，再只转向右子树”。
- 后序双栈里，第一个栈一定是先压左、再压右，这样弹出顺序才是“根 -> 右 -> 左”。
- 后序单栈最容易错在“右子树是否已经处理过”的判断条件。
- 面试里如果只要求写出后序非递归，优先写双栈版，更稳。

## 代码 / 伪代码
仓库中的对应实现：

- [BinaryTreeTraversalIteration.java](../../../solutions-java/practice/algs03_tree/BinaryTreeTraversalIteration.java)

这份代码里包含了 4 种写法：

- 先序非递归：`preOrder`
- 中序非递归：`inOrder`
- 后序非递归双栈版：`posOrderTwoStacks`
- 后序非递归单栈版：`posOrderOneStack`

先序核心代码：

```java
Stack<TreeNode> stack = new Stack<>();
stack.push(head);
while (!stack.isEmpty()) {
    head = stack.pop();
    System.out.print(head.val + " ");
    if (head.right != null) {
        stack.push(head.right);
    }
    if (head.left != null) {
        stack.push(head.left);
    }
}
```

中序核心代码：

```java
Stack<TreeNode> stack = new Stack<>();
while (!stack.isEmpty() || head != null) {
    if (head != null) {
        stack.push(head);
        head = head.left;
    } else {
        head = stack.pop();
        System.out.print(head.val + " ");
        head = head.right;
    }
}
```

后序双栈核心代码：

```java
Stack<TreeNode> stack = new Stack<>();
Stack<TreeNode> collect = new Stack<>();
stack.push(head);
while (!stack.isEmpty()) {
    head = stack.pop();
    collect.push(head);
    if (head.left != null) {
        stack.push(head.left);
    }
    if (head.right != null) {
        stack.push(head.right);
    }
}
while (!collect.isEmpty()) {
    System.out.print(collect.pop().val + " ");
}
```

## 记忆点
- 非递归遍历 = 用自己维护的栈，替代系统递归栈。
- 先序：弹出就处理，先压右再压左。
- 中序：一路压左，弹出后处理，再转右。
- 后序：最难，双栈最稳，单栈最像递归。
