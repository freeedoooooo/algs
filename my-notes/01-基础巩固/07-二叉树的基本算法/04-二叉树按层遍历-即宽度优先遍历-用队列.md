## 一句话结论
二叉树按层遍历，本质上就是二叉树上的宽度优先搜索（BFS），核心数据结构是队列。
谁先进入队列，谁就先被处理，所以它天然适合一层一层、从左到右地遍历整棵树。

## 核心知识点
- 按层遍历的顺序是：先根节点，再下一层，再下一层。
- 它和先序 / 中序 / 后序不同，不是深度优先，而是宽度优先。
- 队列的特点是先进先出，正好对应“上一层先处理完，再进入下一层”。
- 每弹出一个节点时，把它的左孩子、右孩子按顺序加入队列，就能维持层序。
- 如果再配合 `queue.size()`，就能很自然地区分“当前层”和“下一层”。

## 图解
假设二叉树如下：

```text
        1
      /   \
     2     3
    / \   / \
   4   5 6   7
```

初始化时，把根节点放入队列：

```text
队列：[1]
```

### 第 1 层
弹出队头 `1`，处理它，再把它的左、右孩子加入队列：

```text
弹出：1
加入：2, 3
队列变成：[2, 3]
```

### 第 2 层
依次弹出 `2`、`3`：

```text
弹出：2
加入：4, 5
队列：[3, 4, 5]

弹出：3
加入：6, 7
队列：[4, 5, 6, 7]
```

### 第 3 层
再依次弹出：

```text
4 5 6 7
```

最终按层遍历结果就是：

```text
1 2 3 4 5 6 7
```

可以把整个过程记成一句话：

```text
弹出当前节点
加入它的孩子
直到队列为空
```

## 解题思路
### 1. 为什么必须用队列
按层遍历要求的是：

```text
先访问离根近的
再访问离根远的
同一层从左到右
```

而队列正好满足这个顺序：

- 先入队的节点先出队
- 某一层节点在前面
- 它们的孩子统一排到后面

所以等前一层全部出队后，队列头部自然就轮到下一层。

### 2. 基本流程
标准过程非常固定：

1. 如果根节点为空，直接结束
2. 根节点入队
3. 当队列不为空时，循环处理：
4. 弹出一个节点并处理
5. 如果它有左孩子，左孩子入队
6. 如果它有右孩子，右孩子入队

伪代码：

```java
if (root == null) {
    return;
}
queue.offer(root);
while (!queue.isEmpty()) {
    TreeNode cur = queue.poll();
    print(cur.val);
    if (cur.left != null) {
        queue.offer(cur.left);
    }
    if (cur.right != null) {
        queue.offer(cur.right);
    }
}
```

### 3. 如何做到“按层”处理
如果只是单纯打印出队顺序，上面那段代码已经够了。

但如果你还想知道：

- 当前正在处理哪一层
- 每一层有多少个节点
- 每层单独收集成一个列表

那就要在每轮开始时先记录：

```java
int levelSize = queue.size();
```

这个 `levelSize` 的含义是：

```text
此时队列里正好装着“当前层”的所有节点
```

然后只循环 `levelSize` 次，就能保证这一轮只处理当前层，不会把下一层混进来。

### 4. 为什么它是后续很多题的基础
按层遍历是一个基础模版，后面很多题都只是加一点信息：

- 求最大宽度：每层统计节点数，取最大值
- 层序遍历返回二维数组：每层开一个 `list`
- 锯齿层序遍历：奇偶层决定加入顺序
- 右视图：每层取最后一个节点

所以这一题不是孤立题，而是 BFS 模版题。

## 典型例子
还是用这棵树：

```text
    1
   / \
  2   3
 / \
4   5
```

队列变化过程：

```text
初始：[1]

弹出 1，打印 1
加入 2, 3
队列：[2, 3]

弹出 2，打印 2
加入 4, 5
队列：[3, 4, 5]

弹出 3，打印 3
队列：[4, 5]

弹出 4，打印 4
队列：[5]

弹出 5，打印 5
队列：[]
```

结果：

```text
1 2 3 4 5
```

如果按层分组看，就是：

```text
第1层：1
第2层：2 3
第3层：4 5
```

## 复杂度
- 时间复杂度：`O(N)`
- 空间复杂度：`O(W)`，最坏为 `O(N)`

其中：

- `N` 是节点数
- `W` 是某一层上的最大节点数，也就是队列可能达到的最大长度

## 易错点
- 这题用的是队列，不是栈。栈会把它做成深度优先。
- 每次都是先弹出节点，再把它的左右孩子入队，不要顺序写反。
- 左孩子要先于右孩子入队，否则同层的左右顺序会变。
- 如果要分层处理，必须先记下当前层的 `queue.size()`，不能一边遍历一边直接用动态变化后的队列长度。
- 空树要单独处理，否则一开始入队就会出问题。

## 代码 / 伪代码
仓库中的对应实现：

- [BinaryTreeTravelBF.java](../../../solutions-java/practice/algs03_tree/BinaryTreeTravelBF.java)

这份代码里虽然函数名写的是 `maxWidth`，但它的主体就是标准的按层遍历框架：

```java
Queue<TreeNode> queue = new LinkedList<>();
queue.offer(root);

while (!queue.isEmpty()) {
    int levelSize = queue.size();
    for (int i = 0; i < levelSize; i++) {
        TreeNode currentNode = queue.poll();
        if (currentNode.left != null) {
            queue.offer(currentNode.left);
        }
        if (currentNode.right != null) {
            queue.offer(currentNode.right);
        }
    }
}
```

如果只是单纯做按层遍历 / 宽度优先遍历，可以把“统计宽度”的那部分拿掉，只保留队列推进过程。

## 记忆点
- 按层遍历 = 二叉树上的 BFS。
- BFS 的核心结构 = 队列，先进先出。
- 弹一个，进两个，队列空了就结束。
- 想分层，就先记当前层节点数 `queue.size()`。
