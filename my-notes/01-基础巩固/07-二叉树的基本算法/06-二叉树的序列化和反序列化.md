## 一句话结论
二叉树序列化的本质，是把一棵树完整变成可以保存和传输的线性序列；反序列化则是根据这份序列把原树还原回来。
课程里最常见有两类做法：递归序列化和按层序列化，但一定要把空节点也记录下来，否则树结构会丢失。

## 核心知识点
- 只记录节点值不够，必须连空位置一起记录，才能唯一还原树结构。
- 先序和后序都可以配合空节点完成序列化与反序列化。
- 中序遍历即使记录空节点，通常也不适合作为唯一反序列化依据。
- 按层遍历也能完成序列化与反序列化，核心数据结构是队列。
- 反序列化本质上是在回答：当前读到的序列，如何一步步恢复左右子树关系。

## 图片转写 / 题意还原
原始笔记只留下了两个提示：

```text
方法1，先/中/后序递归

方法2，按层遍历
```

这两个提示基本已经给出了这题的框架。

这题真正要解决的是：

- 怎么把树变成线性结构
- 怎么保证线性结构还能唯一恢复原树
- 为什么有些遍历顺序可以恢复，有些不行

关键点就在于：

```text
不仅要记录节点值
还要记录空节点
```

补成完整题目说明后，它讲的是：

- 给定一棵二叉树
- 需要把它转成某种线性表示，方便保存、传输或比较
- 这个过程叫序列化
- 然后再根据这份线性表示，把原树完整恢复出来
- 这个过程叫反序列化

题目真正的要求不是“随便遍历一下”，而是：

```text
序列化后的结果
必须足够完整
以便反序列化时能唯一还原原树结构
```

如果不记录空节点，比如只有：

```text
1 2 3
```

那你根本不知道它原来是：

```text
  1
 / \
2   3
```

还是：

```text
1
 \
  2
   \
    3
```

## 图解
假设二叉树如下：

```text
        1
      /   \
     2     3
    / \   /
   4   5 6
```

### 1. 先序序列化
先序顺序是：

```text
根 -> 左 -> 右
```

如果把空节点也记为 `null`，那么先序序列可以写成：

```text
1, 2, 4, null, null, 5, null, null, 3, 6, null, null, null
```

它的含义是：

```text
读到 1：建根节点
读到 2：建 1 的左子树根
读到 4：建 2 的左子树根
读到 null：4 的左孩子为空
读到 null：4 的右孩子为空
...
```

### 2. 为什么中序通常不适合作为反序列化依据
中序顺序是：

```text
左 -> 根 -> 右
```

即使你把空节点写进去，恢复时也很难确定“当前根节点应该在哪里切分左右子树”。

也就是说：

```text
先序 / 后序
能天然告诉你谁先成为根

中序
只能告诉你左右相对位置
不能天然告诉你根的构建顺序
```

所以课程里通常会强调：

- 中序可以参与“打印”和“遍历”
- 但不适合作为单独的序列化反序列化方案

### 3. 按层序列化
如果用按层遍历，把每一层节点依次写下来，同时把空位也补上：

```text
1, 2, 3, 4, 5, 6, null, null, null, null, null, null, null
```

那么反序列化时就可以：

1. 先拿第一个值建根节点
2. 然后按队列顺序，为每个节点依次分配左孩子和右孩子

这和按层遍历时“父节点先出队，孩子后入队”的顺序是完全一致的。

## 解题思路
### 1. 递归版：先序序列化
最常见的写法是先序，因为它天然先遇到根节点。

做法：

1. 如果当前节点为空，直接把 `null` 放进序列
2. 否则先记录当前节点值
3. 再递归记录左子树
4. 再递归记录右子树

伪代码：

```java
void preSerial(Node head, Queue<String> ans) {
    if (head == null) {
        ans.offer(null);
    } else {
        ans.offer(String.valueOf(head.value));
        preSerial(head.left, ans);
        preSerial(head.right, ans);
    }
}
```

### 2. 递归版：先序反序列化
反序列化时，只需要严格按序列顺序消费数据：

1. 弹出一个值
2. 如果这个值是 `null`，返回空节点
3. 如果不是 `null`，建立当前节点
4. 然后递归构造它的左子树
5. 再递归构造它的右子树

伪代码：

```java
Node buildByPre(Queue<String> queue) {
    String value = queue.poll();
    if (value == null) {
        return null;
    }
    Node head = new Node(Integer.valueOf(value));
    head.left = buildByPre(queue);
    head.right = buildByPre(queue);
    return head;
}
```

这其实就是序列化过程的逆过程。

### 3. 为什么中序不能单独还原
这一点很重要，也是这题最容易被忽略的地方。

原因不是“中序不能记录空节点”，而是：

```text
即便你有中序序列
也很难知道当前哪一个位置应该先作为根
```

先序和后序都天然包含“根先出现”或“根后出现”的构建信息：

- 先序：第一个就是当前子树的根
- 后序：最后一个就是当前子树的根

而中序没有这个优势。

所以课程里常说：

```text
中序可以序列化
但不能单独反序列化
```

### 4. 按层序列化
按层序列化很适合和 BFS 一起理解。

做法：

1. 根节点先入队
2. 每次弹出一个节点：
3. 如果节点为空，序列中记录 `null`
4. 如果节点非空，记录它的值
5. 再把它的左孩子和右孩子都放入队列

伪代码：

```java
queue.offer(head);
while (!queue.isEmpty()) {
    Node cur = queue.poll();
    if (cur == null) {
        ans.offer(null);
    } else {
        ans.offer(String.valueOf(cur.value));
        queue.offer(cur.left);
        queue.offer(cur.right);
    }
}
```

### 5. 按层反序列化
按层反序列化也是队列思路：

1. 先用第一个值建立根节点
2. 根节点入队
3. 每次弹出一个父节点
4. 再从序列里拿两个值，分别作为它的左孩子和右孩子
5. 新建出来的非空孩子继续入队

伪代码：

```java
Node buildByLevel(Queue<String> levelList) {
    Node head = generateNode(levelList.poll());
    Queue<Node> queue = new LinkedList<>();
    if (head != null) {
        queue.offer(head);
    }
    while (!queue.isEmpty()) {
        Node cur = queue.poll();
        cur.left = generateNode(levelList.poll());
        cur.right = generateNode(levelList.poll());
        if (cur.left != null) {
            queue.offer(cur.left);
        }
        if (cur.right != null) {
            queue.offer(cur.right);
        }
    }
    return head;
}
```

## 典型例子
用这棵树：

```text
    1
   / \
  2   3
```

### 先序序列化结果

```text
1, 2, null, null, 3, null, null
```

反序列化时：

```text
读到 1，建根
读到 2，建左树
读到 null，2 的左为空
读到 null，2 的右为空
读到 3，建右树
读到 null，3 的左为空
读到 null，3 的右为空
```

恢复结果就是原树。

### 按层序列化结果

```text
1, 2, 3, null, null, null, null
```

反序列化时：

```text
先建 1
再给 1 分配左孩子 2、右孩子 3
再给 2 分配左右空
再给 3 分配左右空
```

也能恢复出原树。

## 复杂度
### 先序 / 后序 / 按层序列化
- 时间复杂度：`O(N)`
- 空间复杂度：`O(N)`

### 反序列化
- 时间复杂度：`O(N)`
- 空间复杂度：`O(N)`

其中：

- `N` 是节点数
- 空间复杂度包含结果序列、递归栈或辅助队列

## 易错点
- 一定要记录空节点，否则无法唯一还原原树。
- 中序遍历不要单独拿来做反序列化依据，这题最容易在这里答错。
- 按层序列化时，空节点也要进入逻辑上的记录流程，不然结构会丢。
- 反序列化时，读取顺序必须和序列化顺序完全对应，不能乱消费序列。
- 处理空树时，要直接返回空结果或 `null`。

```java
void preSerial(Node head, Queue<String> ans) {
    if (head == null) {
        ans.offer(null);
        return;
    }
    ans.offer(String.valueOf(head.value));
    preSerial(head.left, ans);
    preSerial(head.right, ans);
}

Node buildByPre(Queue<String> queue) {
    String value = queue.poll();
    if (value == null) {
        return null;
    }
    Node head = new Node(Integer.valueOf(value));
    head.left = buildByPre(queue);
    head.right = buildByPre(queue);
    return head;
}
```

```java
Node head = generateNode(levelList.poll());
Queue<Node> queue = new LinkedList<>();
if (head != null) {

while (!queue.isEmpty()) {
    Node cur = queue.poll();
    cur.left = generateNode(levelList.poll());
    cur.right = generateNode(levelList.poll());

```

## 记忆点
- 序列化 = 把树变成线性序列；反序列化 = 把序列恢复成树。
- 不记录空节点，树结构就会丢。
- 先序 / 后序 / 按层都能做；中序不能单独还原。
- 先序反序列化本质是“按顺序递归消费序列”。
