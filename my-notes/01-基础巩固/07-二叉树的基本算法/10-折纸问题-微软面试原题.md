## 一句话结论
折纸问题表面上是在问折痕方向，实际上本质是一棵递归树的中序遍历题。
把每次折痕看成一个节点，根节点是 `down`，左子树全是 `down`，右子树全是 `up`，最后按中序遍历打印即可。

## 核心知识点
- 所有折纸都从下往上折，所以第一次展开后，根折痕永远是 `down`。
- 第 `N` 次折纸会形成一棵深度为 `N` 的“概念二叉树”。
- 这棵树里：
  - 每个节点的左孩子都是 `down`
  - 每个节点的右孩子都是 `up`
- 最终从上到下看到的折痕顺序，正好等于这棵树的中序遍历结果。
- 这题不是模拟真实折纸过程，而是把规律抽象成递归结构。

## 图片转写 / 题意还原
原图内容可以整理为：

- 请把一段纸条竖着放在桌子上，然后从纸条的下边向上方对折 1 次，压出折痕后展开。
- 此时折痕是凹下去的，也就是折痕突起的方向指向纸条的背面。
- 如果从纸条的下边向上方连续对折 2 次，压出折痕后展开，此时有三条折痕，从上到下依次是：下折痕、下折痕、上折痕。
- 给定一个输入参数 `N`，代表纸条都从下边向上方连续对折 `N` 次。
- 请从上到下打印所有折痕的方向。

原图给出的例子是：

```text
N = 1 -> down
N = 2 -> down down up
```

## 图解
### 1. N = 1
只折一次：

```text
down
```

### 2. N = 2
折两次后，从上到下是：

```text
down down up
```

可以把它看成一棵树：

```text
      down
     /    \
  down     up
```

对这棵树做中序遍历：

```text
左 -> 根 -> 右
```

得到：

```text
down down up
```

### 3. N = 3
树会继续长成：

```text
            down
         /        \
      down         up
     /   \       /   \
  down   up   down   up
```

中序遍历结果为：

```text
down down up down down up up
```

这就是三次折纸从上到下看到的顺序。

## 解题思路
### 1. 先看规律，不要硬模拟折纸
这题最容易走偏的地方，是去想：

- 真正折纸时纸条怎么翻
- 每一层折痕怎么映射回物理位置

其实没必要。

课程里最重要的观察是：

```text
每个折痕都可以看成一个节点
```

并且：

- 左边产生的折痕方向总是 `down`
- 右边产生的折痕方向总是 `up`

### 2. 构造概念二叉树
定义：

- 根节点是第一次折出来的折痕，方向是 `down`
- 每个节点的左孩子表示该折痕左侧进一步展开出的折痕，方向为 `down`
- 每个节点的右孩子表示该折痕右侧进一步展开出的折痕，方向为 `up`

折 `N` 次，就等于构造一棵深度为 `N` 的概念树。

### 3. 为什么要中序遍历
从上到下观察纸条展开后的折痕顺序，刚好对应这棵树的：

```text
中序遍历
```

也就是：

```text
先打印左边形成的折痕
再打印当前折痕
最后打印右边形成的折痕
```

这和“左 -> 根 -> 右”的递归顺序完全一致。

### 4. 递归函数怎么写
可以设计函数：

```text
process(level, N, down)
```

含义：

- 当前来到第 `level` 层
- 总共需要打印到第 `N` 层
- 当前节点折痕方向是 `down` 还是 `up`

递归过程：

1. 先递归左子树，左孩子方向固定是 `down`
2. 打印当前节点方向
3. 再递归右子树，右孩子方向固定是 `up`

伪代码：

```java
void process(int i, int N, boolean down) {
    if (i > N) {
        return;
    }
    process(i + 1, N, true);
    print(down ? "down" : "up");
    process(i + 1, N, false);
}
```

初始调用：

```java
process(1, N, true);
```

## 典型例子
### 例 1：N = 1

调用：

```text
process(1, 1, down)
```

输出：

```text
down
```

### 例 2：N = 2

递归树：

```text
      down
     /    \
  down     up
```

中序打印：

```text
down down up
```

### 例 3：N = 3

递归树：

```text
            down
         /        \
      down         up
     /   \       /   \
  down   up   down   up
```

中序打印：

```text
down down up down down up up
```

## 复杂度
- 时间复杂度：`O(2^N)`
- 空间复杂度：`O(N)`

因为：

- 折 `N` 次后总折痕数是 `2^N - 1`
- 递归深度是 `N`

## 易错点
- 这题不是层序遍历题，而是中序遍历题。
- 根节点方向一定是 `down`，不要写反。
- 左孩子永远是 `down`，右孩子永远是 `up`，这是规律核心。
- 不要去硬模拟物理折纸过程，直接抽象成递归树最稳。
- `N = 0` 时没有折痕，应该直接不打印。

## 记忆点
- 折纸题本质 = 一棵概念二叉树的中序遍历。
- 根是 `down`，左永远 `down`，右永远 `up`。
- 打印顺序不是层序，也不是先序，而是中序。
- 不要模拟物理折叠，直接抽象成递归。
