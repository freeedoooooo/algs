## 一句话结论
最小值栈的目标是让普通栈在 `push`、`pop`、`top` 之外，再支持 `getMin()`，并且所有操作都保持 `O(1)`。
实现关键是额外维护一个“同步变化的最小值栈”，让每一层都知道当前最小值是多少。

## 核心知识点
- 普通栈只能快速拿到栈顶，不能快速知道整个栈里的最小值。
- 最小值栈常用“双栈法”：一个数据栈，一个最小值栈。
- 最小值栈需要和数据栈保持同步进出。
- 只要同步得当，`getMin()` 就等价于取最小值栈的栈顶。

## 为什么不能每次都遍历找最小值
如果每次调用 `getMin()` 时都把整个栈扫一遍，时间复杂度就是：

```text
O(N)
```

这不符合题目对高效操作的要求。

所以要把“当前最小值”提前存好，而不是每次临时现算。

## 图解
最小值栈通常维护两份栈：

```text
data 栈：正常压入所有值
min  栈：同步记录“当前最小值”
```

例如依次压入：

```text
3, 5, 2, 4
```

过程像这样：

```text
push 3
data: [3]
min : [3]

push 5
data: [3, 5]
min : [3, 3]

push 2
data: [3, 5, 2]
min : [3, 3, 2]

push 4
data: [3, 5, 2, 4]
min : [3, 3, 2, 2]
```

所以：

```text
getMin() 永远只看 min 栈栈顶
```

## 解题思路
### 1. 准备两个栈
- `dataStack`：正常存放所有入栈元素
- `minStack`：记录“到当前这一层为止的最小值”

### 2. 入栈时同步更新最小值栈
假设要压入 `val`：

- `dataStack.push(val)`
- 如果 `minStack` 为空，直接把 `val` 压进去
- 否则比较 `val` 和当前最小值 `minStack.peek()`

如果 `val` 更小或相等：

- `minStack.push(val)`

如果 `val` 更大：

- 仍然要压入当前最小值 `minStack.peek()`

这样做的意义是：

- `minStack` 的每一层都和 `dataStack` 一一对应
- 当你弹出一层时，两个栈一起弹，最小值状态就会自动回退

### 3. 出栈时两个栈一起弹
因为两个栈是同步的，所以：

```java
dataStack.pop();
minStack.pop();
```

### 4. 获取最小值时直接看最小值栈栈顶

```java
return minStack.peek();
```

这就是 `O(1)` 的来源。

## 典型例子
依次压栈：

```text
3, 5, 2, 4
```

### 压入 3

```text
dataStack = [3]
minStack  = [3]
```

### 压入 5
当前最小值还是 `3`，所以：

```text
dataStack = [3, 5]
minStack  = [3, 3]
```

### 压入 2
`2` 比当前最小值 `3` 更小，所以：

```text
dataStack = [3, 5, 2]
minStack  = [3, 3, 2]
```

### 压入 4
当前最小值仍然是 `2`，所以：

```text
dataStack = [3, 5, 2, 4]
minStack  = [3, 3, 2, 2]
```

此时：

- `top()` 返回 `4`
- `getMin()` 返回 `2`

如果现在弹出一次：

```text
dataStack = [3, 5, 2]
minStack  = [3, 3, 2]
```

再弹一次：

```text
dataStack = [3, 5]
minStack  = [3, 3]
```

这时最小值会自动恢复成 `3`。

## 复杂度
- `push()`：`O(1)`
- `pop()`：`O(1)`
- `top()`：`O(1)`
- `getMin()`：`O(1)`
- 额外空间复杂度：`O(N)`

因为最小值栈会额外保存一份同步信息。

## 易错点
- 入栈时如果值更大，也不能不往 `minStack` 压值，否则两个栈层数会不同步。
- `pop()` 时两个栈必须同时弹出。
- 如果题目允许重复最小值，比较时要注意用 `<=`，不要只用 `<`。
- 空栈时调用 `top()` 或 `getMin()`，要按题目要求处理。

## 代码 / 伪代码
仓库里对应实现：

- [MinStack.java](../../../solutions-java/leetcode/p155_min_stack/MinStack.java)

核心代码如下：

```java
Stack<Integer> dataStack = new Stack<>();
Stack<Integer> minStack = new Stack<>();

void push(int val) {
    dataStack.push(val);
    if (minStack.isEmpty() || val <= minStack.peek()) {
        minStack.push(val);
    } else {
        minStack.push(minStack.peek());
    }
}

void pop() {
    dataStack.pop();
    minStack.pop();
}

int top() {
    return dataStack.peek();
}

int getMin() {
    return minStack.peek();
}
```

## 可选实现思路
还有一种写法是：

- `minStack` 只在遇到更小值时才压栈
- 弹出时如果出栈值等于当前最小值，再同步弹出 `minStack`

这种方式也可以，但判断条件会更多。
仓库当前采用的是“每层同步记录当前最小值”的写法，逻辑更直观。

## 速记方法
- 一个栈放数据。
- 一个栈放“到这一层为止的最小值”。
- 两个栈同进同出，最小值直接看 `minStack.peek()`。
