## 一句话结论
递归复杂度分析最关键的动作，是先把算法写成递推式。
如果递归满足标准分治形式，通常就可以直接套 Master 公式快速得到时间复杂度。

## 核心知识点
- 递归复杂度分析先写 `T(n)`，再谈结果。
- 标准形式通常是：`T(n) = a * T(n / b) + O(n^d)`。
- `a` 表示子问题个数，`b` 表示子问题缩小倍数，`O(n^d)` 表示当前层额外工作。
- 最常见的分析工具是 Master 公式。

## 递归复杂度公式
最常见的递推形式是：

```text
T(n) = a * T(n / b) + O(n^d)
```

含义分别是：

- `T(n)`：规模为 `n` 时的总时间复杂度
- `a`：每一层会分出多少个子问题
- `n / b`：每个子问题的规模
- `O(n^d)`：除递归调用外，这一层本身还要做多少事

这就是分治递归里最常见的“复杂度公式模板”。

## Master 公式结论
比较：

```text
log_b(a) 和 d
```

### 情况 1：`d < log_b(a)`

```text
T(n) = O(n^(log_b(a)))
```

说明递归向下展开后，子问题总成本更重。

### 情况 2：`d = log_b(a)`

```text
T(n) = O(n^d * log n)
```

说明每一层的总工作量差不多，层数贡献出一个 `log n`。

### 情况 3：`d > log_b(a)`

```text
T(n) = O(n^d)
```

说明当前层自己做的事已经占主导。

## 怎么快速套公式
遇到递归题时，可以按这个顺序走：

1. 写出递推式
2. 找出 `a`
3. 找出 `b`
4. 找出本层额外代价对应的 `d`
5. 比较 `d` 和 `log_b(a)`

## 典型例子
### 例1：归并排序

```text
T(n) = 2T(n / 2) + O(n)
```

- `a = 2`
- `b = 2`
- `d = 1`
- `log_2(2) = 1`

所以：

```text
T(n) = O(n log n)
```

### 例2：二分查找

```text
T(n) = T(n / 2) + O(1)
```

- `a = 1`
- `b = 2`
- `d = 0`
- `log_2(1) = 0`

所以：

```text
T(n) = O(log n)
```

### 例3：每层都要做平方级工作

```text
T(n) = 2T(n / 2) + O(n^2)
```

- `a = 2`
- `b = 2`
- `d = 2`
- `log_2(2) = 1`

因为 `d > log_b(a)`，所以：

```text
T(n) = O(n^2)
```

## 易错点
- 没先写出递推式，就直接猜复杂度。
- 把 `O(n^d)` 错当成整个递归总成本，它其实只是“当前层”的额外工作。
- 不是所有递归都能套 Master 公式，前提是子问题规模要比较规整。
- `log_b(a)` 的比较对象是 `d`，不要看错。
