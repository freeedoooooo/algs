## 一句话结论
删除链表中指定值的节点，核心就是把“前一个保留节点”连接到“当前待删节点的下一个节点”。
真正容易出错的地方不在中间节点，而在头节点本身就需要删除，并且可能连续删除多个头节点。

## 核心知识点
- 单链表删除节点，本质是改前驱节点的 `next` 指针。
- 如果删除的是头节点，没有前驱，所以要先单独处理头部。
- 删除中间节点时，常用双指针：`pre` 表示上一个保留节点，`cur` 表示当前扫描节点。
- 连续命中的删除场景要特别小心。

## 为什么头节点要单独处理
假设链表是：

```text
2 -> 2 -> 3 -> 4
```

如果要删除值为 `2` 的节点，那么新的头节点应该直接变成 `3`。

问题在于：

- 头节点没有前驱
- 所以不能像处理中间节点那样写 `pre.next = cur.next`

因此标准做法是：

1. 先把头节点一路往后推，跳过所有需要删除的头节点
2. 找到第一个不需要删除的节点，作为新头
3. 再处理后续部分

## 解题思路
### 1. 先找到新的头节点
只要当前头节点值等于目标值，就不断后移：

```text
while (head != null && head.val == target) {
    head = head.next;
}
```

执行完这一步后：

- 要么链表已经空了
- 要么 `head` 已经是第一个需要保留的节点

### 2. 用 `pre` 和 `cur` 扫描后续节点
令：

- `pre = head`
- `cur = head`

然后从左到右扫描：

- 如果 `cur.val == target`，说明当前节点要删除，让 `pre.next = cur.next`
- 如果 `cur.val != target`，说明当前节点要保留，让 `pre = cur`

无论删不删，`cur` 都继续往后走。

### 3. 为什么删除时 `pre` 不能前进
如果当前 `cur` 被删掉：

- `pre` 仍然应该停在上一个保留节点
- 因为后面可能还是连续多个待删节点

只有当 `cur` 被保留时，`pre` 才能真正向前推进。

## 典型例子
链表：

```text
1 -> 2 -> 2 -> 3 -> 2 -> 4
```

删除值 `2`。

### 第一步：处理头节点
头节点是 `1`，不需要删，所以新头还是 `1`。

### 第二步：开始扫描
初始：

- `pre = 1`
- `cur = 1`

往后走到第一个 `2`：

- `cur` 要删除
- 执行 `pre.next = cur.next`
- 链表临时变成：

```text
1 -> 2 -> 3 -> 2 -> 4
```

继续看新的 `cur`：

- 又是 `2`
- 再删一次

链表变成：

```text
1 -> 3 -> 2 -> 4
```

接着 `3` 保留：

- `pre = 3`

再遇到 `2`：

- 删除

最后结果：

```text
1 -> 3 -> 4
```

## 边界情况
- 空链表：直接返回 `null`
- 所有节点都等于目标值：最后返回 `null`
- 只有一个节点：要么删掉，要么原样返回
- 连续多个头节点都要删：必须在开头的 `while` 中处理干净

## 复杂度
- 时间复杂度：`O(N)`
- 空间复杂度：`O(1)`

因为每个节点最多只会被访问一次。

## 易错点
- 忘了先处理头节点，导致删除头节点时逻辑失效。
- 删除当前节点后，错误地把 `pre` 也往前移了。
- `cur` 前进顺序写乱，导致漏删或空指针。
- 删除全部节点后，没有正确返回 `null`。

## 代码 / 伪代码
仓库里没有单独拆出这个方法，但同一组链表基础题实现可参考：

- [ReverseLinkedList.java](../../../solutions-java/leetcode/p206_reverse_linked_list/ReverseLinkedList.java)
- [ListNode.java](../../../solutions-java/util/collection/ListNode.java)

可以按下面的标准写法实现：

```java
public static ListNode removeValue(ListNode head, int target) {
    while (head != null && head.val == target) {
        head = head.next;
    }
    if (head == null) {
        return null;
    }

    ListNode pre = head;
    ListNode cur = head.next;
    while (cur != null) {
        if (cur.val == target) {
            pre.next = cur.next;
        } else {
            pre = cur;
        }
        cur = cur.next;
    }
    return head;
}
```

## 速记方法
- 先删头，再删中间。
- `pre` 永远指向“上一个保留节点”。
- 当前节点被删时，`pre` 不动；当前节点保留时，`pre` 才前进。
