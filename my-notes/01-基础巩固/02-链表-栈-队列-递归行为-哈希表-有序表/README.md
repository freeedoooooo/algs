## 章节导读
这一章名字很长，内容也很杂，但实际上是在搭“数据结构基础工具箱”。

它包含了几条主线：

- 链表基础操作
- 栈和队列的实现与互相模拟
- 递归复杂度认知
- `Map` 的 key 语义

如果按知识结构压缩，可以这样看：

## 本章结论
- 链表题的核心不是背 API，而是搞清楚指针什么时候断、什么时候接。
- 栈和队列的本质区别在于“出入顺序”，不是实现形式。
- 用数组也能做出环形结构，用链表也能做出栈和队列，本质都是结构抽象。
- 递归复杂度分析要先写递推式，再决定能不能套公式。
- Java 里参数传递只有值传递，`Map` 的 key 判断最终看的是 `equals()` 和 `hashCode()`。

这一章前几题围绕单链表、双向链表展开，核心不是链表定义，而是：

- 如何安全反转
- 如何删除节点
- 如何维护头尾边界

链表题最常见的错误也都在这里：

- 头节点处理漏掉
- 指针断开顺序写错
- 尾节点没有正确更新

### 2. 栈和队列部分是在练“抽象结构”
从题目上看，栈和队列像是两个结构，但这一章真正想讲的是：

- 结构不重要，操作顺序才重要
- 同一种底层容器，可以模拟出不同抽象结构

所以这里会出现：

- 双端链表实现队列和栈
- 数组实现循环结构
- 最小值栈
- 队列实现栈、栈实现队列

这些题合起来训练的是：

```text
你能不能从“行为定义”出发
反推出底层该怎么维护数据
```

### 3. 递归和复杂度部分是在补“算法表达能力”
很多人会写递归，但不会分析复杂度。

这一章后两题想补的是：

- 怎么把递归过程写成递推式
- 什么情况下可以直接套 Master 公式
- 如果不能套公式，还可以怎么估计

这部分虽然不是具体题型，但它会影响你后面看：

- 归并排序
- 快速排序
- 树递归

### 4. `Map` 这一题是在补语言与结构认知
这一题单独放在这里，看起来有点跳，但其实非常有必要。

它解决的是很多初学者常混淆的问题：

- Java 到底是不是引用传递
- `Map` 的 key 到底按什么判断相等
- 基础类型、`String`、普通对象之间差别在哪里

如果这些点不清楚，后面做哈希表和自定义对象题目时，很容易出现“明明值一样却查不到”的困惑。

## 记忆点
- 链表重点在指针连接，不在语法。
- 栈和队列重点在顺序语义，不在底层容器。
- 会写递归，不等于会分析递归。
- `Map` 的 key 判断靠 `equals()` / `hashCode()`，不是“引用传递”这四个字。
