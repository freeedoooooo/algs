## 一句话结论
这类题的本质，不是造一个更实用的数据结构，而是考你能不能用一种“出入顺序”去模拟另一种“出入顺序”。
所以它更像结构思维训练题，面试里常见，但工程里通常不会这样写，因此叫“纯应试题目”很贴切。

## 核心知识点
- 栈是 `LIFO`：后进先出。
- 队列是 `FIFO`：先进先出。
- 用一个结构模拟另一个结构，核心是想办法“重排顺序”。
- 顺序一旦被重新组织出来，接口行为就能伪装成另一种结构。

## 先看本质区别
### 栈

```text
push: 新元素放到顶部
pop : 从顶部拿走
```

后进先出。

### 队列

```text
offer: 新元素放到尾部
poll : 从头部拿走
```

先进先出。

所以两者的矛盾点在于：

- 栈希望“最后来的先出来”
- 队列希望“最早来的先出来”

要模拟，就必须人为调整顺序。

## 解题思路
### 1. 用队列实现栈
目标：

- `push(x)`
- `pop()`
- `top()`

但队列只能：

- 从尾部加
- 从头部出

要把队列伪装成栈，就要让“最新加入的元素”跑到队头。

#### 核心办法
每次入栈 `x` 时：

1. 先把 `x` 加到队尾
2. 再把它前面的所有旧元素，依次从队头拿出，再放回队尾

这样一圈转完后：

- 新元素就会来到队头
- 队头位置就相当于“栈顶”

于是：

- `pop()` 直接 `poll()`
- `top()` 直接 `peek()`

仓库里的实现正是这个思路。

### 2. 用栈实现队列
目标：

- `push(x)`
- `pop()`
- `peek()`

但栈只能从栈顶进出。

队列却要求最早进入的元素先出来，所以需要“倒两次顺序”：

1. 把元素从栈1倒到栈2
2. 原本最早进入的元素就来到栈2顶部
3. 从栈2取出后
4. 再把剩余元素倒回去

这就是仓库里当前实现采用的直观版思路。

## 典型例子
### 例1：用队列实现栈
队列初始为空。

执行：

```text
push(1), push(2), push(3)
```

#### push(1)

```text
[1]
```

#### push(2)
先加入尾部：

```text
[1, 2]
```

再把旧元素 `1` 挪到后面：

```text
[2, 1]
```

#### push(3)
先加入尾部：

```text
[2, 1, 3]
```

再把前两个旧元素轮转到后面：

```text
[3, 2, 1]
```

此时队头就是栈顶。

所以：

- `pop()` 得到 `3`
- 再 `pop()` 得到 `2`

完全符合栈的行为。

### 例2：用栈实现队列
先依次入队：

```text
push(1), push(2), push(3)
```

如果都先压到 `stack1`：

```text
stack1 = [1, 2, 3]   // 3 在栈顶
stack2 = []
```

此时队列真正应该先出的是 `1`，但它在栈底。

于是把 `stack1` 全倒进 `stack2`：

```text
stack1 = []
stack2 = [3, 2, 1]   // 1 来到栈顶
```

现在：

- `stack2.pop()` 得到 `1`

这就满足队列的先进先出。

- [ImplementQueueUsingStacks.java](../../../solutions-java/leetcode/p232_implement_queue_using_stacks/ImplementQueueUsingStacks.java)

### 用队列实现栈
仓库采用的是“单队列轮转”写法：

- 每次 `push` 后把前面的旧元素全部轮转到后面
- 这样队头始终就是栈顶

这个写法很简洁。

### 用栈实现队列
仓库当前采用的是“每次 `pop/peek` 时整批搬运再搬回”的直观写法：

- 逻辑容易懂
- 但每次 `pop/peek` 都要来回倒栈

更优的常见写法是：

- 一个输入栈
- 一个输出栈
- 只有当输出栈为空时，才把输入栈整体倒进去

这样均摊复杂度会更好。

## 复杂度
### 用队列实现栈
- `push()`：`O(N)`
- `pop()`：`O(1)`
- `top()`：`O(1)`

### 用栈实现队列（仓库当前实现）
- `push()`：`O(1)`
- `pop()`：`O(N)`
- `peek()`：`O(N)`

## 易错点
- 用队列实现栈时，轮转元素数量应该是“新元素加入前的队列长度”。
- 用栈实现队列时，别忘了 `peek()` 之后要把元素放回去。
- `empty()` 判断要和底层结构当前状态保持一致。
- 这类题不要把“功能实现出来”和“复杂度最优”混为一谈，很多题只要求实现，未必要求最优。

## 代码 / 伪代码
### 用队列实现栈

```java
void push(int x) {
    int n = queue.size();
    queue.offer(x);
    for (int i = 0; i < n; i++) {
        queue.offer(queue.poll());
    }
}

int pop() {
    return queue.poll();
}
```

### 用栈实现队列

```java
void push(int x) {
    stack1.push(x);
}

int pop() {
    while (!stack1.isEmpty()) {
        stack2.push(stack1.pop());
    }
    int ans = stack2.pop();
    while (!stack2.isEmpty()) {
        stack1.push(stack2.pop());
    }
    return ans;
}
```

## 为什么说是“纯应试题目”
- 工程里通常会直接用现成队列或栈，不会特意互相模拟。
- 这类题更像是在检查你对数据结构行为的理解是否扎实。
- 一旦明白“顺序靠搬运来重排”，题目本身就不复杂了。

## 速记方法
- 队列变栈：让新元素转到最前面。
- 栈变队列：把最早元素倒到最上面。
- 本质都是“为了目标顺序，主动重排元素”。
