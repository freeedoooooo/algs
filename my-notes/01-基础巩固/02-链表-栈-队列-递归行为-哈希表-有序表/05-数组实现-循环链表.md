## 一句话结论
这题本质上讲的是“用数组模拟环形结构”，最典型的应用就是循环队列。
核心不在链表指针，而在于下标走到数组末尾后要回到开头继续使用，也就是把数组首尾接起来当成一个环。

## 核心知识点
- 顺序数组实现队列时，如果只让下标一直右移，前面空出来的位置会浪费掉。
- 环形数组的关键是“下标回绕”，即走到末尾后重新回到 0。
- 判断队列空/满时，最稳妥的方法通常是额外维护 `size`。
- 这类题常见名字叫“循环队列”“环形数组”“数组实现队列”。

## 概念澄清
题目名字写的是“循环链表”，但从仓库实现和常见面试语境看，这里更接近：

- 用数组实现一个循环队列
- 或者说，用数组模拟一个环形结构

也就是说：

- 逻辑上像一个环
- 底层上还是数组

不是“每个节点有 next 指针的那种真正循环链表”。

## 为什么普通数组队列会浪费空间
如果只用两个下标：

- `l` 表示队头
- `r` 表示队尾的下一个位置

那么像下面这样：

```text
offer 1, 2, 3
poll 1, 2
```

数组前面两个位置其实已经空出来了，但如果 `r` 只会一直往右走，就再也不能复用这些位置。

所以要把数组看成一个环：

- 到末尾了，就绕回到开头

## 解题思路
### 1. 用数组保存元素
定义：

- `queue[]`：底层数组
- `head`：当前队头位置
- `tail`：下一个入队位置
- `size`：当前队列中元素个数
- `limit`：数组容量

### 2. 入队时从 `tail` 放入
入队步骤：

1. 把值放到 `queue[tail]`
2. `tail` 向后移动一格
3. 如果走到数组末尾，下一个位置就回到 `0`
4. `size++`

代码上的移动方式通常写成：

```java
tail = (tail + 1 == limit) ? 0 : tail + 1;
```

### 3. 出队时从 `head` 取出
出队步骤：

1. 读取或删除 `queue[head]`
2. `head` 向后移动一格
3. 到末尾后同样回绕到 `0`
4. `size--`

### 4. 为什么要维护 `size`
如果只看 `head == tail`：

- 可能表示空
- 也可能表示满后绕了一圈又追上了

所以最清晰的做法是额外维护 `size`：

- `size == 0` 表示空
- `size == limit` 表示满

## 典型例子
假设数组容量为 `5`。

初始状态：

```text
head = 0, tail = 0, size = 0
```

### 第一步：入队 10、20、30
此时：

```text
queue = [10, 20, 30, _, _]
head = 0
tail = 3
size = 3
```

### 第二步：出队两个元素
出队 10、20 后：

```text
queue = [10, 20, 30, _, _]
head = 2
tail = 3
size = 1
```

注意：

- 前两个位置虽然数还在数组里
- 但逻辑上已经没用了

### 第三步：继续入队 40、50、60、70
当 `tail` 走到末尾后，会绕回前面：

```text
queue = [60, 70, 30, 40, 50]
head = 2
tail = 2
size = 5
```

此时虽然 `head == tail`，但它不是空，而是“满”。
这正是为什么需要 `size` 来辅助判断。

## 边界情况
- 容量为 1 时仍然适用
- 空队列不能出队
- 满队列不能再入队
- `Rear()` 的位置不是 `tail`，而是 `tail` 的前一个位置

## 复杂度
- 入队：`O(1)`
- 出队：`O(1)`
- 取队头 / 取队尾：`O(1)`
- 空间复杂度：`O(N)`

## 易错点
- 把 `tail` 当成“当前尾元素位置”，其实很多实现里它表示“下一个可插入位置”。
- `head == tail` 不能单独判断空还是满。
- 取队尾元素时，应该取 `tail - 1`，但还要注意 `tail == 0` 的回绕情况。
- 出队时逻辑删除即可，不一定非要真的把数组位置清 0。

## 代码 / 伪代码
仓库里有两个很好的对照实现：

- [MyQueue.java](../../../solutions-java/util/collection/MyQueue.java)
- [DesignCircularQueue.java](../../../solutions-java/leetcode/p622_design_circular_queue/DesignCircularQueue.java)

其中：

- `MyQueue.java` 是普通数组队列，下标只右移，不处理回绕
- `DesignCircularQueue.java` 是标准循环队列实现

循环队列核心框架如下：

```java
int[] queue;
int head;
int tail;
int size;
int limit;

boolean enQueue(int value) {
    if (size == limit) {
        return false;
    }
    queue[tail] = value;
    tail = (tail + 1 == limit) ? 0 : tail + 1;
    size++;
    return true;
}

boolean deQueue() {
    if (size == 0) {
        return false;
    }
    head = (head + 1 == limit) ? 0 : head + 1;
    size--;
    return true;
}
```

## 速记方法
- 数组不够灵活，所以要“首尾相接”。
- `head` 看谁该出。
- `tail` 看谁该进。
- `size` 负责区分空和满。
