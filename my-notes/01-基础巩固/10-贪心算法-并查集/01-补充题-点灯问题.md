## 一句话结论
点灯问题也是一道非常经典的贪心题：从左往右扫描，遇到居民点 `.` 时，优先把灯放在尽量靠右的位置，这样当前灯能覆盖更多后续位置。
它训练的核心不是模拟放灯，而是理解“这一盏灯怎样才能最值钱”。

## 核心知识点
- `X` 表示墙，不能放灯，也不需要被点亮。
- `.` 表示居民点，可以放灯，也必须被点亮。
- 一盏灯如果放在位置 `i`，可以照亮 `i-1、i、i+1`。
- 目标是让所有 `.` 都被照亮，且灯的数量最少。
- 贪心策略是从左到右扫描，遇到 `.` 时尽量把灯放在右边可放的位置。

## 题意说明
给定一个只由 `X` 和 `.` 组成的字符串：

- `X` 表示墙
- `.` 表示居民点

规则是：

- 墙不需要被点亮
- 居民点必须被点亮
- 灯只能放在 `.` 位置
- 一盏灯可以点亮自己以及左右相邻位置

要求返回：

```text
点亮所有居民点，最少需要多少盏灯
```

## 图解
先看一个最典型的小例子：

```text
. . X . .
0 1 2 3 4
```

如果从左往右扫：

- 来到位置 `0`，发现是 `.`
- 这时最优做法不是急着放在 `0`
- 而是优先看 `1` 能不能放

因为如果把灯放在 `1`，照亮范围是：

```text
0 1 2
```

这样第一盏灯就顺便把左边和中间都覆盖了。

后半段：

```text
X . .
  3 4
```

同理，再放一盏灯在 `4` 或 `3`，都能完成覆盖。  
最少答案是 `2`。

再看一个更能体现贪心方向的例子：

```text
...X..
012345
```

从位置 `0` 开始：

- 位置 `0` 是 `.`
- 位置 `1` 也是 `.`
- 那就把灯放在 `1`

此时会照亮：

```text
0 1 2
```

然后直接跳到位置 `3` 之后继续看：

```text
...X..
 ^^^
```

右边的 `..` 再放一盏灯即可。  
所以答案仍然是 `2`。

## 解题思路
### 1. 为什么遇到 `.` 要尽量往右放灯
因为一盏灯能照亮连续三个位置：

```text
i-1, i, i+1
```

如果你已经确定当前位置 `i` 是居民点，需要被覆盖，那么：

- 放在 `i`，只能保证覆盖 `i-1、i、i+1`
- 如果 `i+1` 也是居民点，放在 `i+1` 会更划算

因为放在更右边，能在覆盖当前位置的同时，尽量照顾更多后续位置。

### 2. 从左到右怎么做
扫描到位置 `i` 时：

- 如果是 `X`，直接跳过
- 如果是 `.`

这时一定要放灯，但放哪儿要分情况：

- 如果 `i+1` 越界，灯只能放在 `i`
- 如果 `i+1` 是 `X`，灯只能放在 `i`
- 如果 `i+1` 是 `.`，优先把灯放在 `i+1`

无论灯放在 `i` 还是 `i+1`，放完之后，当前这一盏灯最多能把接下来的若干位置一起覆盖，所以指针可以直接往后跳。

### 3. 为什么这是贪心
这题的贪心点在于：

```text
一旦决定当前位置必须被照亮
就让这一盏灯尽量覆盖更右边
```

这个局部最优不会吃亏，因为左边已经扫描过，不需要回头；而右边还没处理，覆盖得越远越有利。

## 过程示例
以字符串：

```text
.X...XX..
012345678
```

为例。

从左往右：

1. `i = 0`，是 `.`
   - `i+1 = 1` 是 `X`
   - 灯只能放在 `0`
   - 覆盖 `0、1`
   - 灯数变成 `1`

2. 跳到 `i = 2`
   - `2` 是 `.`
   - `3` 也是 `.`
   - 优先把灯放在 `3`
   - 覆盖 `2、3、4`
   - 灯数变成 `2`

3. 再跳到 `i = 5`
   - `5` 是 `X`，跳过
   - `6` 是 `X`，跳过

4. 到 `i = 7`
   - `7` 是 `.`
   - `8` 也是 `.`
   - 优先放在 `8`
   - 覆盖 `7、8、9`
   - 灯数变成 `3`

最终答案是 `3`。

## 复杂度
- 时间复杂度：`O(N)`
- 空间复杂度：`O(1)`

因为只需要从左到右扫描一遍。

## 易错点
- 不要把灯优先放在当前点，关键是“尽量往右放”。
- `X` 既不能放灯，也不需要点亮。
- 遇到 `.` 时，不是每次都看三格暴力枚举，按规则分类即可。
- 放完灯后，指针要直接跳过已经被这一盏灯覆盖的范围。

## 代码 / 伪代码
课程中常见伪代码如下：

```java
int minLight(String str) {
    char[] s = str.toCharArray();
    int i = 0;
    int light = 0;
    while (i < s.length) {
        if (s[i] == 'X') {
            i++;
        } else {
            light++;
            if (i + 1 == s.length) {
                break;
            } else if (s[i + 1] == 'X') {
                i += 2;
            } else {
                i += 3;
            }
        }
    }
    return light;
}
```

## 记忆点
- 点灯题看的是“这一盏灯怎么覆盖得更远”。
- 遇到 `.` 时，优先考虑把灯放在右边的 `.` 上。
- 从左到右扫一遍就够了，本质是线性贪心。
- 左边已经处理过，局部选择只需要尽量照顾右边。
