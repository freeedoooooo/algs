## 一句话结论
并查集就是一套专门处理“谁和谁属于同一组”的数据结构。
它最擅长两件事：

- 判断两个元素是不是同一个集合
- 把两个集合快速合并

它之所以快，靠的是两个优化：

```text
路径压缩：查的时候顺手把链拉平
按规模合并：合的时候尽量别让树长高
```

## 先记结论
第一次看并查集，先记住下面 4 句话：

```text
1. 每个集合都有一个代表点
2. find 是找代表点
3. union 是合并两个集合
4. 判断是否同组，本质上就是看代表点是否相同
```

如果只记一句话，可以记成：

```text
并查集 = 动态维护若干不相交集合
```

## 题意说明
并查集不是某一道具体题，而是一种数据结构模型。  
它通常用来解决这一类问题：

- 初始时有很多元素
- 一开始每个元素各自单独成组
- 过程中会不断发生“合并”操作
- 还会频繁询问“这两个元素是不是同一组”

所以它最核心要解决的是：

```text
动态维护分组关系
并高效回答“是否同组 / 是否连通”
```

## 先把这个结构想象出来
并查集底层可以先想成一片森林。

每个节点都有一个父节点指针：

```text
x -> parent[x]
```

不断沿着父指针往上走，最终会走到某个“自己指向自己”的节点：

```text
root -> root
```

这个节点就是这个集合的代表点，也叫根节点。

于是整套逻辑一下就统一了：

- 两个元素是不是同组：看它们的根是不是同一个
- 合并两个集合：让一个根挂到另一个根下面

## 图解
假设开始时有 4 个元素：

```text
1   2   3   4
```

初始每个元素自己就是一个集合：

```text
1    2    3    4
```

现在执行：

```text
union(1, 2)
union(3, 4)
```

可以粗略理解成：

```text
1 <- 2
3 <- 4
```

这时一共有两个集合：

```text
[1,2]   [3,4]
```

如果再执行：

```text
union(2, 4)
```

因为：

- `2` 所在集合的代表点是 `1`
- `4` 所在集合的代表点是 `3`

所以真正发生的不是“2 挂到 4”，而是：

```text
代表点 1 和代表点 3 合并
```

最终 4 个点会收拢成一个集合。

于是：

```text
isSameSet(1, 3) == true
```

## 为什么需要 `find`
并查集里最基础的操作就是 `find(x)`。

它做的事情非常单纯：

```text
顺着父指针一直往上走
找到 x 所在集合的代表点
```

所以很多操作其实最后都会转成 `find`：

- 是否同组：`find(a) == find(b)`
- 合并集合：先分别找到两个集合的根，再决定怎么挂

你可以把 `find` 理解成：

```text
查户口本上的“户主”
```

## 为什么路径压缩这么重要
如果一个节点到根的链很长，比如：

```text
5 -> 4 -> 3 -> 2 -> 1
```

那每次找 `5` 的根，都得一路往上爬，非常亏。

路径压缩做的事就是：

```text
既然这次已经辛苦爬到根了
那就把沿途所有节点直接挂到根上
```

压完之后会变成：

```text
5 -> 1
4 -> 1
3 -> 1
2 -> 1
1 -> 1
```

这样下一次再查就快很多。

所以路径压缩不是额外功能，而是并查集高效的关键。

## 为什么还要按规模合并
合并两个集合时，如果你总是随便挂，就有可能把树挂得很高。

例如：

```text
小树挂大树  -> 还好
大树挂小树  -> 更容易变高
```

所以并查集的常见策略是：

```text
小集合挂到大集合下面
```

这样树高增长更慢，后续 `find` 的成本也更低。

你可以把它想成：

```text
尽量别让“大棵树”再多长一层
```

## 三个核心操作放在一起看
### 1. `find(x)`
找到 `x` 所在集合的代表点。

### 2. `isSameSet(a, b)`
先分别找 `a` 和 `b` 的代表点，再判断是否相同。

### 3. `union(a, b)`
先分别找到两个集合的代表点。  
如果已经相同，说明本来就是同一组，什么都不用做；  
如果不同，就把小集合的根挂到大集合的根下面。

## 典型使用场景
并查集最适合这类题：

- 两个人是不是同一个社交圈
- 两个节点是不是连通
- 一批记录应该合并成几组
- 一堆点或区域是否属于同一块

这类题的共同点是：

```text
你不关心具体路径
你只关心“是不是同一组”
```

## 复杂度
- 单次均摊时间复杂度：近似 `O(1)`
- 空间复杂度：`O(N)`

更严谨地说，带路径压缩和按规模合并后，复杂度非常低，在面试和工程里通常就当作近似 `O(1)` 来理解。

## 易错点
- 判断同组，看的是代表点是否相同，不是编号是否接近。
- `union(a, b)` 合并的是两个集合，不是单纯把 `a` 挂到 `b`。
- 只写 `find` 不做优化，也能对，但效率会差很多。
- 并查集擅长维护连通性，不擅长输出具体路径和结构细节。

## 代码 / 伪代码
前面的伪代码里用了 `Node`，这里容易让人误会成“必须手写树节点类”。  
其实并查集里的 `Node` 只是一个“元素类型”的占位写法，你可以把它理解成：

- 一个整数编号，例如 `1、2、3、4`
- 一个字符串，例如用户名
- 一个对象引用，例如某个 `User`

也就是说，并查集并不要求元素真的长成树节点那样。  
它只要求：

```text
每个元素都能在 parent 表里找到它的父节点
```

如果你第一次学并查集，用整数版本最容易理解。  
下面先给一版更完整、可读性更强的整数版代码骨架：

```java
class UnionFind {
    private int[] parent;
    private int[] size;
    private int sets;

    public UnionFind(int n) {
        parent = new int[n];
        size = new int[n];
        sets = n;
        for (int i = 0; i < n; i++) {
            parent[i] = i; // 一开始每个点的父节点都是自己
            size[i] = 1;   // 每个集合初始大小都是 1
        }
    }

    public int find(int x) {
        if (parent[x] != x) {
            parent[x] = find(parent[x]); // 路径压缩
        }
        return parent[x];
    }

    public boolean isSameSet(int a, int b) {
        return find(a) == find(b);
    }

    public void union(int a, int b) {
        int fa = find(a);
        int fb = find(b);
        if (fa == fb) {
            return;
        }

        if (size[fa] >= size[fb]) {
            parent[fb] = fa;
            size[fa] += size[fb];
        } else {
            parent[fa] = fb;
            size[fb] += size[fa];
        }
        sets--;
    }

    public int sets() {
        return sets;
    }
}
```

如果把它代回到前面的抽象写法里，那么：

- `Node` 版里的 `parent[x]`
- 就等价于整数版里的 `parent[x]`

只是元素类型从“抽象节点”换成了“整数下标”。

### 1. 先看初始化在干什么
初始化时：

```java
parent[i] = i;
size[i] = 1;
```

意思是：

- 每个元素一开始自己单独成集合
- 自己就是自己的代表点
- 每个集合大小都是 `1`

例如 `n = 4` 时，初始状态就是：

```text
0 -> 0
1 -> 1
2 -> 2
3 -> 3
```

### 2. 路径压缩到底写在哪
路径压缩就写在 `find` 里面，而且关键就是这一句：

```java
parent[x] = find(parent[x]);
```

完整代码是：

```java
public int find(int x) {
    if (parent[x] != x) {
        parent[x] = find(parent[x]);
    }
    return parent[x];
}
```

### 1. `find(x)` 到底在做什么
这一段是并查集的核心。

```java
int find(int x) {
    if (parent[x] != x) {
        parent[x] = find(parent[x]);
    }
    return parent[x];
}
```

可以按下面顺序理解：

1. 如果 `x` 的父节点就是自己，说明 `x` 已经是根了
2. 如果不是，就继续往上找 `x` 的父节点的根
3. 找到根以后，顺手把 `x` 直接挂到根上

其中最关键的就是这一句：

```java
parent[x] = find(parent[x]);
```

它同时做了两件事：

- 递归地找到真正的代表点
- 把当前节点直接压缩到代表点下面

所以 `find` 不只是“查”，还顺手完成了路径压缩。

### 3. 用一条长链单独看路径压缩
假设当前父节点关系是：

```text
5 -> 4 -> 3 -> 2 -> 1
1 -> 1
```

这时调用：

```java
find(5)
```

查找过程会一路往上：

```text
find(5)
-> find(4)
-> find(3)
-> find(2)
-> find(1)
```

最终发现根是 `1`。

但递归返回时，不只是“知道答案”，还会顺手改写父节点：

```text
parent[5] = 1
parent[4] = 1
parent[3] = 1
parent[2] = 1
```

于是整条链会被压平为：

```text
5 -> 1
4 -> 1
3 -> 1
2 -> 1
1 -> 1
```

这就是路径压缩。

所以路径压缩的感觉可以记成：

```text
第一次查，顺便修路
后面再查，就几乎直达了
```

### 4. `isSameSet(a, b)` 为什么这么短

```java
boolean isSameSet(int a, int b) {
    return find(a) == find(b);
}
```

因为并查集判断“是否同组”的标准非常统一：

```text
只看代表点是不是同一个
```

所以这里虽然代码很短，但底层依赖的是：

- `find(a)` 找到 `a` 所在集合的根
- `find(b)` 找到 `b` 所在集合的根
- 如果两个根相同，说明它们在同一个集合

### 5. `union(a, b)` 为什么不能直接把 `a` 挂到 `b`

```java
void union(int a, int b) {
    int fa = find(a);
    int fb = find(b);
    if (fa == fb) {
        return;
    }

    if (size[fa] >= size[fb]) {
        parent[fb] = fa;
        size[fa] += size[fb];
    } else {
        parent[fa] = fb;
        size[fb] += size[fa];
    }
}
```

这里第一步不是直接动 `a` 和 `b`，而是先做：

```java
int fa = find(a);
int fb = find(b);
```

原因是：

```text
真正要合并的是两个集合
不是两个普通节点
```

所以你得先找到它们各自集合的代表点，再决定怎么挂。

### 6. 为什么先判断 `fa == fb`

```java
if (fa == fb) {
    return;
}
```

这表示：

```text
a 和 b 本来就已经在同一个集合里了
这次 union 什么都不用做
```

如果不加这个判断，后面重复更新 `parent` 和 `size`，逻辑就会出错。

### 7. `size` 数组是干什么的
这里的 `size[root]` 表示：

```text
以 root 为代表点的这个集合，目前有多少个节点
```

它的作用不是回答业务问题，而是服务于优化：

```text
合并时让小集合挂到大集合下面
```

这样树就不容易长高。

### 8. 为什么挂完之后还要更新 `size`
比如：

```java
parent[fb] = fa;
size[fa] += size[fb];
```

意思是：

- 现在 `fb` 代表的整棵树都挂到 `fa` 下面了
- 所以 `fa` 这个集合的大小，要加上 `fb` 那边的节点数

如果不更新，后面再做“按规模合并”时，判断大小就不准了。

### 9. 把三段代码连起来看
并查集每次操作，其实都可以压缩成下面这条主线：

```text
先用 find 找代表点
再决定是不是同组
如果要合并，就让小集合的根挂到大集合的根下面
```

也就是说：

- `find` 是基础
- `isSameSet` 是对 `find` 的直接应用
- `union` 是在 `find` 的结果上做集合级别的连接

### 10. 一个小过程例子
假设现在：

```text
1 <- 2
1 <- 3
4 <- 5
```

也就是两个集合：

```text
[1,2,3]
[4,5]
```

如果执行：

```java
union(2, 5)
```

代码会这样走：

1. `find(2)` 找到根 `1`
2. `find(5)` 找到根 `4`
3. 比较 `size[1]` 和 `size[4]`
4. 假设 `[1,2,3]` 更大，就让 `4` 挂到 `1`

最后会变成：

```text
[1,2,3,4,5]
```

这就能看出：

```text
union(2, 5)
表面在传 2 和 5
本质上连的是它们背后的两个集合
```

## 记忆点
- 并查集维护“若干不相交集合”。
- `find` 找代表点，`union` 合并集合。
- 路径压缩是“查的时候顺手拉平”。
- 小挂大，是为了别让树长高。
