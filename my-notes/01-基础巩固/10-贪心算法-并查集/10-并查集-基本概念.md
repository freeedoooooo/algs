## 一句话结论
并查集就是一套专门处理“谁和谁属于同一组”的数据结构。
它最擅长两件事：

- 判断两个元素是不是同一个集合
- 把两个集合快速合并

它之所以快，靠的是两个优化：

```text
路径压缩：查的时候顺手把链拉平
按规模合并：合的时候尽量别让树长高
```

## 先记结论
第一次看并查集，先记住下面 4 句话：

```text
1. 每个集合都有一个代表点
2. find 是找代表点
3. union 是合并两个集合
4. 判断是否同组，本质上就是看代表点是否相同
```

如果只记一句话，可以记成：

```text
并查集 = 动态维护若干不相交集合
```

## 题意说明
并查集不是某一道具体题，而是一种数据结构模型。  
它通常用来解决这一类问题：

- 初始时有很多元素
- 一开始每个元素各自单独成组
- 过程中会不断发生“合并”操作
- 还会频繁询问“这两个元素是不是同一组”

所以它最核心要解决的是：

```text
动态维护分组关系
并高效回答“是否同组 / 是否连通”
```

## 先把这个结构想象出来
并查集底层可以先想成一片森林。

每个节点都有一个父节点指针：

```text
x -> parent[x]
```

不断沿着父指针往上走，最终会走到某个“自己指向自己”的节点：

```text
root -> root
```

这个节点就是这个集合的代表点，也叫根节点。

于是整套逻辑一下就统一了：

- 两个元素是不是同组：看它们的根是不是同一个
- 合并两个集合：让一个根挂到另一个根下面

## 图解
假设开始时有 4 个元素：

```text
1   2   3   4
```

初始每个元素自己就是一个集合：

```text
1    2    3    4
```

现在执行：

```text
union(1, 2)
union(3, 4)
```

可以粗略理解成：

```text
1 <- 2
3 <- 4
```

这时一共有两个集合：

```text
[1,2]   [3,4]
```

如果再执行：

```text
union(2, 4)
```

因为：

- `2` 所在集合的代表点是 `1`
- `4` 所在集合的代表点是 `3`

所以真正发生的不是“2 挂到 4”，而是：

```text
代表点 1 和代表点 3 合并
```

最终 4 个点会收拢成一个集合。

于是：

```text
isSameSet(1, 3) == true
```

## 为什么需要 `find`
并查集里最基础的操作就是 `find(x)`。

它做的事情非常单纯：

```text
顺着父指针一直往上走
找到 x 所在集合的代表点
```

所以很多操作其实最后都会转成 `find`：

- 是否同组：`find(a) == find(b)`
- 合并集合：先分别找到两个集合的根，再决定怎么挂

你可以把 `find` 理解成：

```text
查户口本上的“户主”
```

## 为什么路径压缩这么重要
如果一个节点到根的链很长，比如：

```text
5 -> 4 -> 3 -> 2 -> 1
```

那每次找 `5` 的根，都得一路往上爬，非常亏。

路径压缩做的事就是：

```text
既然这次已经辛苦爬到根了
那就把沿途所有节点直接挂到根上
```

压完之后会变成：

```text
5 -> 1
4 -> 1
3 -> 1
2 -> 1
1 -> 1
```

这样下一次再查就快很多。

所以路径压缩不是额外功能，而是并查集高效的关键。

## 为什么还要按规模合并
合并两个集合时，如果你总是随便挂，就有可能把树挂得很高。

例如：

```text
小树挂大树  -> 还好
大树挂小树  -> 更容易变高
```

所以并查集的常见策略是：

```text
小集合挂到大集合下面
```

这样树高增长更慢，后续 `find` 的成本也更低。

你可以把它想成：

```text
尽量别让“大棵树”再多长一层
```

## 三个核心操作放在一起看
### 1. `find(x)`
找到 `x` 所在集合的代表点。

### 2. `isSameSet(a, b)`
先分别找 `a` 和 `b` 的代表点，再判断是否相同。

### 3. `union(a, b)`
先分别找到两个集合的代表点。  
如果已经相同，说明本来就是同一组，什么都不用做；  
如果不同，就把小集合的根挂到大集合的根下面。

## 典型使用场景
并查集最适合这类题：

- 两个人是不是同一个社交圈
- 两个节点是不是连通
- 一批记录应该合并成几组
- 一堆点或区域是否属于同一块

这类题的共同点是：

```text
你不关心具体路径
你只关心“是不是同一组”
```

## 复杂度
- 单次均摊时间复杂度：近似 `O(1)`
- 空间复杂度：`O(N)`

更严谨地说，带路径压缩和按规模合并后，复杂度非常低，在面试和工程里通常就当作近似 `O(1)` 来理解。

## 易错点
- 判断同组，看的是代表点是否相同，不是编号是否接近。
- `union(a, b)` 合并的是两个集合，不是单纯把 `a` 挂到 `b`。
- 只写 `find` 不做优化，也能对，但效率会差很多。
- 并查集擅长维护连通性，不擅长输出具体路径和结构细节。

## 代码 / 伪代码
示例里用数组表示树结构：

```text
parent[x] = x 的父节点
```

元素直接看成整数编号即可。

### 怎么理解“用数组表示树”
这里虽然没有真的写一个“树节点类”，但树结构其实已经藏在 `parent` 数组里了。

你可以把：

```java
parent[2] = 1
```

直接读成：

```text
2 的父节点是 1
也就是 2 -> 1
```

如果再有：

```java
parent[1] = 0
parent[0] = 0
```

那整段关系连起来就是：

```text
2 -> 1 -> 0
0 -> 0
```

这其实就已经是一棵树了：

```text
    0
   /
  1
 /
2
```

所以：

- 数组下标 `x` 表示“节点是谁”
- `parent[x]` 表示“这个节点的父亲是谁”

只要每个点都能找到自己的父节点，这个父子关系整体上就构成了一片树。

### 为什么根节点会写成 `parent[root] = root`
这是并查集里最关键的约定。

例如：

```java
parent[0] = 0
```

意思不是“0 有个奇怪的父节点 0”，而是：

```text
0 已经是根了
它就是这个集合的代表点
```

所以并查集找根时，停止条件就是：

```text
如果 parent[x] == x
说明已经走到代表点
```

### 一个完整的小例子
假设：

```java
parent[0] = 0;
parent[1] = 0;
parent[2] = 1;
parent[3] = 3;
parent[4] = 3;
```

把它翻译成箭头关系，就是：

```text
0 -> 0
1 -> 0
2 -> 1

3 -> 3
4 -> 3
```

再整理成树的样子：

```text
集合 1：
    0
   /
  1
 /
2

集合 2：
  3
 /
4
```

所以这时不是一棵树，而是两棵树，也就是两个集合。

这就是为什么常说：

```text
并查集底层是一片森林
```

### 这个 demo 想让你先看懂什么
这个数组版 demo 不是为了强调“树一定要用数组实现”，而是为了让你先抓住并查集最核心的一点：

```text
每个节点只需要知道自己的父节点是谁
一直往上走，就能找到整组的代表点
```

只要这件事看懂了，后面不管是数组版、`HashMap` 版，还是对象版，本质都一样。

### 最小代码版

```java
int[] parent;

int find(int x) {
    if (parent[x] != x) {
        parent[x] = find(parent[x]);
    }
    return parent[x];
}

boolean isSameSet(int a, int b) {
    return find(a) == find(b);
}

void union(int a, int b) {
    int fa = find(a);
    int fb = find(b);
    if (fa != fb) {
        parent[fa] = fb;
    }
}
```

### 初始化图示
如果有 5 个点，一开始每个点自己单独成组：

```text
0   1   2   3   4
```

此时：

```java
parent[0] = 0
parent[1] = 1
parent[2] = 2
parent[3] = 3
parent[4] = 4
```

图上可以理解成：

```text
0 -> 0
1 -> 1
2 -> 2
3 -> 3
4 -> 4
```

### 1. `find(x)` 图示说明

```java
int find(int x) {
    if (parent[x] != x) {
        parent[x] = find(parent[x]);
    }
    return parent[x];
}
```

作用：顺着父节点一直往上找，直到找到代表点。

比如现在结构是：

```text
4 -> 3 -> 1 -> 1
```

那：

调用 `find(4)`：

查找路线就是：

```text
4 -> 3 -> 1
```

返回 `1`，因为 `1` 是代表点。

### 2. 路径压缩图示说明
关键代码：

```java
parent[x] = find(parent[x]);
```

继续看刚才这条链：

```text
4 -> 3 -> 1 -> 1
```

调用一次 `find(4)` 后：

结构会变成：

```text
4 -> 1
3 -> 1
1 -> 1
```

也就是从：

```text
4 -> 3 -> 1
```

压成：

```text
4 ----\
3 ----- > 1
```

### 3. `isSameSet(a, b)` 图示说明

```java
boolean isSameSet(int a, int b) {
    return find(a) == find(b);
}
```

标准：只看两个点最后找到的根是不是同一个。

例如：

```text
2 -> 1
4 -> 1
```

那么：

```text
find(2) = 1
find(4) = 1
isSameSet(2, 4) == true
```

### 4. `union(a, b)` 图示说明

```java
void union(int a, int b) {
    int fa = find(a);
    int fb = find(b);
    if (fa != fb) {
        parent[fa] = fb;
    }
}
```

真正合并的是：

```text
a 所在集合的代表点
b 所在集合的代表点
```

比如当前有两组：

```text
2 -> 1
1 -> 1

4 -> 3
3 -> 3
```

调用 `union(2, 4)`：

实际上发生的是：

```text
find(2) = 1
find(4) = 3
```

然后把一个根挂到另一个根上：

```text
1 -> 3
```

于是两组就连起来了：

```text
2 -> 1 -> 3
4 -> 3
3 -> 3
```

这时候再查：

```text
isSameSet(2, 4) == true
```

### 5. 把三个函数连起来看
并查集代码就是反复做这 3 件事：

```text
find：找代表点
isSameSet：比较两个代表点
union：把两个代表点连起来
```

### 6. 为什么后面还会学“小挂大”
最小代码版还有一个问题：

```text
可能会把树挂得很高
```

所以正式版本里会再加：

- 按规模合并
- 路径压缩

- 路径压缩：查得更快
- 小挂大：树别太高

## 记忆点
- 并查集维护“若干不相交集合”。
- `find` 找代表点，`union` 合并集合。
- 路径压缩是“查的时候顺手拉平”。
- 小挂大，是为了别让树长高。
