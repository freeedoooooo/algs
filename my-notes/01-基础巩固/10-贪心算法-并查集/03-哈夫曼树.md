## 一句话结论
哈夫曼树的核心思想是：权值越小的节点，应该越早合并、放得越深，这样才能让带权路径长度最小。
从贪心角度看，它的结论就是一句话：

```text
每次先合并当前最小的两个权值
```

## 核心知识点
- 哈夫曼树解决的是带权路径长度最小问题。
- 叶子节点有权值，路径越深，权值被乘上的层数越多。
- 小权值应该更深，大权值应该更浅。
- 正确贪心策略是反复合并当前最小的两个权值。
- 实现上通常用小根堆维护。

## 这篇到底在讲什么
如果给你一组权值：

```text
5 7 2 13
```

你要把它们组织成一棵二叉树。  
不同合并顺序，会得到不同的总代价。

哈夫曼树研究的就是：

```text
怎么构造一棵树
让所有叶子节点的带权路径长度之和最小
```

其中：

```text
带权路径长度 WPL = Σ(叶子权值 × 该叶子到根的路径长度)
```

## 图解
假设有 3 个权值：

```text
10 20 30
```

如果让 `10` 和 `20` 先合并：

```text
10 + 20 -> 30
30 + 30 -> 60
```

它们会更深地待在树里，但因为它们比较小，所以总代价仍然更优。

可以粗略理解成：

```text
小数被多加几次，损失小
大数被多加几次，损失大
```

所以权值小的节点，更适合先合并。

## 为什么贪心成立
哈夫曼树最关键的结论是：

- 当前所有权值里最小的两个
- 在最优树里一定可以作为同一层的兄弟节点先合并

于是整个问题就可以不断缩小：

1. 先把最小的两个合并成一个新权值
2. 把这个新权值重新放回集合
3. 在剩下的权值里继续做同样的事

这就是典型的贪心 + 递归缩小问题规模。

## 和切分金条问题的关系
切分金条问题，其实就是哈夫曼树最常见的落地题。

因为：

```text
一次合并的代价 = 两段长度之和
哈夫曼树每次合并的代价 = 两个权值之和
```

所以两者本质完全一致。

你可以把关系记成：

```text
哈夫曼树 = 理论模型
切分金条 = 经典原型题
```

## 构造过程示例
以权值 `[6, 7, 8, 9]` 为例：

第一轮：

```text
6 + 7 = 13
```

第二轮：

```text
8 + 9 = 17
```

第三轮：

```text
13 + 17 = 30
```

最终构造出的树，对应的总代价就是：

```text
13 + 17 + 30 = 60
```

## 复杂度
- 时间复杂度：`O(N log N)`
- 空间复杂度：`O(N)`

## 易错点
- 哈夫曼树不是普通二叉搜索树，也不是堆。
- 关键目标是“带权路径长度最小”，不是树高最小。
- 不是一次排序后从左到右合并，而是要动态反复取最小两个。
- 新合并出来的节点要重新放回集合继续参与合并。

## 代码 / 伪代码
常见构造写法：

```java
int huffman(int[] arr) {
    PriorityQueue<Integer> heap = new PriorityQueue<>();
    for (int num : arr) {
        heap.add(num);
    }
    int sum = 0;
    while (heap.size() > 1) {
        int cur = heap.poll() + heap.poll();
        sum += cur;
        heap.add(cur);
    }
    return sum;
}
```

## 记忆点
- 哈夫曼树就是“最小权值优先合并”。
- 小权值更适合放深一点，大权值更适合靠近根。
- 带权路径长度最小，是它的核心目标。
- 小根堆是最自然的实现结构。
