## 一句话结论
0/1 背包的递归状态是 `index + rest`，所以最自然的 DP 表就是二维表 `dp[index][rest]`。

## 图片转写 / 题意还原
这篇是在前面“0/1 背包暴力递归”基础上，把原题正式改写成动态规划。
完整题意是：

- 给定两个等长数组 `w` 和 `v`
- `w[i]` 表示第 `i` 件物品重量，`v[i]` 表示第 `i` 件物品价值
- 给定一个背包承重上限 `bag`
- 每件物品最多只能选一次
- 要求在总重量不超过 `bag` 的前提下，使总价值最大

这篇笔记要解决的不是“背包题是什么”，而是：

```text
已知暴力递归状态是 index + rest
怎样把它改写成一张明确的 DP 表
```

## 解题思路
- 递归含义：从 `index` 往后选，剩余容量是 `rest` 时的最大价值
- 表大小：`N+1` 行，`bag+1` 列
- 填表顺序：`index` 从下往上，`rest` 从左往右

## 代码 / 伪代码
```java
int[][] dp = new int[N + 1][bag + 1];
for (int index = N - 1; index >= 0; index--) {
    for (int rest = 0; rest <= bag; rest++) {
        int p1 = dp[index + 1][rest];
        int p2 = -1;
        if (rest - w[index] >= 0) {
            p2 = v[index] + dp[index + 1][rest - w[index]];
        }
        dp[index][rest] = Math.max(p1, p2);
    }
}
return dp[0][bag];
```

## 记忆点
- 状态：`index + rest`
- `dp[index][rest]` 直接抄递归定义
- 先看不要，再看要不要得起
