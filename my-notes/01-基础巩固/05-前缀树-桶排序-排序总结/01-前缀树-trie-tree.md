## 一句话结论
前缀树本质上是“按字符逐层建路”的多叉树，特别适合处理字符串集合中的“是否存在、出现次数、某前缀下有多少单词”这类问题。
它用空间换时间，把公共前缀复用掉，从而把单次查询复杂度压到和字符串长度成正比。

## 核心知识点
- 每个节点通常维护两个信息：`pass` 和 `end`。
- `pass` 表示有多少字符串经过当前节点，`end` 表示有多少字符串在当前节点结尾。
- 字符集固定且较小时适合用数组存边；字符集复杂时适合用 `HashMap` 存边。
- Trie 不按字符串整体比较，而是按字符一层一层向下走。

## 图片转写 / 题意还原
原图表达的核心内容可以还原为：

- 前缀树是一种多叉树结构。
- 插入字符串时，从左到右依次处理每个字符。
- 如果当前字符对应的路不存在，就新建一条路；如果已经存在，就直接复用。
- 沿途经过的每个节点，`pass` 都要加一。
- 字符串结束时，最后一个节点的 `end` 加一。
- 因为公共前缀会共享路径，所以它很适合做前缀相关查询。

可以把它理解成：

```text
cat
car
dog
```

会被组织成：

```text
root
 ├─ c
 │   └─ a
 │       ├─ t
 │       └─ r
 └─ d
     └─ o
         └─ g
```

## 图解
把：

```text
cat
car
dog
```

插入 Trie，可以想成：

```text
插入 cat:
root -> c -> a -> t

插入 car:
root -> c -> a -> r
        ^    ^
      这段路径直接复用

插入 dog:
root -> d -> o -> g
```

所以 Trie 的本质就是：

```text
相同前缀共用同一条路
```

## 解题思路
### 1. 插入
从根节点出发，按字符逐个往下走。

- 没有这条路就创建节点
- 有这条路就直接复用
- 每经过一个节点，`pass++`
- 到单词末尾时，`end++`

### 2. 查询完整单词
沿着字符串把路径走完：

- 中途断路，说明不存在，返回 `0`
- 能完整走到结尾，就返回最后节点的 `end`

这表示“这个单词被插入过几次”。

### 3. 查询前缀数量
同样沿着前缀往下走：

- 中途断路，返回 `0`
- 能走到前缀末尾，就返回当前节点的 `pass`

这表示“有多少单词以这个前缀开头”。

### 4. 删除
删除前要先确认这个单词存在。

- 沿途把 `pass--`
- 如果某条路减完后 `pass == 0`，说明后面整棵子树都没人用了，可以直接断开
- 最后把终点节点的 `end--`

## 典型例子
依次插入：

```text
apple
app
apple
```

那么：

- `countWordsEqualTo("apple") = 2`
- `countWordsEqualTo("app") = 1`
- `countWordsStartingWith("app") = 3`

如果再删除一次 `apple`，就变成：

- `countWordsEqualTo("apple") = 1`
- `countWordsStartingWith("app") = 2`

这就是 `pass` 和 `end` 分工的意义：

- `end` 管“完整命中”
- `pass` 管“前缀经过”

## 复杂度
- 时间复杂度：插入、删除、查询都为 `O(L)`，`L` 是字符串长度。
- 空间复杂度：和插入后实际创建的节点总数有关，最坏为所有字符串总字符数级别。

## 易错点
- `pass` 不是当前字符出现次数，而是“有多少字符串经过这个节点”。
- 删除前要先判断单词是否存在，否则会把计数减错。
- 数组版 Trie 一般默认字符集固定，比如只处理 `a~z`。
- 如果字符集很大，数组会浪费空间，这时更适合用 `HashMap`。

## 代码 / 伪代码
仓库中的对应实现：

- [TrieTree.java](../../../solutions-java/practice/algs03_tree/TrieTree.java)

这份实现给了两种写法：

- `Trie1`：用 `TrieNode[26]` 存边，适合只处理小写字母
- `Trie2`：用 `HashMap<Integer, TrieNode>` 存边，适合更通用的字符集

核心结构可以简化成：

```java
class TrieNode {
    int pass;
    int end;
    TrieNode[] nextNodes = new TrieNode[26];
}
```

删除逻辑的关键判断是：

```java
if (--node.nextNodes[path].pass == 0) {
    node.nextNodes[path] = null;
    return;
}
```

## 记忆点
- Trie = 字符串按位建树。
- `pass` 管前缀，`end` 管结尾。
- 复用公共前缀，是它比普通哈希表更适合前缀查询的根本原因。
