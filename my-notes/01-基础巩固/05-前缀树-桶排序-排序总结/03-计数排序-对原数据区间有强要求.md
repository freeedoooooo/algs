## 一句话结论
计数排序适合“值域不大、重复值很多”的场景，它不比较元素，而是直接统计每个值出现了多少次，再把结果按顺序写回去。
它的性能非常好，但前提很硬：如果原数据区间太大，计数数组会变得不划算。

## 核心知识点
- 计数排序的核心不是比较，而是“值 -> 次数”的映射。
- 如果要保持稳定性，计数数组还要进一步转成前缀和数组。
- 当数据包含负数时，可以通过“整体偏移”处理。
- 真正限制它的不是 `N`，而是值域大小 `max - min + 1`。

## 解题思路
### 1. 统计频次
先扫描一遍数组，找到：

- 最小值 `min`
- 最大值 `max`

然后开一个长度为：

```text
max - min + 1
```

的计数数组 `counter`。

再扫描原数组，把每个元素对应位置的计数加一。

### 2. 普通版：直接回填
如果只想完成排序，不强调稳定性，那么可以按照计数数组从小到大把数字写回原数组。

### 3. 稳定版：转成前缀和
如果还要保持“相同元素的相对顺序不变”，就需要把计数数组转成前缀和数组，再从右往左回填到辅助数组。

这样同值元素会按原顺序落位，排序才是稳定的。

### 4. 负数怎么处理
如果数组里有负数，不能直接拿值当下标。

做法是统一偏移：

```text
下标 = num - min
```

这样最小值会映射到 `0`，其他值顺次映射。

## 典型例子
数组：

```text
[-2, 3, -2, 1, 0]
```

那么：

- `min = -2`
- `max = 3`
- `range = 6`

计数数组下标和真实值的关系是：

```text
0 -> -2
1 -> -1
2 ->  0
3 ->  1
4 ->  2
5 ->  3
```

统计后：

```text
[-2 出现 2 次, -1 出现 0 次, 0 出现 1 次, 1 出现 1 次, 2 出现 0 次, 3 出现 1 次]
```

按顺序回填，结果就是：

```text
[-2, -2, 0, 1, 3]
```

## 复杂度
- 时间复杂度：`O(N + range)`
- 空间复杂度：`O(range)`

其中 `range = max - min + 1`。

所以它快不快，关键不是只看 `N`，而是要看值域。

## 易错点
- 计数排序不是“适合所有整数数组”，值域太大时会非常浪费空间。
- 想要稳定排序时，不能只统计次数后直接回填。
- 处理负数时，要记得做偏移。
- 从右往左回填辅助数组，是稳定性的关键步骤。

## 代码 / 伪代码
仓库中的实现：

- [CountingSort.java](../../../solutions-java/practice/algs01_sort/CountingSort.java)

这份代码做了两件很重要的事：

- 用 `min` / `max` 计算真实值域
- 用 `num - min` 支持负数

核心逻辑可以概括成：

```java
for (int num : arr) {
    counter[num - min]++;
}

for (int i = 1; i < range; i++) {
    counter[i] += counter[i - 1];
}

for (int i = arr.length - 1; i >= 0; i--) {
    output[counter[arr[i] - min] - 1] = arr[i];
    counter[arr[i] - min]--;
}
```

## 记忆点
- 计数排序快，靠的是“直接统计”，不是“聪明比较”。
- 它最怕的不是数据多，而是值域大。
- 判断能不能用它，先问自己一句：`max - min` 大不大？
