## 一句话结论
基数排序的核心不是一次把数整体排好，而是按个位、十位、百位这样的“每一位”逐轮做稳定排序。
只要数据满足位数结构可拆解，尤其是非负整数场景，它就能做到 `O(N * d)`，其中 `d` 是最大数的位数。

## 核心知识点
- 基数排序依赖“按位分发 + 每一轮稳定”。
- 常见写法是 LSD（Least Significant Digit），即从低位到高位排序。
- 传统课程里通常先讲“非负十进制整数”，这样最容易理解。
- 如果存在负数，可以先整体平移成非负数，再做基数排序。

## 解题思路
### 1. 为什么要按位排
如果直接比较整个数，还是比较排序。
基数排序换了个思路：

- 先看个位
- 再看十位
- 再看百位
- ...

每一轮都只关心当前这一位的数字落到哪个桶。

### 2. 为什么必须稳定
假设两个数当前位相同，如果这一轮排序不稳定，就会破坏前面低位已经排好的相对顺序。

所以“稳定”是基数排序正确性的核心。

### 3. LSD 做法
以十进制为例：

1. 先统计个位 `0~9` 各出现几次
2. 转成前缀和，得到每个桶的落位范围
3. 从右往左回填辅助数组，保证稳定
4. 拷回原数组
5. 再处理十位、百位……

直到处理完最大数的最高位。

### 4. 负数怎么兼容
课程题常直接限定“非负数”。
仓库这份实现更进一步，先找到最小值 `min`，把所有数减去 `min` 变成非负数，排序完再加回去。

这相当于先做了一次整体平移。

## 典型例子
数组：

```text
[53, 3, 542, 748, 14, 214]
```

### 按个位排

```text
53 -> 3
3  -> 3
542 -> 2
748 -> 8
14 -> 4
214 -> 4
```

这一轮稳定排序后，会让个位小的先走，个位相同的保持原相对次序。

### 按十位排
再基于上一轮结果按十位排，低位已经整理好的顺序不会被破坏。

### 按百位排
继续处理到最高位后，整个数组就有序。

## 复杂度
- 时间复杂度：`O(N * d)`，其中 `d` 是最大数的位数；十进制下也常写作 `O(N * log10(max))`
- 空间复杂度：`O(N + BASE)`，其中 `BASE` 是进制大小，十进制时为 `10`

如果把进制视为常数，通常就把它写成 `O(N)` 的额外空间。

## 易错点
- 基数排序不是随便按位排，必须保证每一轮稳定。
- 从低位到高位是 LSD 写法；如果顺序错了，结果可能不对。
- 只说“适合整数”不够准确，关键是位数结构要可拆，且实现时通常先要求非负。
- `bits(0)` 这类边界值要特别注意，不同实现会有不同处理方式。

## 代码 / 伪代码
仓库中的实现：

- [RadixSort.java](../../../solutions-java/practice/algs01_sort/RadixSort.java)

这份代码的亮点有两个：

- 用 `bits(max)` 计算要做几轮
- 先减去最小值，把可能含负数的数组整体转成非负数组

核心框架可以概括成：

```java
for (int offset = 1; bits > 0; offset *= BASE, bits--) {
    Arrays.fill(cnts, 0);
    for (int i = 0; i < n; i++) {
        cnts[(arr[i] / offset) % BASE]++;
    }
    for (int i = 1; i < BASE; i++) {
        cnts[i] += cnts[i - 1];
    }
    for (int i = n - 1; i >= 0; i--) {
        help[--cnts[(arr[i] / offset) % BASE]] = arr[i];
    }
    for (int i = 0; i < n; i++) {
        arr[i] = help[i];
    }
}
```

## 记忆点
- 基数排序 = 一位一位排。
- 低位先排，高位兜底。
- 每一轮都必须稳定，否则前面白排。
- 看到“整数、位数有限、值域不适合计数排序”时，就要想到它。
