## 章节导读
这一章其实包含两条主线：

- 一条是字符串结构：前缀树（Trie）
- 一条是排序体系：非比较排序、稳定性、排序算法选型总结

如果只看知识结构，可以这样记：

```text
前缀树
└─ 解决前缀查询、词频统计、字符串集合管理

排序总结
├─ 基于比较的排序
│  ├─ 选择 / 冒泡 / 插入
│  ├─ 归并 / 快排 / 堆排
│  └─ 理论下界 O(N log N)
└─ 不基于比较的排序
   ├─ 计数排序
   └─ 基数排序
```

## 本章结论
- 前缀树适合处理“字符串是否存在、出现几次、前缀命中多少次”。
- 计数排序和基数排序都属于桶思想下的非比较排序。
- 非比较排序可以突破 `O(N log N)`，但前提条件比比较排序严格得多。
- 稳定性是排序算法在工程场景里非常重要的一个维度，尤其当元素不是基础类型时。

## 图片转写 / 章节笔记还原
### 1. 不基于比较的排序
原图信息可以整理为：

- 桶排序思想下的排序，典型代表是计数排序和基数排序。
- 这类排序都不是通过“比较两个样本大小”来完成排序的。
- 在前提满足时，时间复杂度可以做到 `O(N)` 级别。
- 代价通常是要付出额外空间，例如 `O(M)` 或辅助数组空间。
- 适用范围有限，要求样本数据本身满足可分桶、可按位拆解等条件。

继续细化后：

- 计数排序通常要求：样本是整数，且值域范围比较窄。
- 基数排序通常要求：样本可以按位拆解，课程里一般先讲十进制非负整数。
- 一旦题目要求升级，比如值域更大、数据类型更复杂、需要处理更多边界，改写成本会明显增加。

### 2. 排序算法的稳定性
原图信息可以整理为：

- 稳定性是指：相同大小的样本，在排序后相对次序不变。
- 对基础类型来说，稳定性通常意义不大。
- 对非基础类型来说，稳定性很重要，因为“主关键字相同但附加信息不同”的对象很多。
- 有些排序算法天然可以做稳定，有些算法则无论怎么改都很难稳定。

可以这样理解：

- 如果排序对象只是纯数字，稳定与否往往无所谓。
- 如果排序对象是“学生分数、订单时间、任务优先级”这类结构体，稳定性就可能决定业务结果是否正确。

### 3. 排序算法总结
原图给出了一份排序算法对比表，整理如下：

| 排序算法 | 时间复杂度 | 额外空间复杂度 | 稳定性 |
| --- | --- | --- | --- |
| 选择排序 | `O(N^2)` | `O(1)` | 否 |
| 冒泡排序 | `O(N^2)` | `O(1)` | 是 |
| 插入排序 | `O(N^2)` | `O(1)` | 是 |
| 归并排序 | `O(N log N)` | `O(N)` | 是 |
| 随机快排 | `O(N log N)` | `O(log N)` | 否 |
| 堆排序 | `O(N log N)` | `O(1)` | 否 |
| 计数排序 | `O(N)` | `O(M)` | 是 |
| 基数排序 | `O(N)` | `O(N)` | 是 |

原图结论还包括：

1. 不基于比较的排序，对样本数据要求严格，不容易随意改写。
2. 基于比较的排序更通用，只要定义清楚比较规则就能直接复用。
3. 基于比较的排序，理论时间下界是 `O(N log N)`。
4. 同时满足“时间复杂度 `O(N log N)`、额外空间低于 `O(N)`、并且稳定”的基于比较排序，并不存在。
5. 如果追求绝对速度常常会考虑快排；如果强调省空间会考虑堆排；如果强调稳定性通常会考虑归并。

### 4. 常见的坑
原图中的几个提醒可以整理为：

- 有人会说“归并排序额外空间可以做到 `O(1)`”，这种所谓“归并排序内部缓存法”通常会让代码极其复杂，而且还会失去稳定性。
- 所谓“原地归并排序”常常只是概念贴，很多实现会把时间复杂度拖到 `O(N^2)`，工程上并不划算。
- 快速排序也存在稳定化改造思路，但通常会对样本数据、空间使用或实现复杂度提出更多要求。

还有一个常见题型：

- 在整型数组中，把奇数放左边、偶数放右边；
- 并要求奇数之间原相对次序不变，偶数之间原相对次序不变；
- 同时要求时间复杂度 `O(N)`、额外空间复杂度 `O(1)`。

这类题的难点就在于：

- 它既想要“分类 partition”
- 又想要“稳定性”
- 还想要“原地低空间”

这几个目标很容易互相冲突，所以往往是故意拿来考察你是否真的理解稳定性和空间复杂度之间的取舍。

## 记忆点
- Trie 是字符串前缀工具，不是排序工具。
- 计数排序和基数排序都快，但快得有条件。
- 稳定性在对象排序里很重要，在纯数字排序里常常没那么重要。
- 排序算法没有绝对最优，工程里永远是在“速度、空间、稳定性、适用条件”之间取平衡。
