## 一句话结论
这题是机器人走路问题，最适合第一次体验“递归 -> 记忆化搜索 -> 经典 DP”这条完整转化链。
因为它的状态天然就是两个参数：当前位置 `cur` 和剩余步数 `rest`。

## 核心知识点
- 位置在 `1...N` 的直线上。
- 机器人从 `M` 出发，必须走 `K` 步，到达 `P`。
- 到两端时只有一个方向可走。
- 状态只和 `cur、rest` 有关。

## 图片转写 / 题意还原
原图内容为：

- 假设有排成一行的 `N` 个位置，记为 `1 ~ N`
- 机器人开始时在 `M` 位置
- 到 `1` 只能往 `2`
- 到 `N` 只能往 `N-1`
- 在中间位置可以左或右走
- 必须走 `K` 步，最终到达 `P`
- 返回方法数

原始笔记还强调了：

- 方法 1：使用数组缓存优化，最粗糙的动态规划，也就是记忆化搜索
- 方法 2：直接写动态规划表格
- 不是所有暴力递归都能改成动态规划

## 解题思路
### 1. 暴力递归
定义：

```text
process(cur, rest)
表示当前在 cur，还剩 rest 步时，走到 P 的方法数
```

### 2. 记忆化搜索
用 `dp[cur][rest]` 缓存已经算过的状态。

### 3. 经典 DP
建立二维表：

- 行：位置 `1...N`
- 列：剩余步数 `0...K`

边界：

```text
dp[P][0] = 1
其他 dp[x][0] = 0
```

再按步数从小到大填表。

## 代码 / 伪代码
```java
int process(int N, int cur, int rest, int P) {
    if (rest == 0) return cur == P ? 1 : 0;
    if (cur == 1) return process(N, 2, rest - 1, P);
    if (cur == N) return process(N, N - 1, rest - 1, P);
    return process(N, cur - 1, rest - 1, P)
         + process(N, cur + 1, rest - 1, P);
}
```

## 记忆点
- 这是递归转 DP 的标准入门题。
- 状态是 `cur + rest`。
- 记忆化搜索是粗粒度缓存。
- 经典 DP 是把状态表显式填出来。
