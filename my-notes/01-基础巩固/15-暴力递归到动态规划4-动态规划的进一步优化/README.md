## 章节导读
这一章是在给整个“暴力递归 -> 动态规划”主线收尾，总结的是方法论而不只是某一道题：

- 什么时候记忆化搜索反而比经典 DP 更自然
- 如何设计递归状态参数
- 如何按模型归类题目
- 未来还可以往哪些优化方向继续走

## 本章结论
- 经典 DP 不一定永远优于记忆化搜索，状态稀疏时记忆化可能更好。
- 设计递归参数时，可变参数越少越好，类型越简单越好。
- 找 DP 的流程很固定：先合法递归，再看重复解，再做缓存，再组织成表。
- 常见尝试模型的识别，比死背某题的代码更有价值。
- 多样本位置全对应和业务限制模型，是后续更复杂 DP 的重点。

## 图片转写 / 章节笔记还原
原始章节页主要讲了三条主线：

### 1. 暴力递归和动态规划的关系
- 某个暴力递归有解的重复调用，就可以把它优化成动态规划
- 任何动态规划问题，都一定对应着某个有重复调用的暴力递归
- 但不是所有暴力递归都对应着动态规划

### 2. 设计递归和找 DP 的方法
1. 先设计不违反原则的暴力递归
2. 分析是否有重复解
3. 用记忆化搜索
4. 用严格表结构实现经典 DP
5. 再看能不能继续优化

### 3. 常见尝试模型的总表
1. 从左往右
2. 范围模型
3. 多样本位置全对应
4. 寻找业务限制
