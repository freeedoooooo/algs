## 一句话结论
快速排序 3.0 的核心升级，不是 partition 规则本身，而是“随机选择 pivot”。
它的目的，是在概率上避开每次都选到极端划分的坏情况，从而把快排的表现稳定在接近 `O(N log N)`。

## 核心知识点
- 快排退化成 `O(N^2)` 的根本原因，是 pivot 选得太差。
- 如果每次都选到当前区间的最小值或最大值，划分会极不平衡。
- 随机快排通过随机选择 pivot，让坏情况很难持续发生。
- 实战里最常搭配三路 partition 一起使用。

## 为什么需要随机选 pivot
假设数组本来就是有序的：

```text
[1, 2, 3, 4, 5, 6]
```

如果你每次固定选最左或最右作为 `pivot`，
那么每次划分都会变成：

```text
一边几乎空
一边几乎是全部
```

递归树会退化成链：

```text
N + (N-1) + (N-2) + ...
```

总复杂度就会变成：

```text
O(N^2)
```

随机选 pivot 的意义就是：

- 不让输入数据结构性地针对你
- 把最坏划分变成低概率事件

## 解题思路
### 1. 在当前区间随机挑一个数
不是固定挑：

- 最左
- 最右
- 中间

而是从 `[l...r]` 中随机选一个位置。

仓库里的写法是：

```java
int x = arr[l + (int)(Math.random() * (r - l + 1))];
```

### 2. 再做 partition
随机出的 `pivot` 仍然交给 partition 去划分。

在仓库实现里：

- 快排 1.0 的经典 partition 也用了随机值
- 快排 2.0 的推荐版同样用了随机值 + 荷兰国旗

### 3. 继续递归左右区间
流程本身没变：

- 选 `pivot`
- partition
- 递归子区间

变化只在于：

- `pivot` 不再是固定策略选出来的

## 典型例子
数组：

```text
[1, 2, 3, 4, 5, 6, 7]
```

如果总是选最左 `1`：

```text
[] | 1 | [2,3,4,5,6,7]
```

下一层又选最左 `2`：

```text
[] | 2 | [3,4,5,6,7]
```

会一路退化。

但如果随机选到 `4`：

```text
[1,2,3] | 4 | [5,6,7]
```

划分就很均衡，递归树高度会显著降低。

随机快排不是保证每次都均衡，
而是保证“长期看，不均衡情况不会总发生”。

## 为什么标题里说“先随机取一个数，放到最右侧 R 位置”
这是很多教材或代码里的经典写法：

1. 先随机从区间里挑一个元素
2. 把它交换到最右侧
3. 统一把 `arr[r]` 当作 pivot 来写 partition

这样代码更规整。

仓库当前实现没有显式“先交换到右侧再 partition”，
而是直接把随机选中的值取出来作为 `x` 使用。

本质完全一样：

- 重点不是物理上放在哪
- 重点是 `pivot` 是随机选出来的

## 复杂度
- 期望时间复杂度：`O(N log N)`
- 最坏时间复杂度：`O(N^2)`
- 额外空间复杂度：递归栈期望 `O(log N)`

注意：

- 随机化降低的是最坏情况发生的概率
- 不是把最坏情况从理论上消灭掉

## 易错点
- 随机快排的“随机”作用在 pivot 选择，不是作用在整个数组乱排。
- 随机选 pivot 后，partition 逻辑本身并没有改变。
- 不要误以为随机化后就绝对不会退化，只是概率上更稳。

## 代码 / 伪代码
仓库里的对应实现：

- [QuickSort.java](../../../solutions-java/practice/algs01_sort/QuickSort.java)

核心随机选 pivot 的写法：

```java
int x = arr[l + (int)(Math.random() * (r - l + 1))];
```

随机快排 3.0 的推荐理解方式是：

```java
void quickSort(int[] arr, int l, int r) {
    if (l >= r) {
        return;
    }
    int pivot = arr[l + random(0, r - l)];
    int[] range = dutchNationalFlag(arr, l, r, pivot);
    quickSort(arr, l, range[0] - 1);
    quickSort(arr, range[1] + 1, r);
}
```

## 和 1.0 / 2.0 的关系
- 1.0：经典两路 partition
- 2.0：三路 partition，返回等于区间
- 3.0：在 2.0 的基础上，再把 `pivot` 随机化

所以最推荐的完整版本通常是：

```text
随机选 pivot + 荷兰国旗三路划分
```

## 记忆点
- 1.0 解决“怎么分”
- 2.0 解决“重复值怎么少递归”
- 3.0 解决“最坏情况怎么尽量避开”
