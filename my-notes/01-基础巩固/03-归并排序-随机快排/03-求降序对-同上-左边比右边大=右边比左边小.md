## 一句话结论
降序对问题和最小和问题本质上是同一类题，都是利用归并排序在 `merge` 时批量统计跨左右区间的信息。
区别只在于：最小和统计的是“左边小于等于右边”的贡献，而降序对统计的是“左边大于右边”的配对数量。

## 题目含义
如果数组中存在一对下标：

```text
i < j
```

且满足：

```text
arr[i] > arr[j]
```

那么 `(arr[i], arr[j])` 就构成一个降序对，也常被叫作逆序对。

例如数组：

```text
[3, 1, 2]
```

降序对有：

- `(3, 1)`
- `(3, 2)`

所以答案是 `2`。

## 核心知识点
- 暴力枚举所有二元组需要 `O(N^2)`。
- 归并排序能把问题拆成左右两部分 + 跨区间部分。
- 左右两边内部的降序对由递归计算。
- 真正高效的是：在 `merge` 时一次性统计“左边某个数能和右边多少个数配对”。

## 为什么这题也能用归并排序
和小和题完全一样，答案可以拆成三部分：

1. 左半部分内部的降序对
2. 右半部分内部的降序对
3. 左边和右边跨区间形成的降序对

前两部分递归搞定。

关键是第三部分：

- 左边下标天然小于右边下标
- 所以只要左值大于右值，就能直接形成降序对

## merge 时怎么批量统计
假设当前左右两边已经分别有序：

```text
左：1 3 5
右：2 4 6
```

现在比较：

```text
左[i] = 3
右[j] = 2
```

因为：

```text
3 > 2
```

而左边从 `i` 往后都是有序的，所以：

```text
3 后面的 5 也一定 > 2
```

这说明：

```text
左边从 i 到末尾的所有元素，都能和 2 形成降序对
```

于是这一轮可以一次性加上：

```text
左边剩余元素个数
```

这就是批量统计的本质。

## 解题思路
### 1. 递归拆分
不断把数组一分为二，直到区间只剩一个元素。

单个元素不可能形成降序对，所以返回 `0`。

### 2. 递归求左右部分答案
分别计算：

- 左半部分内部有多少降序对
- 右半部分内部有多少降序对

### 3. merge 时统计跨区间答案
设：

- 左指针 `p1` 指向左半部分
- 右指针 `p2` 指向右半部分

如果：

```text
arr[p1] <= arr[p2]
```

说明当前左值不能和当前右值形成降序对，先正常拷贝较小值。

如果：

```text
arr[p1] > arr[p2]
```

说明：

```text
左边从 p1 到中点的所有元素，都 > arr[p2]
```

因此一次性增加：

```text
m - p1 + 1
```

然后把右侧元素先放进辅助数组。

## 典型例子
数组：

```text
[5, 3, 2, 4]
```

最终某轮 merge 时可能出现：

```text
左：3 5
右：2 4
```

### 比较 3 和 2
因为：

```text
3 > 2
```

而左边后面还有 `5`，
所以 `2` 会和左边剩余 `[3, 5]` 全部形成降序对。

一次性增加：

```text
2
```

对应的配对就是：

- `(3, 2)`
- `(5, 2)`

### 再比较 3 和 4
这时：

```text
3 <= 4
```

不能新增降序对，正常拷贝 `3`。

### 再比较 5 和 4
因为：

```text
5 > 4
```

左边剩余只有 `5` 一个数，
所以新增：

```text
1
```

对应配对：

- `(5, 4)`

这一轮 merge 的跨区间贡献就是 `3`。

## 和小和题的关系
这两题本质完全相同，区别只在“统计规则”：

### 小和
- 看到左值 `<=` 右值
- 统计左值对右边剩余元素的贡献

### 降序对
- 看到左值 `>` 右值
- 统计右值和左边剩余元素能形成多少对

所以题目标题里说：

```text
左边比右边大 = 右边比左边小
```

本质就是同一件事换了个观察角度。

## 复杂度
- 时间复杂度：`O(N log N)`
- 空间复杂度：`O(N)`

因为整体框架仍然是归并排序，只是在 `merge` 时多做了常数级统计。

## 易错点
- 统计的是“左边剩余多少元素都大于当前右值”，不是只加 `1`。
- 一旦统计完成，当前应先拷贝右边元素，因为它更小。
- 左右两边必须已经有序，这个批量统计才成立。
- 要分清“翻转对（大于 2 倍）”和普通逆序对/降序对，它们不是同一道题。

## 代码 / 伪代码
仓库里虽然直接给的是“翻转对 + 小和”的实现，但思想完全一致：

- [ReversePairs.java](../../../solutions-java/leetcode/p493_reverse_pairs/ReversePairs.java)
- [MergeSort.java](../../../solutions-java/practice/algs01_sort/MergeSort.java)

普通降序对在 `merge` 中的核心写法可以理解为：

```java
int mergeCount(int[] arr, int l, int m, int r) {
    int p1 = l;
    int p2 = m + 1;
    int[] help = new int[r - l + 1];
    int i = 0;
    int ans = 0;

    while (p1 <= m && p2 <= r) {
        if (arr[p1] <= arr[p2]) {
            help[i++] = arr[p1++];
        } else {
            ans += m - p1 + 1;
            help[i++] = arr[p2++];
        }
    }

    while (p1 <= m) {
        help[i++] = arr[p1++];
    }
    while (p2 <= r) {
        help[i++] = arr[p2++];
    }

    // 拷回原数组
    return ans;
}
```

## 和记忆点
- 小和：左值给右边批量贡献。
- 降序对：右值和左边批量配对。
- 一切关键都在 `merge` 时的“批量结算”。
