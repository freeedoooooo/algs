## 一句话结论
归并排序的核心不是“拆开再拼回去”这么简单，而是它在合并两个有序部分时，没有浪费比较结果。
一旦两个元素在归并过程中比出大小，这个结果会直接服务于后续整体有序的构建，所以总复杂度能稳定做到 `O(N log N)`。

## 核心知识点
- 归并排序是典型的分治算法。
- 每次先把数组一分为二，分别排好，再合并。
- 归并过程依赖“左右两边已经有序”这个前提。
- 归并排序时间复杂度稳定是 `O(N log N)`。
- 归并排序的很多衍生题，本质都在利用 `merge` 这一步。

## 为什么说“比较行为没有浪费”
这句话是归并排序最值得理解的地方。

假设左半部分和右半部分已经分别有序：

```text
左：1 4 7
右：2 3 8
```

在合并时你比较：

- `1` 和 `2`

一旦发现 `1 < 2`，你不仅知道 `1` 应该先放进结果数组，
还顺带知道：

- `1` 也小于右边此时后面还没出场的 `3`
- `1` 也小于 `8`

也就是说，这次比较产生的信息会被整体保留下来，不会像某些排序那样“比完就算了，后面还得重新比很多次”。

这正是归并排序高效的根本原因，也是后面“最小和”“逆序对”这类题能借助 `merge` 一步算答案的原因。

## 解题思路
### 1. 分
把数组不断二分，直到每个子数组只剩一个元素。

因为：

- 一个元素天然就是有序的

### 2. 治
左右两边分别递归排好序。

### 3. 合
用两个指针分别指向左右有序部分的开头：

- 谁小就先拷贝谁
- 对应指针向后移动

当一边耗尽后：

- 另一边剩下的元素直接全部拷贝过去

最后把辅助数组中的结果写回原数组。

## 典型例子
数组：

```text
[5, 2, 4, 1]
```

### 第一步：拆分

```text
[5, 2]    [4, 1]
```

继续拆：

```text
[5] [2]   [4] [1]
```

### 第二步：各自合并成有序

```text
[5] 和 [2] -> [2, 5]
[4] 和 [1] -> [1, 4]
```

### 第三步：最终 merge
合并：

```text
[2, 5] 和 [1, 4]
```

比较过程：

- `2` 和 `1`，取 `1`
- `2` 和 `4`，取 `2`
- `5` 和 `4`，取 `4`
- 剩下 `5`

得到：

```text
[1, 2, 4, 5]
```

## 为什么复杂度是 O(N log N)
从递归树角度看：

- 一共会分成 `log N` 层
- 每一层所有 `merge` 加起来，总工作量都是 `O(N)`

所以总复杂度：

```text
O(N log N)
```

对应递推式：

```text
T(N) = 2T(N / 2) + O(N)
```

套 Master 公式可得：

```text
T(N) = O(N log N)
```

## 归并排序的实质
很多人只记住了“归并排序会排序”，但更重要的是它的实质：

- 不是暴力比较所有元素对
- 而是借助“左右部分分别有序”这个条件
- 在 `merge` 的过程中一次性得到大量有用信息

这就是后面这些题为什么都能用归并思想：

- 最小和
- 逆序对 / 降序对
- 某些区间统计题

因为它们都在利用：

```text
一旦左右有序，跨区间的信息就能在 merge 时被批量结算
```

## 复杂度
- 时间复杂度：`O(N log N)`
- 空间复杂度：`O(N)`
- 稳定性：稳定排序

为什么稳定：

- 如果写 `merge` 时在相等情况下优先拷贝左侧元素
- 那么相同元素的相对顺序就能保持

## 易错点
- `merge` 的前提是左右两部分已经各自有序。
- 写回原数组时范围必须是当前区间 `l...r`，不能写错边界。
- 中点写法要注意防止溢出，通常写成 `l + (r - l) / 2`。
- 归并排序不是原地排序，需要额外辅助数组。
- 不理解 `merge` 的价值，后面就很难看懂最小和、逆序对这些题。

## 代码 / 伪代码
仓库里的对应实现：

- [MergeSort.java](../../../solutions-java/practice/algs01_sort/MergeSort.java)

仓库里同时给出了：

- 递归版 `mergeSortRecursive`
- 迭代版 `mergeSortIterative`

核心 `merge` 过程如下：

```java
int i = l;
int j = m + 1;
int k = l;
while (i <= m && j <= r) {
    if (arr[i] < arr[j]) {
        temp[k++] = arr[i++];
    } else {
        temp[k++] = arr[j++];
    }
}
while (i <= m) {
    temp[k++] = arr[i++];
}
while (j <= r) {
    temp[k++] = arr[j++];
}
for (int p = l; p <= r; p++) {
    arr[p] = temp[p];
}
```

## 和记忆点
- 先分到最小，再往上归并。
- 每层总工作量都是 `O(N)`。
- 真正要理解的是 `merge`，不是“递归”两个字。
- 后面最小和、降序对，都是在 `merge` 上做文章。
