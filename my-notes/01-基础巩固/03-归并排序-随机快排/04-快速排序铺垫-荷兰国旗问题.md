## 一句话结论
荷兰国旗问题的核心，是把无序数组按某个划分值 `pivot` 一次性分成三段：

- 小于 `pivot`
- 等于 `pivot`
- 大于 `pivot`

这正是快速排序高效 partition 的基础。

## 核心知识点
- `partition` 的本质是按某个标准对数组重新分区。
- 荷兰国旗问题是三路划分，不是简单的两路划分。
- 这道题最关键的是维护好每个区间的边界含义。
- 快速排序之所以能高效，很大程度上靠的就是 partition 做得好。

## 为什么要先学这题
如果只会“把小于某个值的放左边，大于某个值的放右边”，那只是初级 partition。

但在快速排序里：

- 如果数组里有很多和 `pivot` 相等的元素
- 只做两路划分会导致重复递归这些相等元素

而三路划分可以把：

```text
等于 pivot 的部分一次性锁定
```

后续递归只需要处理：

- 小于区
- 大于区

这就是荷兰国旗问题的价值。

## 区间定义
设当前处理区间是：

```text
[l ... r]
```

并给定划分值 `pivot`。

用三个指针维护：

- `low`：小于区的右边界
- `high`：大于区的左边界
- `i`：当前正在检查的位置

在仓库实现里，循环过程中维护的含义可以理解为：

```text
[l ... low-1]      小于 pivot
[low ... i-1]      等于 pivot
[i ... high]       未处理
[high+1 ... r]     大于 pivot
```

这 4 段定义是整道题最关键的地方。

## 图解
假设：

```text
arr = [3, 5, 2, 1, 3, 4, 3]
pivot = 3
```

初始时可以把数组理解成：

```text
[ 未处理 | 未处理 | 未处理 | 未处理 | 未处理 | 未处理 | 未处理 ]
  ^
  i
```

随着指针推进，数组会逐渐被分成 4 段：

```text
[ <3 区 | =3 区 | 未处理区 | >3 区 ]
```

例如某一时刻可能变成：

```text
[2, 1 | 3, 3 | 5, 4, 3 | ]
             ^
             i
```

如果当前 `arr[i]`：

- 小于 `3`：丢到左边 `<3` 区
- 等于 `3`：留在中间 `=3` 区
- 大于 `3`：丢到右边 `>3` 区

最终收敛成：

```text
[2, 1 | 3, 3, 3 | 4, 5]
```

关键感受是：

```text
左边界负责扩小于区
右边界负责扩大于区
i 负责扫描未知区
```

## 解题思路
### 1. 当前值小于 pivot
如果：

```text
arr[i] < pivot
```

说明它应该进入小于区。

做法：

1. 交换 `arr[i]` 和 `arr[low]`
2. `low++`
3. `i++`

为什么 `i` 可以前进：

- 因为换过来的位置已经属于“等于区的下一个位置”
- 并且原 `arr[i]` 已经被放到了正确的小于区

### 2. 当前值等于 pivot
如果：

```text
arr[i] == pivot
```

它天然就属于中间区域。

做法：

```text
i++
```

### 3. 当前值大于 pivot
如果：

```text
arr[i] > pivot
```

说明它应该进入大于区。

做法：

1. 交换 `arr[i]` 和 `arr[high]`
2. `high--`
3. `i` 不动

为什么 `i` 不能动：

- 因为从右边换过来的新元素还没检查过
- 不能假设它已经在正确区域

这恰恰是很多人第一次写错的地方。

## 典型例子
数组：

```text
[3, 5, 2, 1, 3, 4, 3]
```

设：

```text
pivot = 3
```

目标是分成：

```text
小于 3 | 等于 3 | 大于 3
```

### 初始

```text
low = l
high = r
i = l
```

### 遇到 3
等于 `pivot`，直接 `i++`。

### 遇到 5
大于 `pivot`，和 `high` 位置交换，`high--`，`i` 不动。

### 遇到 2
小于 `pivot`，和 `low` 位置交换，`low++`，`i++`。

不断做下去，最后整个数组会变成：

```text
[2, 1, 3, 3, 3, 4, 5]
```

此时中间 `[3, 3, 3]` 这段就是“等于区”。

## 和快速排序的关系
快速排序的核心流程就是：

1. 选一个 `pivot`
2. 用 partition 把数组分区
3. 只递归处理需要继续排序的那两边

如果使用荷兰国旗问题的三路划分：

- 小于区递归
- 等于区跳过
- 大于区递归

这就比简单二路划分更稳定，也更适合处理重复元素很多的情况。

## 复杂度
- 单次 partition 时间复杂度：`O(N)`
- 额外空间复杂度：`O(1)`

因为整个过程只是一趟扫描，加上常数个指针变量。

## 易错点
- `arr[i] > pivot` 时，交换后 `i` 不能自增。
- 没有把每个区间的语义想清楚，就容易写乱边界。
- 三路划分不是排序完成，只是完成了一次按 `pivot` 的分区。
- `low`、`high` 的初始定义不同，代码细节也会随之变化，关键是始终保证区间语义自洽。

## 代码 / 伪代码
仓库里对应实现：

- [QuickSort.java](../../../solutions-java/practice/algs01_sort/QuickSort.java)

核心方法就是：

```java
public static int[] dutchNationalFlag(int[] arr, int l, int r, int pivot) {
    int low = l;
    int high = r;
    int i = l;

    while (i <= high) {
        if (arr[i] < pivot) {
            swap(arr, i, low);
            low++;
            i++;
        } else if (arr[i] > pivot) {
            swap(arr, i, high);
            high--;
        } else {
            i++;
        }
    }
    return new int[]{low, high};
}
```

返回的 `[low, high]` 就是“等于区”的左右边界。

## 记忆点
- 小于：换到左边，`low++`，`i++`
- 等于：跳过，`i++`
- 大于：换到右边，`high--`，`i` 不动

只要把这三句记住，荷兰国旗问题就基本拿下了。
