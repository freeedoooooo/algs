## 章节导读
这一章是排序体系里非常关键的一章，因为它把两条重要主线放到了一起：

- 分治：归并排序
- 划分：快速排序

如果按知识结构来看，可以这样记：

```text
O(N log N) 排序体系
├─ 归并排序
│  ├─ 分成两半
│  ├─ 各自排好
│  └─ merge 时顺带统计信息
└─ 快速排序
   ├─ 选 pivot
   ├─ partition
   ├─ 等于区间优化
   └─ 随机化避免退化
```

## 本章结论
- 归并排序的核心价值，不只是排序稳定，而是 `merge` 阶段可以批量结算跨区间信息。
- 最小和、降序对这类题，本质上都不是“排序题”，而是“借排序过程做统计题”。
- 快速排序的核心不在递归，而在 partition。
- 快排 1.0、2.0、3.0 是一条连续升级路线：单点划分 -> 等于区间 -> 随机 pivot。
- 这一章会直接影响后面你对分治、递归和优化思路的理解。

## 章节笔记还原
### 1. 归并排序在训练“分治 + merge”
归并排序的基本流程是：

```text
分
排
合
```

但这一章真正想强调的不是流程本身，而是：

- 为什么归并排序复杂度稳定是 `O(N log N)`
- 为什么 `merge` 过程没有浪费比较结果

一旦你理解了这点，后面的“最小和”“降序对”就很好接受，因为它们都是在 `merge` 的时候顺便把跨左右区间的信息统计掉。

### 2. 这一章里“merge sort 的实质”非常关键
最小和和降序对放在这里，不是为了多讲两个题，而是为了让你看到：

- 排序过程本身可以携带额外信息
- 递归分区后的“跨区间关系”可以在合并阶段统一处理

这也是很多进阶题的共同套路：

```text
先把问题拆到左右子问题
再在合并阶段处理横跨两边的贡献
```

一旦这一点理解透了，归并排序就不再只是“排序算法”，而是一种题目模型。

### 3. 快速排序在训练“partition 思维”
这一章的后半部分是一条完整升级路线：

1. 先学荷兰国旗问题
2. 再写快排 1.0
3. 再升级到快排 2.0
4. 最后用随机化得到快排 3.0

这条路线真正想讲的是：

- 快排的核心不是递归壳子
- 快排的核心是 partition 怎么做

其中最关键的升级是：

- 从“返回一个位置”
- 到“返回一个等于区间”

这样所有等于 `pivot` 的元素都一次性归位，递归范围会更干净。

### 4. 随机快排是在解决“退化问题”
快排 3.0 的重点不是写法变复杂，而是引入了概率思想：

- 如果每次都选到极端 `pivot`
- 划分就会严重失衡
- 快排就可能退化成 `O(N^2)`

随机选 `pivot` 的目的，就是在概率上避免长期遇到坏划分，让整体表现更稳定地接近 `O(N log N)`。

## 记忆点
- 归并排序强在 `merge`，快速排序强在 `partition`。
- 最小和、降序对，本质是“借排序做统计”。
- 快排 2.0 的关键是“等于区间”。
- 快排 3.0 的关键是“随机 pivot”。
