## 一句话结论
在大根堆里，堆顶就是当前最大值，所以“弹出最大值”最自然的做法就是拿走堆顶。
但数组实现里不能真的删除中间元素，因此标准操作是：交换堆顶和末尾，缩小堆大小，再对新的堆顶执行一次 `heapify`。

## 核心知识点
- 大根堆的堆顶永远是当前最大值。
- 数组实现的堆最适合删除最后一个位置，不适合直接删除开头再整体左移。
- 所以 `pop max` 的标准动作一定是“堆顶和末尾交换”。
- 交换后只有堆顶位置可能破坏堆序，因此只需要对堆顶执行一次 `heapify`。

## 图解
假设当前大根堆数组为：

```text
[9, 7, 8, 3, 5, 6]
```

对应树形结构：

```text
        9
      /   \
     7     8
    / \   /
   3   5 6
```

### 第 1 步：堆顶和末尾交换

```text
[6, 7, 8, 3, 5, 9]
```

此时：

```text
前 5 个位置 = 还在堆里
最后 1 个位置 = 已弹出的最大值 9
```

### 第 2 步：对新的堆顶做 heapify
当前树变成：

```text
        6
      /   \
     7     8
    / \
   3   5
```

因为 `6` 比孩子小，所以要下沉：

```text
        8
      /   \
     7     6
    / \
   3   5
```

对应数组：

```text
[8, 7, 6, 3, 5, 9]
```

这就是一次完整的 `pop max`。

## 为什么不能直接删堆顶
如果堆是数组：

```text
[9, 7, 8, 3, 5, 6]
```

堆顶 `9` 是最大值。

如果你想“直接删除第 0 位置”，那么后面的元素都要整体前移，
这会让数组实现变得很低效。

因此真正高效的办法是：

1. 把堆顶和最后一个元素交换
2. 让最后一个位置脱离堆范围
3. 对新的堆顶重新调整

## 解题思路
### 1. 交换堆顶与末尾
因为堆顶是最大值，把它和当前堆的最后一个元素交换后：

- 最大值就被放到了数组最右边
- 这个位置以后不再参与堆结构

### 2. 堆大小减 1
交换完成后：

- 有效堆的范围缩小
- 原末尾位置已经变成有序区

### 3. 对新的堆顶执行 `heapify`
交换后，新的堆顶是原来末尾那个元素。

它可能比较小，
因此会破坏“大根堆里父节点 >= 子节点”的规则。

但是注意：

- 除了堆顶之外，其余节点原本的相对结构都没有破坏

所以只需要从堆顶开始向下调整一次就够了。

## 为什么只需要 heapify 堆顶
因为在交换之前：

- 除了堆顶和末尾，其余位置都满足大根堆结构

交换之后：

- 最大值被放到末尾，已经离开堆范围
- 只有新放到堆顶的那个元素可能不合规

也就是说：

```text
问题只发生在根节点这一路向下
```

因此从 `0` 位置开始执行一次 `heapify`，
就能把整棵堆重新恢复好。

## 典型例子
假设当前大根堆数组：

```text
[9, 7, 8, 3, 5, 6]
```

### 第一步：交换堆顶与末尾

```text
[6, 7, 8, 3, 5, 9]
```

此时：

- `9` 已经确定是当前最大值
- 堆的有效范围变成前 5 个元素

### 第二步：heapify 堆顶
对 `[6, 7, 8, 3, 5]` 这部分从根开始向下调整：

- `6` 和左右孩子 `7、8` 比
- 选最大的 `8` 上来

得到：

```text
[8, 7, 6, 3, 5, 9]
```

现在前 5 个元素又重新变成了大根堆。

这样下一轮还可以继续弹出最大值 `8`。

## 和堆排序的关系
堆排序其实就是反复做这一个动作：

1. 弹出当前最大值
2. 放到数组右边的有序区
3. 对新堆顶 `heapify`

所以这题虽然单独拆开讲，
本质上就是堆排序每一轮循环的核心步骤。

## 复杂度
- 取堆顶最大值：`O(1)`
- 交换堆顶与末尾：`O(1)`
- 一次 `heapify`：`O(log N)`
- 整体一次 `pop max`：`O(log N)`

## 易错点
- 交换之后要先缩小 `size`，再按新的堆范围 `heapify`。
- `heapify` 的起点是 `0`，不是末尾。
- 末尾位置已经是有序区，不应该再参与后续堆调整。
- 不要误以为交换后整棵堆都乱了，真正需要修复的主要是堆顶往下这一条路径。

## 代码 / 伪代码
仓库里的对应实现就在：

- [HeapSort.java](../../../solutions-java/practice/algs01_sort/HeapSort.java)

这一步的核心代码，其实就是堆排序循环里的这几行：

```java
size = size - 1;
swap(arr, 0, size);
heapify(arr, 0, size);
```

如果单独写成“弹出最大值”的逻辑，可以理解为：

```java
int popMax(int[] arr, int size) {
    int ans = arr[0];
    swap(arr, 0, size - 1);
    size--;
    heapify(arr, 0, size);
    return ans;
}
```

## 记忆点
- 堆顶就是最大值
- 最大值先换到末尾
- 堆大小减 1
- 新堆顶向下 `heapify`
