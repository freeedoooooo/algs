## 章节导读
这一章虽然标题叫“堆、比较器”，但当前条目里的主线其实非常集中，核心就是：

- 理解堆是什么
- 理解堆排序怎么做
- 理解堆在工程题里怎么用

如果压缩成结构图，可以这样看：

```text
堆体系
├─ 结构认知
│  └─ 完全二叉树 + 堆序性质
├─ 排序流程
│  ├─ 建堆
│  ├─ 弹出堆顶
│  └─ heapify
├─ 性能特点
│  └─ 原地、额外空间 O(1)
└─ 工程扩展
   ├─ 几乎有序数组
   └─ 自定义堆 / resign
```

## 本章结论
- 堆本质上是“数组上的完全二叉树”，不是普通意义上的链式树。
- 堆排序最重要的两个动作是：建堆、`heapify`。
- 堆排序的突出优势是额外空间复杂度 `O(1)`。
- 小根堆特别适合解决“动态维护当前最小值”这一类题。
- 一旦元素入堆后还会改值，语言自带堆通常不够，需要自己维护索引和 `resign`。

## 章节笔记还原
### 1. 前半章是在讲“堆是什么”
这一章起手先讲大根堆、小根堆和完全二叉树，是为了把堆从概念上讲清楚：

- 形状上必须是完全二叉树
- 顺序上还要满足堆序性质

也就是说，堆不是“任意一个树结构”，而是有非常明确的结构约束。

对应条目：

- [构建堆 & 大/小根堆，就是完全二叉树，从左往右变满](01-构建堆-&-大-小根堆-就是完全二叉树-从左往右变满.md)

### 2. 中间几题是在讲“堆排序怎么跑起来”
堆排序这一组题的逻辑顺序非常明确：

1. 先把数组建成大根堆
2. 堆顶就是最大值
3. 把堆顶和末尾交换
4. 缩小堆范围
5. 对新的堆顶继续 `heapify`

所以这几题其实是在拆解同一个流程：

- 堆顶为什么是最大值
- 弹出堆顶后为什么要 `heapify`
- 建堆为什么可以做到 `O(N)`
- 堆排序为什么能原地完成

### 3. 后半章是在讲“堆怎么拿来解决题”
如果只会用堆做排序，其实还没真正掌握堆。

堆真正强的地方在于：

- 它能动态维护当前最大 / 最小元素
- 它能只关注局部候选集

所以“几乎有序数组”这题非常经典，因为它让你看到：

- 并不一定每次都要全量排序
- 如果信息足够强，可以只维护一个大小为 `K+1` 的小根堆

对应条目：

- [移动距离小于K的排序，定义一个 size=K+1 的数组，然后使用小根堆](06-移动距离小于K的排序-定义一个-size=K+1-的数组-然后使用小根堆.md)

### 4. `resign` 是这一章最工程化的点
很多语言自带的 `PriorityQueue` 只擅长：

- 放进去
- 弹出来

但如果一个元素已经在堆里，后来优先级变了怎么办？

这时就会遇到这一章最关键的工程点：

```text
元素改值后
如何快速找到它
并重新调整到正确位置
```

这就是 `resign` 的意义。

它通常需要配合：

- 反向索引表
- 比较器规则
- 向上调整和向下调整

对应条目：

- [自定义堆，动态修改的堆排序，关键方法 resign](07-自定义堆-动态修改的堆排序-关键方法-resign.md)

## 记忆点
- 堆 = 完全二叉树 + 堆序性质。
- 堆排序重点记两个词：建堆、`heapify`。
- 堆排序省空间，小根堆擅长维护局部最优。
- `resign` 是“堆中元素改值后重新归位”的关键动作。
