## 一句话结论
堆排序的核心流程是：

1. 先把整个数组建成大根堆
2. 再不断把堆顶最大值交换到末尾
3. 缩小堆范围后继续 `heapify`

其中“从后往前执行 `heapify`”这件事，就是高效建堆的关键。

### 图 1：堆排序的整体步骤
1. 先让整个数组变成大根堆。

建堆有两种方法：

- 从上到下建堆，时间复杂度 `O(N log N)`
- 从下到上建堆，时间复杂度 `O(N)`

2. 把堆顶最大值和堆末尾的值交换。

然后堆大小减 1，再针对新的堆顶执行调整，循环往复。

这一阶段的时间复杂度是：

```text
O(N log N)
```

3. 当堆的大小减小到只剩 1 个元素时，排序完成。

### 图 2：为什么从后往前 `heapify` 建堆是 O(N)
直观理解如下：

- 树的底层节点（叶子节点）不需要调整
- 靠近底层的节点只需要少量调整
- 只有靠近根节点的少数节点才需要较多调整

由于完全二叉树的大部分节点都分布在底层，
所以整体调整代价不会达到 `O(N log N)`，而是均摊为 `O(N)`。

## 核心知识点
- 堆排序依赖大根堆。
- 每次把堆顶最大值放到数组末尾，就相当于确定了一个最终位置。
- `heapify` 是“某个位置变小后，向下恢复堆结构”的操作。
- 从后往前建堆，本质上是从最后一个非叶子节点开始，依次向上 `heapify`。

## 堆排序流程
### 1. 建堆
让整个数组先变成大根堆。

这样堆顶 `arr[0]` 一定是当前最大值。

### 2. 交换堆顶和末尾
把堆顶元素与当前堆的最后一个元素交换：

```text
最大值 -> 数组末尾
```

这说明末尾位置已经有序，不再参与后续堆结构。

### 3. 缩小堆大小并继续调整
交换之后，新的堆顶元素可能破坏堆序，
所以要对堆顶执行一次 `heapify`，恢复大根堆。

然后重复：

- 交换堆顶和末尾
- 堆大小减 1
- 堆顶 `heapify`

直到堆大小为 `1`。

## 为什么是“从后往前执行 heapify”
数组建堆时，如果采用高效版本，
会从最后一个位置开始一路往前：

```java
for (int i = n - 1; i >= 0; i--) {
    heapify(arr, i, n);
}
```

虽然看起来对每个位置都调用了 `heapify`，
但并不是每个位置都会真的走满 `O(log N)`。

原因是：

- 叶子节点根本不需要往下调整
- 越靠下的节点，高度越低
- 真正调整得多的只有上层少量节点

所以总成本是：

```text
O(N)
```

而不是 `O(N log N)`。

## 典型例子
假设数组：

```text
[3, 5, 1, 2, 4]
```

### 第一步：建成大根堆
调整后可能得到：

```text
[5, 4, 1, 2, 3]
```

### 第二步：交换堆顶与末尾

```text
[3, 4, 1, 2, 5]
```

此时：

- `5` 已经到最终位置
- 堆大小减为 4

### 第三步：对堆顶 heapify
前 4 个元素重新恢复成大根堆：

```text
[4, 3, 1, 2, 5]
```

然后继续重复，最终数组会变成升序。

## 复杂度
- 从顶到底建堆：`O(N log N)`
- 从后往前 `heapify` 建堆：`O(N)`
- 排序阶段：`O(N log N)`
- 整体堆排序时间复杂度：`O(N log N)`
- 额外空间复杂度：`O(1)`

## 易错点
- 堆排序用的是大根堆，不是小根堆。
- 交换堆顶和末尾后，堆的有效范围要减 1。
- `heapify` 的 `size` 必须传当前堆大小，而不是整个数组长度。
- “从后往前 heapify”讲的是建堆阶段，不是说排序时从后往前遍历所有节点。

## 代码 / 伪代码
仓库里的对应实现：

- [HeapSort.java](../../../solutions-java/practice/algs01_sort/HeapSort.java)

这题对应的核心实现是 `heapSort2`：

```java
public static void heapSort2(int[] arr) {
    int n = arr.length;
    for (int i = n - 1; i >= 0; i--) {
        heapify(arr, i, n);
    }
    int size = n;
    while (size > 1) {
        size = size - 1;
        swap(arr, 0, size);
        heapify(arr, 0, size);
    }
}
```

## 记忆点
- 建堆：从后往前 `heapify`
- 排序：堆顶和末尾交换
- 缩小堆，再 `heapify`
- 最大值一个个被送到数组右边
