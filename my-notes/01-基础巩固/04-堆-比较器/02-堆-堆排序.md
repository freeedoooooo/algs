## 一句话结论
堆排序可以先记成一句非常顺的话：

```text
先把数组建成大根堆
再把堆顶最大值不断送到数组右边
每送走一个，就对新的堆顶做一次 heapify
```

其中最容易混淆的地方是：

- 建堆阶段：让“无序数组”先变成大根堆
- 排序阶段：每一轮只做“交换堆顶、缩小堆、恢复堆”

## 核心知识点
- 堆排序依赖的是大根堆，而不是小根堆。
- 大根堆的堆顶永远是当前堆里的最大值。
- 一次“把堆顶放到末尾”的操作，就等于确定了一个最终位置。
- 建堆可以有两种做法：
  - 一个个 `heapInsert`，总复杂度 `O(N log N)`
  - 从后往前整体 `heapify`，总复杂度 `O(N)`
- 真正排序时，每一轮的核心都只有三步：交换、缩小范围、`heapify`。

## 图片转写 / 题意还原
这篇原始内容真正要讲的是：

```text
为什么堆排序能一边维护堆，
一边把最大值一个个放到数组最右边，
最后得到升序结果。
```

如果把整个算法压缩成流程图，就是：

```text
无序数组
  ↓
建成大根堆
  ↓
堆顶最大值换到末尾
  ↓
堆的有效范围减 1
  ↓
对新堆顶做 heapify
  ↓
重复，直到堆大小为 1
```

## 图解
先用一组固定数组把整体感觉建立起来：

```text
[3, 5, 1, 2, 4]
```

把它看成完全二叉树：

```text
      3
    /   \
   5     1
  / \
 2   4
```

### 建堆后
从后往前执行 `heapify`，可以把它调整成大根堆：

```text
      5
    /   \
   4     1
  / \
 2   3
```

对应数组：

```text
[5, 4, 1, 2, 3]
```

### 第一轮排序
堆顶 `5` 是当前最大值，把它和末尾交换：

```text
[3, 4, 1, 2, 5]
```

此时：

```text
前 4 个位置 = 还在堆里
最后 1 个位置 = 已经排好
```

然后只对前 4 个位置的堆顶做 `heapify`：

```text
[4, 3, 1, 2, 5]
```

### 第二轮排序
继续把堆顶 `4` 送到右边：

```text
[2, 3, 1, 4, 5]
```

前 3 个位置重新 `heapify`：

```text
[3, 2, 1, 4, 5]
```

### 第三轮排序
把堆顶 `3` 送到右边：

```text
[1, 2, 3, 4, 5]
```

前 2 个位置再做一次 `heapify`：

```text
[2, 1, 3, 4, 5]
```

### 最后一轮
把堆顶 `2` 和末尾交换：

```text
[1, 2, 3, 4, 5]
```

排序完成。

这时你应该能感受到：

```text
右边的有序区是从后往前一点点长出来的
每一轮只是把“当前最大值”送到右边最前面那个空位
```

## 解题思路
### 1. 先建成大根堆
为什么一定要先建大根堆？

因为只有大根堆才能保证：

```text
arr[0] 一定是当前最大值
```

这就让“找最大值”这件事从扫描整个数组，变成了直接看堆顶。

高效建堆的写法是：

```java
for (int i = n - 1; i >= 0; i--) {
    heapify(arr, i, n);
}
```

这里的含义不是“排序时从后往前遍历”，而是：

```text
建堆时，从较底层的节点开始，
逐步把每棵子树都调整成合法堆
```

### 2. 交换堆顶和末尾
建好大根堆之后：

- 堆顶是最大值
- 末尾是当前堆的最后一个位置

所以直接交换：

```text
swap(arr, 0, size - 1)
```

交换之后：

- 最大值到了数组最右边
- 这个位置就不再属于堆

这一步最重要的理解是：

```text
不是“删除最大值”
而是“把最大值移出堆的有效范围”
```

### 3. 堆大小减 1
因为末尾位置已经有序，所以堆的有效范围要缩小：

```text
size--
```

现在数组会被逻辑上分成两段：

```text
[0 ... size-1]   还在堆里
[size ... n-1]   已排序区
```

### 4. 只对堆顶做一次 `heapify`
交换后，原来末尾那个值被放到了堆顶。  
真正可能破坏堆序的，主要就是根节点往下这一条路径。

所以不需要重建整个堆，只要：

```text
heapify(arr, 0, size)
```

让新的堆顶一路下沉到正确位置即可。

### 5. 重复直到堆大小为 1
之后的每一轮都完全一样：

1. 堆顶和末尾交换
2. 堆大小减 1
3. 堆顶 `heapify`

直到：

```text
size == 1
```

排序结束。

## 为什么“从后往前 heapify”能高效建堆
很多人第一次看到这里会下意识想：

```text
一共有 N 个位置
每次 heapify 最坏 O(log N)
那不就是 O(N log N) 吗？
```

但实际不是，因为：

- 叶子节点根本不用下沉
- 靠近底层的节点，最多只下沉 1~2 层
- 只有很少的上层节点，才可能走比较长的路径

所以总成本并不会达到 `O(N log N)`，而是：

```text
O(N)
```

你可以把它理解成：

```text
大部分节点都“几乎不用调”
真正调得多的节点只有少数
```

## 完整过程示例
继续用同一个数组，把堆排序完整串起来：

初始数组：

```text
[3, 5, 1, 2, 4]
```

### 阶段 1：建堆

```text
[5, 4, 1, 2, 3]
```

### 阶段 2：第 1 轮
交换堆顶和末尾：

```text
[3, 4, 1, 2, 5]
```

有效堆范围变成前 4 个位置。  
对堆顶 `heapify` 后：

```text
[4, 3, 1, 2, 5]
```

### 阶段 3：第 2 轮
交换堆顶和末尾：

```text
[2, 3, 1, 4, 5]
```

前 3 个位置 `heapify` 后：

```text
[3, 2, 1, 4, 5]
```

### 阶段 4：第 3 轮
交换堆顶和末尾：

```text
[1, 2, 3, 4, 5]
```

前 2 个位置 `heapify` 后：

```text
[2, 1, 3, 4, 5]
```

### 阶段 5：第 4 轮
交换堆顶和末尾：

```text
[1, 2, 3, 4, 5]
```

结束。

如果把这个过程压缩成一句话，就是：

```text
先把最大值放到最右边
再把剩余部分恢复成新的大根堆
```

## 复杂度
- 从顶到底建堆：`O(N log N)`
- 从后往前整体 `heapify` 建堆：`O(N)`
- 排序阶段：`O(N log N)`
- 整体时间复杂度：`O(N log N)`
- 额外空间复杂度：`O(1)`

## 易错点
- 堆排序必须用大根堆，才能把最大值送到数组右边形成升序。
- 建堆阶段的“从后往前 `heapify`”和排序阶段的循环不是一回事。
- 交换堆顶和末尾后，要先缩小 `size`，再按新的堆范围做 `heapify`。
- `heapify(arr, 0, size)` 里的 `size` 是当前堆大小，不是整个数组长度。
- 一次交换后不是整个堆都乱了，真正需要修复的是堆顶往下这一条路径。

## 代码 / 伪代码
仓库里的对应实现：

- [HeapSort.java](../../../solutions-java/practice/algs01_sort/HeapSort.java)

推荐理解这版 `heapSort2`：

```java
public static void heapSort2(int[] arr) {
    int n = arr.length;

    // 1. 从后往前整体建堆
    for (int i = n - 1; i >= 0; i--) {
        heapify(arr, i, n);
    }

    // 2. 不断把堆顶最大值送到数组右边
    int size = n;
    while (size > 1) {
        size--;
        swap(arr, 0, size);
        heapify(arr, 0, size);
    }
}
```

## 记忆点
- 先建大根堆，再反复把堆顶送到最右边。
- 每一轮固定动作：交换、缩小堆、堆顶 `heapify`。
- 右边的有序区会越来越长。
- 建堆 `O(N)`，整体排序 `O(N log N)`。
