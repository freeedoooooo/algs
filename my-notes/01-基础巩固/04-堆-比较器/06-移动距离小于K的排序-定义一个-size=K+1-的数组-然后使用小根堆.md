## 一句话结论
如果一个数组几乎有序，并且每个元素离它最终位置的距离都不超过 `K`，那么最适合的策略不是全量排序，而是维护一个大小为 `K+1` 的小根堆。
这样总时间复杂度可以做到 `O(N log K)`。

- 并且 `K` 相对于数组长度来说较小

要求选择一种合适的排序策略，对这个数组进行排序。

## 核心知识点
- 元素最终位置最多偏移 `K`，意味着某个位置的答案只可能出现在它后面的 `K` 个范围内。
- 因此每次只需要看一个长度为 `K+1` 的窗口。
- 在这个窗口里取最小值，最适合用小根堆。
- 整体时间复杂度是 `O(N log K)`，比普通 `O(N log N)` 更优。

## 为什么是 K+1 个元素
假设当前要确定排序后第 `0` 个位置上的值。

因为每个元素最多只会偏移 `K`，
所以原数组中真正应该放在第 `0` 位的元素，
一定只会出现在原数组的：

```text
[0 ... K]
```

也就是前 `K+1` 个元素里。

因此：

- 先把前 `K+1` 个元素放进小根堆
- 堆顶就是第一个应该输出的最小值

接下来每往后推进一步，
再补一个新元素进堆，
就能继续得到下一个位置的答案。

## 解题思路
### 1. 先把前 K+1 个元素放进小根堆
因为第一个正确位置的答案一定在这里。

### 2. 每次弹出堆顶写回数组
堆顶是当前窗口里的最小值，
也就是当前最该放到排序结果里的元素。

### 3. 同时把下一个新元素放进堆
这样堆中始终维护着“当前答案可能出现的候选范围”。

### 4. 最后把堆里剩余元素依次弹完
当原数组扫描结束后，
堆中剩余的元素本来就已经是最后那部分的有序答案。

## 典型例子
假设数组：

```text
[2, 1, 4, 3, 6, 5]
```

并且：

```text
K = 1
```

说明每个元素最多离目标位置偏移 1。

### 第一步
先把前 `K+1 = 2` 个元素放入小根堆：

```text
[2, 1]
```

堆顶是 `1`，所以排序后第一个位置就是 `1`。

### 第二步
弹出 `1`，再加入下一个元素 `4`：

堆里变成：

```text
[2, 4]
```

堆顶是 `2`，所以第二个位置就是 `2`。

继续这个过程，
最终就能输出：

```text
[1, 2, 3, 4, 5, 6]
```

## 复杂度
- 时间复杂度：`O(N log K)`
- 空间复杂度：`O(K)`

因为堆的大小始终不超过 `K+1`。

## 易错点
- 不是 `K` 个元素，而是 `K+1` 个元素。
- `K = 0` 时，数组本来就已经有序。
- 这里最适合的是小根堆，不是大根堆。
- 题目成立的前提是“每个元素的位移都不超过 K”，不是“整体看起来差不多有序”。

## 代码 / 伪代码
仓库里的对应实现就在：

- [HeapSort.java](../../../solutions-java/practice/algs01_sort/HeapSort.java)

对应方法：

```java
public void sortArrDistanceLessK(int[] arr, int k)
```

核心逻辑：

```java
PriorityQueue<Integer> heap = new PriorityQueue<>();
int index = 0;
for (; index <= Math.min(k, arr.length - 1); index++) {
    heap.add(arr[index]);
}
int i = 0;
for (; index < arr.length; i++, index++) {
    heap.add(arr[index]);
    arr[i] = heap.poll();
}
while (!heap.isEmpty()) {
    arr[i++] = heap.poll();
}
```

## 和记忆点
- 位移不超过 `K`
- 当前正确答案一定在前 `K+1` 个候选里
- 用小根堆维护这个窗口
- 复杂度从 `log N` 降到 `log K`
