## 章节导读
这一章开始从“会写递归”走向“会优化递归”：

- 先用若干脑筋题热身“分析能力”
- 再看 `N` 皇后这种高质量搜索题
- 再判断什么递归值得优化
- 最后第一次把递归直接改写成 DP

## 本章结论
- 不是所有暴力递归都能改成动态规划，关键看有没有重复子问题。
- `N` 皇后是典型搜索题，位运算优化能显著加速。
- 记忆化搜索是最粗糙的 DP 形式。
- 经典 DP 则是在分析状态和依赖后，按表格顺序填出来。

## 图片转写 / 章节笔记还原
原始笔记页最关键的三条信息是：

### 1. `N` 皇后问题
- 在 `N * N` 棋盘上摆 `N` 个皇后
- 任意两个皇后不能同行、同列、同斜线
- 求摆法数量

### 2. 什么递归可以继续优化
- 如果有重复调用同一个子问题的解，这种递归可以优化
- 如果每个子问题都不同，就无从优化

### 3. 第一次改成 DP 的题目
- 给定 `N, M, K, P`
- 机器人从 `M` 出发，必须走 `K` 步，到达 `P`
- 每次在一条直线上左右移动
- 求方法数

## 记忆点
- 有重复解，才谈 DP。
- `N` 皇后练搜索剪枝。
- 记忆化搜索是自顶向下。
- 经典 DP 是自底向上。
