## 一句话结论
岛问题的基础版，本质是二维网格上的连通块计数：把每一片上下左右连在一起的 `1` 看成一个岛，问总共有多少个岛。
经典做法是感染 / DFS / BFS；而图里给出的进阶版，则是在问数据量很大时，如何用并行切分和并查集把局部结果拼成全局答案。

## 核心知识点
- 只有 `0` 和 `1` 两种值，通常 `1` 表示陆地，`0` 表示水。
- 连通关系只看上、下、左、右四个方向，不看对角线。
- 数岛屿本质上是数“由 1 组成的连通分量”。
- 基础版常用 DFS / BFS / 感染法，把访问过的陆地整片标记掉。
- 进阶版可把大矩阵分块并行处理，再在分块边界上用并查集合并跨块岛屿。

### 题目五
岛问题

【题目】

一个矩阵中只有 `0` 和 `1` 两种值，每个位置都可以和自己的上、下、左、右四个位置相连，如果有一片 `1` 连在一起，这个部分叫做一个岛，求一个矩阵中有多少个岛？

【举例】

```text
001010
111010
100100
000000
```

这个矩阵中有三个岛。

【进阶】

如何设计一个并行算法解决这个问题？

把题目补成完整说明，就是：

- 输入一个只含 `0/1` 的二维矩阵
- 每个位置只能和上下左右四邻接方向建立连通
- 所有连在一起的 `1` 算作同一个岛
- 输出岛屿数量
- 进阶问题进一步要求考虑：矩阵非常大时如何并行计算

## 图解
看图中的例子：

```text
001010
111010
100100
000000
```

把坐标连通关系展开后，可以分成三片：

### 岛 1

```text
001010
111010
100100
000000
```

左下这大片连在一起的 `1`：

```text
  1
111
1
```

它们彼此通过上下左右能走通，所以只算 1 个岛。

### 岛 2

右上中间这条竖着的：

```text
001010
111010
100100
000000
```

对应两格：

```text
第1行第4列
第2行第4列
```

它们上下相连，所以是第 2 个岛。

### 岛 3

第三行中间单独的一个 `1`：

```text
100100
```

它和其他 `1` 都不相连，所以单独算第 3 个岛。

因此总答案是：

```text
3
```

## 解题思路
### 1. 基础版：感染法 / DFS
最经典的思路是遍历矩阵：

1. 从左到右、从上到下扫描
2. 遇到一个还没访问过的 `1`
3. 岛屿数量加一
4. 以它为起点，把与它连通的整片 `1` 全部标记掉

这个“整片标记”的过程可以用：

- 递归 DFS
- 栈模拟 DFS
- 队列 BFS

都可以。

### 2. 为什么这样不会重复计数
因为每次发现一个新的 `1`，都会把整片连通区域全部改成“已访问”状态。

所以之后再扫描到这片区域时，就不会再次加答案。

这就保证了：

```text
每个岛只会被统计一次
```

### 3. 感染函数怎么写
如果当前位置是 `(i, j)`：

- 越界就返回
- 不是 `1` 就返回
- 如果是 `1`，先把它改掉，例如改成 `2`
- 再递归感染上下左右四个方向

这其实就是在网格图上做连通块搜索。

### 4. 进阶版为什么会想到并查集
当矩阵极大时，单机递归 / DFS 仍然能做，但可能：

- 数据太大，单机慢
- 想分给多台机器并行

这时可以把矩阵切成多个子块：

```text
块1 | 块2 | 块3
----+----+----
块4 | 块5 | 块6
```

每个子块内部先独立算局部岛屿，并给局部陆地分配编号。

然后只在分块边界上检查：

- 哪些 `1` 实际属于同一片跨块岛屿

这些跨块合并关系，就可以交给并查集来完成。

### 5. MapReduce 风格怎么理解
可以把进阶版看成两个阶段：

第一阶段：局部计算

- 每个计算单元处理自己那一块
- 算出本块内局部岛屿编号

第二阶段：边界合并

- 收集相邻分块边界上的连通关系
- 用并查集把局部岛屿合并成全局岛屿

最后得到总岛数。

所以它是一个很典型的：

```text
局部统计 + 边界归并
```

模型。

## 复杂度
### 基础版
- 时间复杂度：`O(R * C)`
- 空间复杂度：
  - 递归写法最坏 `O(R * C)`
  - 显式栈 / 队列写法最坏也可能到 `O(R * C)`

其中：

- `R` 是行数
- `C` 是列数

因为每个位置最多被访问一次。

### 进阶并行版
- 局部处理仍然是线性于各分块大小
- 额外代价主要在边界合并和并查集维护

整体思路仍然保持近似线性，但能通过多机并行缩短墙钟时间。

注意中间的 `0` 把上下左右路径切断了吗？

其实没有，因为外围一圈 `1` 仍然通过左右和上下连成一整片，所以答案是：

```text

```

### 例 3

```text

0011
```

```text

```

## 易错点
- 只看上下左右，不看左上右上这类对角线。
- 感染法里必须先把当前格子标记掉，再继续递归，否则会死循环。
- 如果直接修改原矩阵，题目若要求保留输入，就要先拷贝或使用 `visited` 数组。
- 进阶版并行处理时，难点不在块内统计，而在跨块边界的正确合并。
- 岛问题本质是连通分量计数，不要把它只当成一个“矩阵遍历小技巧”。

## 代码 / 伪代码
基础感染法框架如下：

```java
int countIslands(int[][] grid) {
    int ans = 0;
    for (int i = 0; i < grid.length; i++) {
        for (int j = 0; j < grid[0].length; j++) {
            if (grid[i][j] == 1) {
                ans++;
                infect(grid, i, j);
            }
        }
    }
    return ans;
}

void infect(int[][] grid, int i, int j) {
    if (i < 0 || i == grid.length || j < 0 || j == grid[0].length) {
        return;
    }
    if (grid[i][j] != 1) {
        return;
    }
    grid[i][j] = 2;
    infect(grid, i - 1, j);
    infect(grid, i + 1, j);
    infect(grid, i, j - 1);
    infect(grid, i, j + 1);
}
```

进阶并行版可以抽象为：

```text
分块
-> 各块独立统计局部岛
-> 收集边界连通关系
-> 用并查集合并局部编号
-> 得到全局岛数量
```

## 记忆点
- 岛问题 = 二维网格上的连通分量计数。
- 只看上下左右四联通。
- 基础版：扫描到 `1` 就感染整片。
- 进阶版：分块并行 + 边界并查集合并。
