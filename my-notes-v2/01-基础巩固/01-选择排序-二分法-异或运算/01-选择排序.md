## 一句话结论
选择排序的核心是把数组分成“已排好序的左边”和“还没处理的右边”，每一轮都从右边选出最小值，放到左边当前应在的位置。
它实现简单、空间复杂度低，但比较次数固定较多，所以实际性能通常不如插入排序和快速排序。

## 核心知识点
- 每一轮只确定一个位置上的最终值。
- 第 `i` 轮的任务，是在区间 `i...n-1` 中找到最小值。
- 交换次数最多是 `n-1` 次，但比较次数大约是 `n(n-1)/2`。
- 选择排序通常是不稳定排序。

## 直观理解
可以把它理解成“从一把乱牌里，每次挑出最小的一张，依次摆到最前面”。

和其他排序的区别：

- 选择排序：每轮选最小值，再放到前面。
- 插入排序：每拿到一张新牌，就往前插到合适位置。
- 计数排序：不是靠比较，而是靠“统计出现次数”来恢复顺序。

## 解题思路
### 1. 划分两个区域
从左到右看数组：

- 左边部分：已经排好序。
- 右边部分：还没处理。

初始时，左边为空，右边是整个数组。

### 2. 每轮选最小值
假设当前来到位置 `i`：

- 先认为 `i` 位置就是最小值位置。
- 再往右扫描 `i+1...n-1`。
- 如果发现更小的数，就更新最小值下标。

扫描结束后，就得到了本轮最小值的位置 `minIndex`。

### 3. 把最小值放到正确位置
将 `minIndex` 和 `i` 交换。

这样一来：

- `i` 位置就确定好了。
- 下一轮只需要处理 `i+1...n-1`。

## 典型例子
以数组 `[5, 3, 1, 4, 2]` 为例：

### 第1轮
- 在 `[5, 3, 1, 4, 2]` 中找最小值，找到 `1`
- 和第 0 位交换
- 结果变成 `[1, 3, 5, 4, 2]`

### 第2轮
- 在 `[3, 5, 4, 2]` 中找最小值，找到 `2`
- 和第 1 位交换
- 结果变成 `[1, 2, 5, 4, 3]`

### 第3轮
- 在 `[5, 4, 3]` 中找最小值，找到 `3`
- 和第 2 位交换
- 结果变成 `[1, 2, 3, 4, 5]`

后面再继续时，数组已经有序。

## 复杂度
- 时间复杂度：`O(N^2)`
- 空间复杂度：`O(1)`
- 稳定性：不稳定

为什么时间复杂度是 `O(N^2)`：

- 第一轮比较 `n-1` 次
- 第二轮比较 `n-2` 次
- ...
- 最后一轮比较 `1` 次

总比较次数为：

```text
(n - 1) + (n - 2) + ... + 1 = n(n - 1) / 2
```

所以整体是二次复杂度。

## 易错点
- 内层循环的起点应该是当前轮次的位置右边，或者从当前最小值下标开始继续扫，关键是不要漏掉当前候选最小值。
- 如果最小值本来就在 `i` 位置，交换可以省略。
- 选择排序不是稳定排序，因为交换可能改变相等元素的相对顺序。

## 代码 / 伪代码
仓库里已有对应实现：

- [SelectionSort.java](D:\PROJECT\Project_HU\freeedoooooo\algs\solutions-java\practice\algs01_sort\SelectionSort.java)

核心伪代码如下：

```java
for (int i = 0; i < n - 1; i++) {
    int minIndex = i;
    for (int j = i + 1; j < n; j++) {
        if (arr[j] < arr[minIndex]) {
            minIndex = j;
        }
    }
    swap(arr, i, minIndex);
}
```

这段逻辑和仓库中的 `selectionSort` / `mySelectionSort` 方法是一致的，都是每轮先找最小值，再交换到当前边界位置。
