## 一句话结论
二分查找的本质，不是“数组有序”这四个字，而是每次看中间位置后，都能排除掉一半不可能的答案区间。
所谓“排他性的判断标准”，就是你必须能根据 `mid` 位置的性质，明确知道答案一定不在左边或者一定不在右边。

## 核心知识点
- 二分的核心是“能排除一半”，不是“机械地取中点”。
- 只要问题满足某种单调性、边界性或局部趋势，也可以用二分。
- “局部最小”类问题虽然数组不整体有序，但仍然可以构造出二分判断。
- 面试里常见的变体有：找某值、找最左满足条件的位置、找最右满足条件的位置、找峰值/谷值。

## 什么叫排他性的判断标准
如果一个问题想用二分，你必须回答下面这个问题：

> 当我检查 `mid` 之后，能不能确定某一半区域里绝对不可能有答案？

如果能，就能继续二分。

### 经典有序数组为什么能二分
在有序数组里查找 `target`：

- 如果 `arr[mid] > target`，那么右半边一定都更大，答案不可能在右边。
- 如果 `arr[mid] < target`，那么左半边一定都更小，答案不可能在左边。

这就是最标准的排他性。

## 为什么“局部最小”也能二分
局部最小常见定义：

- 对于数组中的某个位置 `i`
- 如果它比左边小，且比右边小
- 那么 `i` 就是一个局部最小

通常还会附带一个条件：

- 相邻元素不相等

这样局部趋势会更清晰。

### 关键观察
假设我们看中点 `mid`：

- 如果 `arr[mid] > arr[mid - 1]`，说明从 `mid-1` 到 `mid` 是“往上走”的。
- 既然往左走能下降，那么左边一定存在局部最小。

反过来：

- 如果 `arr[mid] > arr[mid + 1]`，说明右边有下降趋势。
- 那么右边一定存在局部最小。

如果：

- `arr[mid] < arr[mid - 1]`
- 且 `arr[mid] < arr[mid + 1]`

那 `mid` 自己就是答案。

这就实现了排他性：

- 要么答案在左边
- 要么答案在右边
- 要么答案就是中点

## 解题思路
### 1. 先处理边界
局部最小问题一般先看边界：

- 如果数组只有一个数，那么它就是局部最小。
- 如果 `arr[0] < arr[1]`，那么 0 位置就是局部最小。
- 如果 `arr[n - 1] < arr[n - 2]`，那么最后一个位置就是局部最小。

这样可以把后续二分范围缩到中间区域。

### 2. 在中间做二分
令搜索范围为 `left...right`，每次取 `mid`：

- 若 `arr[mid] < arr[mid - 1] && arr[mid] < arr[mid + 1]`，返回 `mid`
- 若 `arr[mid] > arr[mid - 1]`，去左边找
- 否则去右边找

这里“否则去右边找”的前提，是相邻元素不相等。

### 3. 为什么这样不会漏
因为一旦某一侧存在下降趋势，那一侧就一定能走到一个局部最小：

- 要么一路下降到边界，边界就是局部最小
- 要么下降到某个点后开始上升，这个拐点就是局部最小

## 典型例子
数组：

```text
[9, 7, 3, 4, 6, 8]
```

看中点 `mid = 2`，值为 `3`：

- `3 < 7`
- `3 < 4`

所以 `3` 就是局部最小，直接返回。

再看一个例子：

```text
[8, 6, 4, 5, 7, 9]
```

中点如果落在值 `5` 的位置：

- `5 > 4`

说明左边存在下降趋势，因此左边一定存在局部最小，可以把右半边整体排除。

## 和普通二分的关系
这道题最重要的不是“记住局部最小怎么做”，而是理解二分的通用套路：

1. 找到一个可判断的位置 `mid`
2. 根据 `mid` 的性质排除一半
3. 保证剩余那一半里一定还有答案

有序数组查找只是最经典的一种；局部最小、峰值元素、边界查找，都是这个思想的变体。

## 复杂度
- 时间复杂度：`O(log N)`
- 空间复杂度：`O(1)`

## 易错点
- 如果题目没有说明“相邻元素不相等”，很多结论就不能直接照搬。
- 局部最小和全局最小不是一回事，只要比相邻位置小就够了。
- 边界位置只有一个邻居，因此判断规则和中间位置不同。
- 二分时不要写成死循环，区间更新必须严格缩小。

## 代码 / 伪代码
仓库里和这题最接近的实现文件是：

- [FindPeekElement.java](../../../solutions-java/leetcode/p162_find_peek_element/FindPeekElement.java)

它实现的是“峰值元素”，但思想完全一致：通过中点附近的趋势判断，排除一半区间。

局部最小的伪代码可以写成：

```java
if (n == 1) return 0;
if (arr[0] < arr[1]) return 0;
if (arr[n - 1] < arr[n - 2]) return n - 1;

int left = 1;
int right = n - 2;

while (left <= right) {
    int mid = left + (right - left) / 2;
    if (arr[mid] < arr[mid - 1] && arr[mid] < arr[mid + 1]) {
        return mid;
    } else if (arr[mid] > arr[mid - 1]) {
        right = mid - 1;
    } else {
        left = mid + 1;
    }
}
return -1;
```

## 速记方法
- 二分不是看“有序”，而是看“能不能排除一半”。
- 局部最小不是背模板，而是看“下降趋势指向哪里”。
